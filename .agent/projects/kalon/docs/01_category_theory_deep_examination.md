## 1. 随伴関手とは何か — 本質の理解

### 1.1 定義（hom-set bijection）

二つの圏 C, D と関手 F: D→C, G: C→D が随伴 (F ⊣ G) であるとは：

```
Hom_C(F(d), c) ≅ Hom_D(d, G(c))    ∀ d∈D, c∈C
```

**意味**: F(d) から c への射（C の中での変換）と、d から G(c) への射（D の中での変換）が **一対一に対応する**。

→ 二つの異なる世界で考えることが **等価** になる。

### 1.2 直観（Free ⊣ Forgetful の例）

```
F: Set → Grp    （自由群生成 — 構造を追加）
U: Grp → Set    （忘却関手 — 構造を剥がす）

F ⊣ U

意味: 「集合 S から群 G への群準同型」
    ≅ 「集合 S から U(G) への関数」
```

**なぜこれが強力か**: 群の言葉で考えるのが難しい問題を、集合の言葉に翻訳できる。そして翻訳は可逆。情報の損失なし。

### 1.3 Unit と Counit

```
η: id_D → G ∘ F    （unit — 埋め込み）
ε: F ∘ G → id_C    （counit — 評価）
```

- η は「元の世界から、変換して戻った世界への埋め込み」
- ε は「戻って再変換した結果を、元に戻す操作」
- 三角恒等式: ε_F ∘ F(η) = id_F, G(ε) ∘ η_G = id_G

### 1.4 Phillips & Wilson の洞察

> **Systematicity**: ある認知能力を持つことは、構造的に関連する別の認知能力を持つことを必然的に含意する

Phillips & Wilson (2010) は、この systematicity を圏論の universal construction（随伴関手を含む）で説明した。

- 古典的説明（Fodor）: 言語的合成性 → ad hoc
- 結合主義的説明: ネットワーク構造 → ad hoc
- 圏論的説明: 随伴関手の構造から systematicity が **必然的に** 導かれる

## 2. CCL `+` ⊣ `-` の対応 — 精緻な検証

### 2.1 現在の主張（浅い版）

旧ドキュメントの主張:

```
+ = Free 的操作（構造追加）
- = U 的操作（構造忘却）
+ ⊣ -
```

### 2.2 検証：これは随伴か？

随伴関手の定義に照らして検証する。

**必要条件**:

1. `+` と `-` は **関手** でなければならない（対象だけでなく射も写す）
2. 二つの **圏** が特定されなければならない
3. hom-set の bijection が成立しなければならない

**検証**:

| 条件 | 現状 | 判定 |
|:-----|:-----|:-----|
| `+` は関手か？ | `+` は CCL 演算子（WF に作用する修飾子）。入力WFと修飾後WFの間の写像。**だが射（WF間の変換）に対する作用が未定義** | ⚠️ 不完全 |
| `-` は関手か？ | 同上 | ⚠️ 不完全 |
| 二つの圏は何か？ | **未特定**。Free⊣U では Set と Grp が明確だが、CCL では「深い版の圏」と「浅い版の圏」が曖昧 | ❌ 未定義 |
| hom-set bijection | 二つの圏が未定義のため検証不可能 | ❌ 検証不可 |

### 2.3 診断

**`+` ⊣ `-` は、現時点では比喩である。構造ではない。**

理由: 随伴の定義に必要な3条件（関手性、圏の特定、bijection）のうち、**どれも厳密には満たされていない**。

### 2.4 しかし、完全に間違いでもない

比喩としての `+` ⊣ `-` が捉えている直観は正しい:

- `+` は「構造を豊かにする」操作（Free 的）
- `-` は「構造を忘れる」操作（Forgetful 的）
- 両者には「元に戻せるが同一にはならない」関係がある（η, ε の非恒等性）

**問題は、この直観を形式化する「圏」がまだ存在しないこと**。

## 3. どこで比喩が壊れるか

### 3.1 壊れるポイント

| ポイント | 説明 | 深刻度 |
|:---------|:-----|:-------|
| **圏の不在** | CCL の「対象」が何かが不明。WF か？ WF の出力か？ 認知状態か？ | 🔴 致命的 |
| **射の不在** | WF 間の「変換」が定義されていない。合成則も未定義 | 🔴 致命的 |
| **non-compositionality** | `+` と `-` の合成 `+-` が何を意味するか不明。恒等に戻るのか？ | 🟡 中程度 |
| **自然性の未検証** | η, ε が自然変換であるための可換図式が未検証 | 🟡 中程度 |

### 3.2 壊れないポイント（比喩として有効な部分）

| ポイント | 説明 |
|:---------|:-----|
| **方向性** | `+` と `-` が逆方向の操作であるという直観 |
| **非対称性** | `+-` ≠ `-+` である可能性（unit ≠ counit） |
| **最適性** | Free は「最も自由な」構造追加。CCL の `+` も「制約なしの深化」 |

## 4. 圏論が CCL に本当に必要なもの

### 4.1 Phillips & Wilson から学べること

彼らが示したのは:

> **認知のsystematicity = 圏論の universal construction の帰結**

これを CCL に翻訳すると:

> **CCL が systematic であるとは**: `/noe` を使えるなら、`/dia` も使えることが **構造的に保証** される

現状の CCL は、これを **規約** として保証している（「すべての hub command は4つの定理を持つ」）。圏論はこの規約を **必然** にする可能性がある。

### 4.2 CCL に圏を定義するなら

**候補**:

```
圏 Cog（認知の圏）:
  対象: 認知状態（WF 実行前後の状態）
  射:   WF の実行（状態間の変換）
  合成: WF のパイプライン実行
  恒等: 何もしない（no-op）
```

**この定義が機能するかの検証**:

| 公理 | 検証 | 判定 |
|:-----|:-----|:-----|
| 合成の結合律 | (f∘g)∘h = f∘(g∘h) — WF実行順序が結合的か？ | [仮説] 成立する（出力が次の入力になるため） |
| 恒等律 | id∘f = f = f∘id — no-op との合成は恒等か？ | [推定] 成立する |
| 射の閉包 | 合成結果も射か？ — 二つの WF の連続実行も WF か？ | [仮説] 成立する（CCL 式の結合） |

### 4.3 この圏の上に `+` ⊣ `-` を定義し直す

```
圏 Cog_simple: 対象 = 認知状態、射 = 基本WF
圏 Cog_rich:   対象 = 認知状態、射 = 詳細WF

+ : Cog_simple → Cog_rich    （深化 — 構造追加）
- : Cog_rich → Cog_simple    （縮約 — 構造忘却）

+ ⊣ - ⟺ Hom_rich(+(s), r) ≅ Hom_simple(s, -(r))
```

**意味**: 「単純な状態 s を深化させた +s から、豊かな状態 r への変換」は、
「s から、r を縮約した -r への変換」と一対一に対応する。

**直観的検証**: `/noe` (深い認識) を使って `/dia` (判定) に到達する方法と、
最初から `/dia` の簡約版に直接到達する方法は、情報的に等価か？

→ [仮説] **部分的に成立するが、完全ではない可能性がある**。
深化によって得られる「副産物」（洞察の蓄積）が縮約では失われるため。

## 5. 暫定的結論

### 5.1 圏論は CCL にとって何か

| 評価軸 | 結論 |
|:-------|:-----|
| **比喩としての有用性** | ✅ 高い。方向性・非対称性・最適性の直観を与える |
| **構造としての厳密性** | ❌ 現時点では不十分。圏と射が未定義 |
| **必要性** | [仮説] あり。Phillips & Wilson の systematicity 論証は、CCL の体系的整合性を保証するために必要 |
| **即座の実装可能性** | ❌ 圏 Cog の厳密な定義が先決 |

1. **圏 Cog の厳密な定義** — 対象と射を明確に
2. **`+` と `-` の関手性の証明** — 射に対する作用の定義
3. **hom-set bijection の検証** — 具体例で計算
4. **Phillips (2021) の精読** — "A category theory principle for cognitive science" で使われている framework

### 7.2 [仮説] Arche = Completeness

> **Beauty = 演繹量/表現コスト** (Arche) と **Limit/Colimitの最適性** が同じ構造

- Limit = max(情報)/min(コスト) = Beauty (合意の美しさ)
- Colimit = min(コスト)/max(到達範囲) = Beauty (包含の美しさ)
- 圏 Cog が complete かつ cocomplete ⟹ CCL のすべての操作に最適解が存在
- **Arche = completeness** — Hegemonikón の美の原理と圏論の完備性が等価

この一致自体が Arche の基準（少数から多くを演繹、必然性、選択の卓越性）を満たしている。

### 7.3 [仮説] モナド ≈ FEP の予測モデル

- FEP: 認知状態は常に **暫定的な予測** であり、感覚入力で更新される
- モナド: T(x) = **x に文脈を被せた暫定的な値**、μ = 二重文脈の統合
- 接続: T(x) = 「x についての暫定的な予測」、μ = 予測の予測を一つに統合

### 7.4 `+` の関手性の精緻化

Creator の問い: `+` が関手なら、異なるWFでも同じ構造で機能すべきか？

→ YES。しかし `+(f∘g) = +(f)∘+(g)` が成立するかは未検証。
全体深化と個別深化で接合点の扱いが異なる可能性 → **lax functor** の候補。
