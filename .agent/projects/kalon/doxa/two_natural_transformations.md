## 第一自然変換: η_客観 (主観より客観)

> **Base Table（客観）に存在するものだけが残り、主体の「こうであってほしい」という願望は壊れる。**

### 形式化

```
F_主観: Reality → Model   (主観で現実を見る関手)
F_客観: Reality → Model   (客観で現実を見る関手)

η_客観: F_主観 ⟹ F_客観   (自然変換)

∀X ∈ Reality:
  η_X: F_主観(X) → F_客観(X)
  = 主観的解釈を客観的構造に修正する
```

### Kalon での実証

| Field | 主観 (願望) | 客観 (残ったもの) |
|:------|:----------|:--------------|
| 1 | 「+ は随伴であってほしい」 | + は自然変換 |
| 2 | 「O は Z/4Z であってほしい」 | O は圏の閉路 |
| 5 | 「レイヤーは表現であってほしい」 | レイヤーは関手 |

### 自然性条件

この変換は **どの問題に適用しても同じやり方で作用する**:

- 数学の命名 → 条件を満たすか検証
- コード設計 → 実際に動くか検証
- 人生の判断 → 事実に基づくか検証

## 第二自然変換: η_抽象 (具体より抽象)

> **見るべきものは具体ではなく抽象。具体的な名前は壊れ、原理レベルの構造は残る。**

### 形式化

```
F_具体: Reality → Model   (具体的名前で現実を見る関手)
F_抽象: Reality → Model   (構造的本質で現実を見る関手)

η_抽象: F_具体 ⟹ F_抽象   (自然変換)

∀X ∈ Reality:
  η_X: F_具体(X) → F_抽象(X)
  = 具体的ラベルを構造的本質に昇華する
```

### Kalon での実証

| 具体 (壊れた) | 抽象 (残った) |
|:------------|:-----------|
| 随伴 (名前) | 自然変換 (構造) |
| Z/4Z (名前) | 圏の閉路 (構造) |
| 表現 (名前) | 関手 (構造) |
| 6D 空間 (名前) | [0,1]^6 多様体 (構造) |

### 自然性条件

この変換も **どの問題に適用しても同じやり方で作用する**:

- 数学 → 名前より構造
- 設計 → 実装より原理
- 哲学 → 用語より本質

## 二つの自然変換の関係

```
η_客観 ∘ η_抽象 ≈ η_抽象 ∘ η_客観

「主観→客観」してから「具体→抽象」しても、
「具体→抽象」してから「主観→客観」しても、
たどり着く先は同じ:

        Base Table の構造的本質
```

**可換性の仮説**: この 2 つの自然変換は可換。
