### 1.2 追加構造

| 圏の構造 | Hegemonikón | CCL | 意味 |
|:---------|:-----------|:----|:-----|
| **積** (Product) | 融合 | `*` | 不可分な統合 |
| **極限** (Limit) | Hub WF (内積) | `/o`, `/s`, ..., `lim` | 4定理 → 最適な1点に収束 |
| **余極限** (Colimit) | 反転 (外積) | `\` | 4定理 → テンソルに拡散 |
| **射** (Morphism) | 構造的変換 | `>>` | A → B。A は消え、B になる |
| **自然変換** *(候補)* | 射的融合 | `>*` | A が B を通じて変容。両方が残る |

### 1.3 演算子の三段階スペクトル (Creator 確認済み)

| 演算子 | 名称 | 意味 | A の運命 | B の運命 | 圏論 |
|:-------|:-----|:-----|:---------|:---------|:-----|
| `>>` | 射 | 構造的変換 | **消える** | 出力型 = B | 射 (morphism) |
| `>*` | 射的融合 | 変容 | **残る** (変容) | **残る** (レンズ) | Pullback? Enrichment? |
| `*` | 融合 | 不可分な統合 | **融合** | **融合** | 積 (product) |

> **Creator**: 「>> = 射。>*= 射 + 融合。* = 融合。」
> これは**置換 → 変容 → 統合**のスペクトル。

## 2. 内積と外積 — Hub WF の本質

### 2.1 内積 (Inner Product) = 収束 = Limit

```
/a = ⟨A1, A2, A3, A4⟩
結果: 1次元スカラー（統合スコア）

hub WF の役割:
  入力: Series 内の4定理ベクトル
  処理: 4定理を順次実行し、結果を統合
  出力: 1つの収束した結論（スカラー）
```

**圏論的意味**: Hub WF は **関手** (Functor)。4定理の圏を1つの値の圏に写す構造保存写像。

### 2.2 外積 (Outer Product) = 拡散 = Colimit

```
\a = A1 ⊗ A2 ⊗ A3 ⊗ A4
結果: 4^n 次元テンソル（全組み合わせ）
```

**圏論的意味**: Colimit。すべての定理を含む最も節約的な展開。派生（nous, phro, meta 等）の生成源。

### 2.3 対応表

| 操作 | CCL | 圏論 | 動き |
|:-----|:----|:-----|:-----|
| Hub WF `/o` | `/o` | **Limit (極限)** | 4定理 → 最適な1点に収束 |
| 反転 `\o` | `\o` | **Colimit (余極限)** | 4定理 → テンソル（拡散） |
| 射 | `>>` | Morphism | 定理A → 定理B（変換） |
| 射的融合 | `>*` | Natural transformation | A を B の視点で変容 |

## 4. 射の操作 — CCL の圏論的構文

### 4.1 射 `>>`（構造的変換）

```ccl
/noe >> /met           # O1 → S1 (X-OS morphism)
/noe >> /met >> /kho   # 射の合成: O1 → S1 → P1
```

**非可換性**: `/noe >> /met ≠ /met >> /noe`

- O→S = 本質が様態を規定（下向き）
- S→O = 様態が本質を根拠づけ（上向き）

### 4.2 射的融合 `>*`（自然変換）

```ccl
/noe >* /met           # O1 を S1 の視点で変容
```

`>>` は A が B に **変わる**。`>*` は A が B の視点で **変容する**。

### 4.3 内積 `/`（関手：収束）

```ccl
/a = ⟨A1, A2, A3, A4⟩  → スカラー（統合判定）
```

### 4.4 外積 `\`（余極限：拡散）

```ccl
\a = A1 ⊗ A2 ⊗ A3 ⊗ A4  → テンソル（全派生展開）
```

## 5. 経路同値性の再検討

### 5.1 v1.0 での実験結果

3つの O→A 経路 (α,β,γ) は**異なる結論を生んだ**。
→ Cog は薄い圏ではなく**豊かな圏**。

### 5.2 v2.0 での再解釈

taxis.md の圏論セクションが確認するもの:

- **射は非可換**: `/noe >> /met ≠ /met >> /noe` — 経路は方向依存
- **Bridge vs Anchor**: 同構造間 (Bridge) は同型的、異構造間 (Anchor) は類比的
- **動的優先**: 不確実性 V[] > 0.5 → Bridge 優先 (explore); V[] < 0.3 → Anchor 優先 (exploit)

**結論**: 経路が異なる結論を生むのは**設計通り**。認知は経路依存的であり、FEP の explore/exploit トレードオフと一致する。
