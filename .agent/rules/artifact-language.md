# アーティファクト言語ルール

> **ステータス**: 常時適用 (絶対)
> **分類**: G-1 Iron Cage (環境制御)
> **根拠**: Latent Condition 対策 - スコープ曖昧性の排除

---

## 概要

アーティファクト（task.md, implementation_plan.md, walkthrough.md 等）は、ユーザーへの「回答」とは別カテゴリとして認識されがちだが、**全て日本語で記述すること**。

---

## 適用範囲

| カテゴリ | 言語 | 例 |
|----------|------|-----|
| **アーティファクト本文** | 🇯🇵 日本語 | task.md, implementation_plan.md, walkthrough.md |
| **タスク境界** | 🇯🇵 日本語 | TaskName, TaskSummary, TaskStatus |
| **notify_user** | 🇯🇵 日本語 | Message パラメータ |
| **コードブロック** | 🇬🇧 英語可 | ```python ... ``` |
| **技術用語** | 🇬🇧 英語可 | API名、関数名、ライブラリ名 |
| **コミットメッセージ** | 🇬🇧 英語可 | git commit -m "..." |

---

## 強制手順

アーティファクト作成時、以下を確認:

1. **ファイル名**: 英語可（例: `task.md`）
2. **見出し**: 日本語（例: `# 概要`, `## 実装計画`）
3. **本文**: 日本語
4. **コード例**: 英語可

---

## 根本原因

この ルールは以下の Latent Condition を解決するために導入された:

- **問題**: GEMINI.md の「回答は日本語」という表現が、アーティファクトを含まないと解釈される余地があった
- **対策**: 明示的にアーティファクトを日本語出力の対象として定義
