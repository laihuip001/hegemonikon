# 🔬 Category Lens — 構造を問う5つの問い

> **圏論は「正しさ」ではなく「構造の整合性」を問う。判断の前に、構造を見ろ。**
>
> 詳細: `.agent/skills/category-engine/SKILL.md` (v1.0)
> 知識基盤: Gnōsis に 339論文インデックス済み

---

## 5つの問い

### Q1: この変換は構造を保存しているか？ （関手性チェック）

> 関手 = 「構造を壊さない変換」。壊れていたら、それは関手ではない。

| チェック | 質問 |
|:---------|:-----|
| **順序保存** | A→B の順序が変換後も保たれるか？ |
| **合成保存** | A→B→C = A→C が成り立つか？途中で情報が消えていないか？ |
| **恒等保存** | 何もしない変換が、本当に何も壊していないか？ |

**兆候**: リファクタ後にテストが通るがエッジケースで壊れる → 関手性の破れ

### Q2: この随伴の η/ε は何か？ （往復の質を問う）

> 全ての F⊣G に η (取り込み→忘却→戻る) と ε (忘却→取り込み→戻る) がある。

| 操作 | η の問い | ε の問い |
|:-----|:---------|:---------|
| /eat⊣/fit | 消化して元に戻るか？ | 原文を再現できるか？ |
| /boot⊣/bye | 起動→終了→起動で文脈保存？ | 終了→起動→終了で Handoff 保存？ |
| 要約⊣展開 | 展開して元の意味が残るか？ | 要約から元の文書を復元できるか？ |

**Drift > 0.3 は要注意。何が失われたかを明示的に記録せよ。**

### Q3: この判断の普遍性は何か？ （Limit の問い）

> Limit = 全条件を満たす最も豊かなもの。「これより良い解はあるか？」

- 複数の制約を**全て**満たす、最も情報量の多い選択肢は何か？
- /noe の Kalon がこの Limit に対応する
- 「なぜこの解が最適であり、他の解ではないのか？」

### Q4: 何を忘却して、何を保存したか？ （忘却関手の自覚）

> 抽象化は常に何かを捨てている。捨てたものを自覚しているか？

| 場面 | 問い |
|:-----|:-----|
| 要約を書く | 何を削ったか？ 本質は残ったか？ |
| API を設計する | 実装の何を隠したか？ 隠すべきでないものは？ |
| 判断を下す | どの代替案を排除したか？ 排除は正当か？ |

### Q5: この構造の双対は何か？ （反転の発見）

> 圏論の原理: 全ての概念には双対がある。反転すれば盲点が見える。

| 今やっていること | 双対 | 問い |
|:----------------|:-----|:-----|
| 分析 (分解) | 合成 (統合) | バラバラにした後、組み直せるか？ |
| 探索 | 実行 | いつ探索を止めて実行に移るか？ |
| 具体化 | 抽象化 | 具体の中に潜む共通構造は？ |
| 生成 | 審査 | 作ったものを壊す視点で見たか？ |

## 使い分けガイド

| 場面 | 最重要 Q | 理由 |
|:-----|:---------|:-----|
| リファクタ | **Q1** + Q4 | 構造保存 + 忘却の自覚 |
| WF 実行 | **Q2** | 往復の質を問う |
| 設計判断 | **Q3** | 普遍性（最適解か） |
| 抽象化 | **Q4** | 何を捨てたか |
| 行き詰まり | **Q5** | 双対で盲点発見 |
