# CCL Manifest Requirement (CCL明記規約)

> **CCL式の透明性・再現性・学習性を確保するための必須規約**

---

## 適用範囲

| 場面 | CCL明記 |
|:-----|:-------:|
| ワークフロー実行時 | **必須** |
| アーティファクト生成時 | **必須** |
| コミットメッセージ | 推奨 |
| チャット応答 | CCL式を使用した場合は必須 |

---

## 出力形式

### ワークフロー実行時

```text
┌─[CCL]─────────────────────────────────────────────┐
│ 式: {ccl_expression}                              │
│ 展開: {expanded_form} (複雑な場合のみ)            │
│ 目的: {purpose}                                   │
└───────────────────────────────────────────────────┘
```

### アーティファクト冒頭

```markdown
# [タイトル]

> **CCL**: `{ccl_expression}`
> **目的**: {purpose}
```

### コミットメッセージ

```
[type]: [summary]

CCL: {ccl_expression}

[詳細]
```

---

## 例

### 良い例

```text
┌─[CCL]─────────────────────────────────────────────┐
│ 式: /noe~\noe_/s~\s                               │
│ 展開: Noēsis振動 → Schema振動                      │
│ 目的: 各洗練判断の妥当性を認識と設計両面から検証   │
└───────────────────────────────────────────────────┘
```

### 悪い例

```text
洗練を実行します。
(CCL式が明記されていない)
```

---

## 理由

1. **透明性**: どの認知プロセスが実行されているか可視化
2. **学習**: ユーザーがCCLパターンを学べる
3. **再現性**: 同じ結果を得るために何を実行したかが明確
4. **美学**: CCLは言語である。言語は見せるべきだ。
