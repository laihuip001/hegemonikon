# Gemini 引継書生成プロトコル

> **目的**: Geminiが作業完了時に、Claudeセッションへ引き継ぐための文書を自動生成する。

---

## 発動条件

以下のいずれかの条件を満たした時:

1. `/manual` に基づく更新作業が完了した
2. 複数ファイルを変更してコミットした
3. ユーザーから「引継書を作成」と指示された

---

## 引継書の出力先

```
docs/handoff_<YYYYMMDD>_<タスク名>.md
```

**例**: `docs/handoff_20260123_taxis_update.md`

## 自動コミットについて

### 推奨: 手動コミット

引継書の自動コミットは**推奨しない**。理由:

1. **レビュー機会の確保**: ユーザーが内容を確認してからコミットすべき
2. **コミット粒度の制御**: 引継書を別コミットにするか、作業と一緒にするかはユーザーが決める
3. **エラー時のリカバリ**: 自動コミットはロールバックを複雑にする

### 代替案: コミット提案

引継書生成後、以下を出力:

```
[Hegemonikon] 引継書生成完了
  出力: docs/handoff_<date>_<task>.md
  
推奨コミット:
  git add docs/handoff_*.md
  git commit -m "docs: add handoff for <task>"
```

---

## 出力形式

作業完了時、Geminiは以下を出力:

```
[Hegemonikon] M6 Praxis
  入力: タスク「<タスク名>」
  実行: <N>ファイルの更新 + Git Commit完了
  状態: <完了状態の要約>

[Hegemonikon] 引継書生成
  出力: docs/handoff_<date>_<task>.md
  次: Claudeセッションで /v (検証) を実行
```
