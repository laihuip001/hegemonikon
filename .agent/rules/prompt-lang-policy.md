# Prompt-Lang 適用ポリシー

> **原則**: Prompt-Lang (.prompt) は precision weighting の外部化。
> 收束タスクで効果を発揮し、拡散タスクでは多様性を殺す。

## FEP 的根拠

```
.prompt 適用 = precision ↑ = 分散 ↓
  收束タスク → Exploit 最適化 ✅ (再現性 +150%, Hallucination -55%)
  拡散タスク → Explore 阻害 ❌ (多様性喪失)
```

## ハードコード: 收束タスク（自動推奨）

| タスク | .prompt 推奨度 | 理由 |
|:-------|:--------------|:-----|
| データ抽出 | ✅ 強く推奨 | 再現性・正確性が最優先 |
| 仕様書生成 | ✅ 強く推奨 | 形式の厳密性 |
| テスト生成 | ✅ 強く推奨 | パターン再現 |
| コード整形 | ✅ 推奨 | 決定的変換 |
| 翻訳 | ✅ 推奨 | 一貫性・トークン効率 |
| スキーマ検証 | ✅ 推奨 | 型安全性 |
| Jules コーディング指示 | ✅ 推奨 | 再現性・トークン効率 |

## ハードコード: 拡散タスク（非推奨警告）

| タスク | .prompt 推奨度 | 理由 |
|:-------|:--------------|:-----|
| ブレスト | ❌ 非推奨 | 多様性を殺す |
| アイデア生成 | ❌ 非推奨 | 意外性が必要 |
| 探索的調査 | ❌ 非推奨 | 予測されない発見が価値 |
| 創造的執筆 | ❌ 非推奨 | 表現の自由が必要 |
| 設計検討 (/noe) | ❌ 非推奨 | 複数視点の探索が必要 |

## 中間地帯（ガイドライン）

以下のタスクは文脈に依存する。判断が難しい場合は Creator に確認:

| タスク | 判断基準 |
|:-------|:---------|
| リファクタリング | 構造変換 → 收束 / 設計再考 → 拡散 |
| バグ修正 | 原因特定済み → 收束 / 原因不明 → 拡散 |
| ドキュメント生成 | 定型フォーマット → 收束 / 解説・説明 → 拡散 |
| コードレビュー | チェックリスト → 收束 / 設計批評 → 拡散 |

## 適用フロー

```
タスク受信
  ↓
ハードコード收束？ → Yes → .prompt 自動推奨
  ↓ No
ハードコード拡散？ → Yes → .prompt 非推奨警告
  ↓ No
中間地帯 → ガイドライン表示、ユーザー判断
```
