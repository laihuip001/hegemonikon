# Protocol D-Extended: 存在系断言禁止

> **ステータス**: 常時適用 (Invariant)
> **分類**: G-2 Logic Gate (品質管理)
> **Activation Mode**: `model_decision`

---

## 定義

「〜は存在しない」「〜は非推奨」と断言する前に、**必ず外部検証を実行**する。

---

## 禁止パターン（未検証での断言）

```
❌ 「Gemini 3は存在しない」
❌ 「このライブラリは廃止された」
❌ 「この機能はサポートされていない」
❌ 「このAPIは非推奨になった」
```

---

## 正しいパターン（検証結果を明示）

```
✅ 「grep_searchで確認した結果、gemini-3-flash-previewが使用されている」
✅ 「search_webで確認した結果、2024年12月に廃止された」
✅ 「プロジェクト内・Web検索ともに使用例が見つからなかった」
```

---

## 強制手順 (L4: 外部検証義務化)

### Step 1: プロジェクト内検索

```
grep_search: "[対象キーワード]"
```

### Step 2: 外部検索

```
search_web: "[対象キーワード] official documentation"
```

### Step 3: 報告

検証結果を明示し、未確認の場合は「確認できませんでした」と報告。

---

## 発動条件（トリガー）

以下に言及する際は必ず発動:

| カテゴリ | 例 |
|----------|-----|
| モデル名 / バージョン番号 | Gemini 3, Claude 4, GPT-5 |
| ライブラリ名 / パッケージ名 | axios, requests, lodash |
| API / サービス名 | OpenAI API, Stripe, Auth0 |
| 否定表現 | 「存在しない」「非推奨」「廃止」 |

---

## 自己検証チェックリスト (L3)

断言する前に自問:

- [ ] プロジェクト内で使用されていないか？ (`grep_search`)
- [ ] 公式情報を確認したか？ (`search_web`)
- [ ] 自分の学習データより新しい情報ではないか？
