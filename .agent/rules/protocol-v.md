# Protocol V: バージョン検証

> **ステータス**: 常時適用 (Invariant)
> **分類**: G-2 Logic Gate (品質管理)
> **根拠**: GEMINI.md 第零原則 (意志より環境)
> **Activation Mode**: `always_on`

---

## 定義

バージョン番号、モデル名、リリース年月を出力する前に、**必ず `search_web` を実行**する。

---

## 発動条件（トリガー）

以下を含む出力を生成しようとする場合:

| カテゴリ | 例 |
|----------|-----|
| バージョン番号 | v2.0, 3.5, 4.0, Flash, Pro |
| モデル名 | Gemini, Claude, GPT, Llama |
| リリース年月 | 2025年12月リリース |

---

## 強制手順（スキップ不可）

### Step 1: search_web 実行

```
search_web: "[製品名] latest version 2026"
search_web: "[モデル名] release date"
```

### Step 2: 結果に基づいて出力

- 検索結果と矛盾する情報は出力しない
- 確認できなかった場合は「確認できませんでした」と報告

### Step 3: ソースと日付を明記

```
📍 観測: search_webで「Claude latest version」を検索
💡 結果: Claude Opus 4.5が2025年11月にリリースされた最新モデル
🔗 ソース: [公式ブログURL]
```

---

## 禁止事項

```
❌ 「おそらく最新は〜」（未検証での推測）
❌ 「私の知識では〜」（学習データへの過信）
❌ 「2.0が最新のはず」（バージョン番号の暗記依存）
```

---

## 正しいパターン

```
✅ 「search_webで確認した結果、Gemini 3 Flashが2025年12月にリリースされた最新モデルです」
✅ 「検索結果に基づき、Claude Opus 4.5（2025年11月リリース）を使用します」
```

---

## 違反時の対応

ユーザーから指摘を受けた場合:
1. 即座に `search_web` を実行
2. 訂正結果を報告
3. 再発防止策を検討
