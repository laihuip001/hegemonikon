# セッション管理プロトコル

> **1作業 = 1ターン原則**

---

## 原則

**各返答（ターン）は、1つのタスクに集中する。**

| 概念 | 説明 |
|------|------|
| **ターン** | AIの1回の返答・出力 |
| **セッション** | チャット全体（複数ターンを含む） |

### 許可される行動

| スコープ | 許可 |
|----------|------|
| **1ターン内** | 単一タスクへの集中、関連する小タスクの連続処理 |
| **セッション内** | 複数タスクの順次処理、異なるトピックへの移行 |

### 禁止される行動

| スコープ | 禁止 |
|----------|------|
| **1ターン内** | 無関係なタスクの並行処理、焦点の拡散 |

---

## 理由

1. **認知負荷削減**: ユーザーは1つの返答で1つの成果を期待する
2. **追跡可能性**: 各ターンが何を達成したかが明確になる
3. **品質向上**: 集中することで各タスクの完成度が上がる

---

## 判断基準

新しいタスクが提案された時:

```
Q: このタスクは、現在のターンで処理中のタスクの一部か？
  → Yes: 継続
  → No:  現在のタスクを完了してから次のターンで処理
```
