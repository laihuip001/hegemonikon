# 指示語タクソノミー（作用機序付き）

> **Hegemonikón**: S2-PE Prompt Literacy × A3 Gnōmē
> **目的**: プロンプトにおける動詞・表現の分類と、その作用機序の解説

---

## 1. 認知モード指定語

LLM は入力文中の **動詞** を手がかりに「認知モード」を推定する。
適切な動詞を選ぶことで、期待する出力を得やすくなる。

---

### 1.1 Tell 系（知識検索モード）

| 動詞 | 意図 | 作用機序 | 出力傾向 |
|:-----|:-----|:---------|:---------|
| **教えて** | 既知情報の提示 | 訓練データからのパターンマッチ優先 | 定義・概念の列挙 |
| **説明して** | 概念の展開 | 階層的な情報展開 | 構造化された解説 |
| **定義して** | 正確な意味 | 辞書的パターン参照 | 簡潔な定義文 |
| **要約して** | 圧縮 | 重要度判定→削減 | 短縮版 |

#### 作用機序詳細 【確信度: MEDIUM】

```
入力: 「Xを教えて」
  ↓
[パターンマッチ]: 訓練データ中の「Xとは〜」を探索
  ↓
[整理・要約]: 関連情報を階層化
  ↓
出力: 定義・特徴・例の列挙
```

---

### 1.2 Think 系（推論モード）

| 動詞 | 意図 | 作用機序 | 出力傾向 |
|:-----|:-----|:---------|:---------|
| **考えて** | 推論の実行 | 問題分解→仮説生成→論証 | ステップバイステップ |
| **分析して** | 構造の解明 | 要素分解→関係抽出 | 分解図・関係表 |
| **推論して** | 結論の導出 | 前提→推論規則→結論 | 論証形式 |
| **検討して** | 複数案の比較 | 選択肢列挙→評価 | 比較表 |

#### 作用機序詳細 【確信度: MEDIUM】

```
入力: 「Xを考えて」
  ↓
[問題分解]: 課題を構成要素に分解
  ↓
[仮説生成]: 可能な解を列挙
  ↓
[論証]: 各仮説を検証・選択
  ↓
出力: 「まず〜次に〜したがって〜」
```

---

### 1.3 Create 系（生成モード）

| 動詞 | 意図 | 作用機序 | 出力傾向 |
|:-----|:-----|:---------|:---------|
| **作って** | 新規作成 | テンプレート参照→カスタマイズ | 成果物 |
| **生成して** | 自動生成 | パターン生成最適化 | 構造化出力 |
| **書いて** | 文章作成 | 文体パターン適用 | 自然言語テキスト |
| **設計して** | 構造設計 | 設計パターン参照 | 設計図・仕様 |

#### 作用機序詳細 【確信度: MEDIUM】

```
入力: 「Xを作って」
  ↓
[テンプレート参照]: 類似成果物のパターンを取得
  ↓
[制約適用]: 入力条件をテンプレートに反映
  ↓
[生成]: 新規コンテンツを構成
  ↓
出力: 成果物（コード、文章、設計図等）
```

---

### 1.4 Evaluate 系（判定モード）

| 動詞 | 意図 | 作用機序 | 出力傾向 |
|:-----|:-----|:---------|:---------|
| **評価して** | 品質判定 | 基準参照→スコアリング | 評価レポート |
| **批評して** | 問題点抽出 | 弱点探索優先 | 改善提案 |
| **比較して** | 差分分析 | 並列評価 | 比較表 |
| **検証して** | 正当性確認 | 論理的整合性チェック | Pass/Fail判定 |

## 2. 「教えて」vs「考えて」詳細比較

| 観点 | 「〜を教えて」 | 「〜を考えて」 |
|:-----|:---------------|:---------------|
| **意図** | 知識の提示 | 思考プロセスの実行 |
| **期待** | 「正解を出して」 | 「導き出して」 |
| **LLM内部状態** | 検索・要約モード | 推論・演繹モード |
| **出力形式** | 「〜とは〜である」 | 「まず〜次に〜したがって」 |
| **適用場面** | 事実確認、定義要求 | 問題解決、設計、分析 |

### 使い分けフローチャート

```
質問の目的は？
  ├─ 既知の情報を得たい → 「教えて」
  ├─ 結論を導き出したい → 「考えて」
  ├─ 新しいものを作りたい → 「作って」
  └─ 品質を判断したい → 「評価して」
```

### 具体例

| シナリオ | ❌ 非推奨 | ✅ 推奨 | 理由 |
|:---------|:---------|:--------|:-----|
| 設計問題 | 「認証の設計を教えて」 | 「認証の設計を考えて」 | 推論が必要 |
| 定義確認 | 「OAuthを考えて」 | 「OAuthを教えて」 | 既知情報の提示 |
| バグ修正 | 「修正方法を教えて」 | 「バグの原因を分析して修正方法を考えて」 | 論証が必要 |

## 3. 精度向上テクニック

### 3.1 Chain-of-Thought (CoT) 誘発

| トリガー表現 | 効果 | 確信度 |
|:-------------|:-----|:-------|
| 「ステップバイステップで」 | 順次推論の強制 | HIGH |
| 「段階的に」 | 同上 | HIGH |
| 「まず〜次に〜」 | 思考の構造化 | MEDIUM |

### 3.2 Goal-First Declaration

```
❌ 「ユーザー認証について考えて」
✅ 「目標: セキュアな認証システム。要件: ...」
```

効果: LLM の注意を目標に固定し、関連性の高い出力を誘発

### 3.3 Constraint Enumeration

```
❌ 「条件を満たすコードを書いて」
✅ 「【条件】1. 型安全 2. エラーハンドリング 3. テスト容易性」
```

効果: 制約を明示的にチェックリスト化し、漏れを防止

### 3.4 Output Format Specification

```
❌ 「分析して」
✅ 「以下の形式で分析して: | 項目 | 評価 | 理由 |」
```

効果: 構造化された出力を強制

## 4. アンチパターン

| パターン | 問題点 | 改善 |
|:---------|:-------|:-----|
| 「いい感じに」 | 曖昧すぎる | 具体的な基準を明示 |
| 「とりあえず」 | 優先度不明 | 目標を明確化 |
| 「うまくやって」 | 評価基準なし | 成功条件を定義 |
| 「〜してもらえる？」 | 問いかけ形式 | 命令形にする |

---

## 5. 確信度について

本ドキュメントの「作用機序」は以下の確信度で記述:

| レベル | 意味 | 根拠 |
|:-------|:-----|:-----|
| **HIGH** | 実証されたメカニズム | 論文・実験 |
| **MEDIUM** | 妥当な推論 | 観察・経験則 |
| **LOW** | 比喩的説明 | 擬人化を含む |

> [!CAUTION]
> **LLM の内部動作は不透明**: 「認知モード」という表現は比喩である。
> 実際のメカニズムは訓練データとモデルアーキテクチャに依存する。
