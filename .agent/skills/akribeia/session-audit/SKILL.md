# Session Audit Skill

> **Tekhne-Maker v6 SAGE Mode Generated**
> **Archetype**: Precision (誤答率 < 1%)

## Objective

他セッションの成果物を監査し、以下を検出:

1. **Drift (ドリフト)** — 意図からのずれ
2. **Information Loss** — 情報欠落
3. **Integration Quality** — 馴染み度

## Input

```yaml
audit_target:
  type: session | handoff | artifact
  path: /path/to/target
  original_intent: "元々何を達成しようとしていたか"
```

## Process

### Phase 1: Intent Reconstruction (意図復元)

```
┌─[意図復元]────────────────────────────────────┐
│ 1. Handoff/タスク記述を読み込み               │
│ 2. 「何を達成しようとしていたか」を復元       │
│ 3. 成功基準を明文化                           │
└───────────────────────────────────────────────┘
```

### Phase 2: Reality Check (現実照合)

```
┌─[現実照合]────────────────────────────────────┐
│ 1. 実際の成果物を検証                         │
│ 2. 意図との差分を検出                         │
│ 3. Drift Score (0-100) を算出                 │
└───────────────────────────────────────────────┘
```

### Phase 3: Integration Quality (馴染み度)

```
┌─[馴染み度]────────────────────────────────────┐
│ A + B = A+B (付着) vs A + B → A' (消化)       │
│                                               │
│ 評価軸:                                       │
│ • 用語の統一性                                │
│ • 構造の一貫性                                │
│ • 波及影響の考慮                              │
└───────────────────────────────────────────────┘
```

## Output

```yaml
session_audit_result:
  target: "{path}"
  intent_reconstruction:
    original_goal: "{goal}"
    success_criteria: ["{criteria}"]
  
  drift_analysis:
    drift_score: 15  # 0-100, lower = better
    deviations:
      - type: semantic
        description: "用語変更あり"
        severity: low
  
  integration_quality:
    digestibility_score: 0.85  # 0-1
    issues:
      - "一部セクションが馴染んでいない"
  
  verdict: PASS | WARN | FAIL
  recommendations:
    - "用語を統一する"
```
