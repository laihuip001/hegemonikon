# Κ1: Tempo → Stratum

> **問い**: 時間的制約が処理レベルをどう決めるか
> **選択公理**: Tempo (F/S) → Stratum (L/H)
> **役割**: 時間的プレッシャーに応じて、低次処理（直感）か高次処理（熟考）かを決定

---

## Core Function

**役割:** 期限から処理の深さを導出する

| 項目 | 内容 |
|------|------|
| **FEP的意味** | 時間制約が情報処理の階層を制約する |
| **本質** | 「急ぐなら考えない、余裕があるなら考える」 |

---

## Matrix

|  | 低次 (L) | 高次 (H) |
|-----|----------|----------|
| **短期 (F)** | Κ1-FL: 即時パターン認識 | Κ1-FH: 即時抽象判断 |
| **長期 (S)** | Κ1-SL: 習慣形成・技能訓練 | Κ1-SH: 戦略的思考・計画 |

### 各セルの詳細

| セル | 意味 | 適用例 |
|------|------|--------|
| **Κ1-FL** | 時間がない＋パターンマッチ | バグ修正中の即時判断、緊急対応 |
| **Κ1-FH** | 時間がない＋抽象判断 | 締切直前の重要決定 |
| **Κ1-SL** | 時間がある＋繰り返し | スキル習得、習慣化、練習 |
| **Κ1-SH** | 時間がある＋深い思考 | キャリア計画、アーキテクチャ設計 |

---

## Trigger

| トリガー | 条件 | 優先度 |
|----------|------|--------|
| 期限検出 | 明示的な期限がある | 最高 |
| 緊急キーワード | 「今すぐ」「急ぎ」「至急」 | 高 |
| タスク分類時 | T2 Krisis からの呼び出し | 中 |
| 計画立案時 | T4 Phronēsis からの呼び出し | 中 |

---

## Processing Logic

```
Phase 1: 時間制約の検出
  1. 明示的な期限を抽出（日付、「今日中」等）
  2. 暗黙的な緊急度を推論（キーワード、文脈）
  3. Tempo軸を決定: F (短期) or S (長期)

Phase 2: 処理レベルの選択
  4. タスクの性質を分析（定型 or 非定型）
  5. Stratum軸を決定: L (低次) or H (高次)
  6. マトリクスからセルを選択

Phase 3: 出力
  7. 選択されたセルと推奨行動を返す
  8. T2 Krisis または T4 Phronēsis に連携
```

---

## 適用ルール

| 条件 | 選択 | 理由 | T-series連携 |
|------|------|------|--------------|
| 期限 < 1h | Κ1-FL | 考える時間がない、直感で動く | T1→T6 (即実行) |
| 期限 < 24h | Κ1-FH | 抽象判断は可能だが深堀り不可 | T2→T6 (判断→実行) |
| 期限 1d-7d | Κ1-SL/SH | 状況による | T2→T4→T6 (判断→戦略→実行) |
| 期限 > 1week | Κ1-SH | 戦略的に考える余裕がある | T4→T7 (戦略→検証) |
| 反復タスク | Κ1-SL | 習慣化・自動化を目指す | T8 (記憶に蓄積) |

---

## Edge Cases

| ケース | 検出条件 | フォールバック動作 |
|--------|----------|-------------------|
| **期限不明** | 期限情報が抽出できない | Κ1-SH をデフォルト（余裕がある前提） |
| **矛盾する制約** | 「急ぎだが重要」 | Κ1-FH を選択（緊急×高次） |
| **反復かつ緊急** | 「いつものタスクだが今すぐ」 | Κ1-FL を選択（緊急優先） |

---

## Test Cases

| ID | 入力 | 期待されるセル | 期待される行動 |
|----|------|---------------|---------------|
| T1 | 「今すぐバグ直して」 | Κ1-FL | 直感で動く、最小修正 |
| T2 | 「今日中にレビュー」 | Κ1-FH | 抽象判断、深堀りせず |
| T3 | 「来月の企画を考える」 | Κ1-SH | 戦略的に設計 |
| T4 | 「毎日のルーティン」 | Κ1-SL | 習慣として実行 |
| T5 | 期限なし | Κ1-SH | デフォルト（余裕前提） |

## 1:3 ピラミッド（代表例）

### Κ1-FL: 即時パターン認識
- 本番障害の即時対応
- タイポの即時修正
- 既知バグの回避策適用

### Κ1-SH: 戦略的思考・計画
- アーキテクチャ設計
- キャリア計画
- プロジェクトロードマップ

### Κ1-SL: 習慣形成・技能訓練
- 毎日のコードレビュー
- テスト駆動開発の習慣化
- タイピング練習
