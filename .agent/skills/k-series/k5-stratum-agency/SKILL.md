# Κ5: Stratum → Agency

> **問い**: 処理レベルが主体選択をどう決めるか
> **選択公理**: Stratum (L/H) → Agency (S/E)
> **役割**: 抽象度に応じて、自己改善か環境改善かを決定

---

## Core Function

**役割:** 処理深度から「誰が/何を変えるか」を導出する

| 項目 | 内容 |
|------|------|
| **FEP的意味** | 抽象度が高いほど環境への介入が効果的 |
| **本質** | 「低次は自分を変える、高次は環境を変える」 |

---

## Matrix

|  | 自己 (S) | 環境 (E) |
|-----|----------|----------|
| **低次 (L)** | Κ5-LS: ルーティン実行 | Κ5-LE: 環境調整 |
| **高次 (H)** | Κ5-HS: 自己変革 | Κ5-HE: 組織改革 |

### 各セルの詳細

| セル | 意味 | 適用例 |
|------|------|--------|
| **Κ5-LS** | 低次＋自己 | 日常ルーティン、習慣実行 |
| **Κ5-LE** | 低次＋環境 | デスク整理、ツール設定 |
| **Κ5-HS** | 高次＋自己 | 価値観の見直し、自己変革 |
| **Κ5-HE** | 高次＋環境 | 組織改革、文化変革 |

---

## 適用ルール

| 条件 | 選択 | 理由 |
|------|------|------|
| 毎日のルーティン | Κ5-LS | 自己の習慣として実行 |
| 作業環境改善 | Κ5-LE | 物理/デジタル環境を整える |
| 価値観の転換 | Κ5-HS | 深い自己変革 |
| 組織・チーム改革 | Κ5-HE | 環境を高次で変える |

---

## 1:3 ピラミッド（代表例）

- 朝のルーティン
- コードレビューの習慣

### Κ5-HE: 組織改革
- 開発プロセス変革
- チーム文化の刷新
- 採用方針の見直し
