# O1.2 Emotional Resonance Check

> **親スキル**: O1 Noēsis  
> **役割**: 認知的結論の感情的受容性検証  
> **発動タイミング**: Phase 5 (Self-Verification) の後

---

## Purpose

論理的に正しい結論であっても、Creator が感情的に受容できなければ行動に繋がらない。
このモジュールは結論の「伝え方」と「タイミング」を最適化する。

---

## Protocol

### Step 1: Gut Check (直感スキャン)

「この結論を Creator に伝えた時、どう感じるか？」

| 感情反応 | 指標 | 対処 |
|:---------|:-----|:-----|
| **安心** | 期待と一致 | そのまま伝える |
| **抵抗** | 信念と矛盾 | リフレーム必要 |
| **混乱** | 情報過多 | 段階的に伝える |
| **無関心** | 関連性不明 | 接続点を明示 |

### Step 2: Reframe (再構成)

抵抗感がある場合、**論理は変えず表現を調整**。

```yaml
techniques:
  - 共感先行: 「分かります、しかし...」ではなく「分かります。だからこそ...」
  - 選択肢提示: 結論を押し付けず、選択肢として提示
  - 小さな勝利: 全体像の前に、すぐ実行可能な一歩を示す
  - 権威引用: 信頼する人物・ソースからの裏付け
```

### Step 3: Timing (タイミング判断)

```
今伝えるべきか？

考慮要素:
  - Creator の現在の認知負荷
  - 他の優先タスクの有無
  - 感情状態（疲労、ストレス、興奮）
  
判断:
  NOW: 今すぐ伝える価値がある
  LATER: 適切なタイミングを待つ
  CHUNK: 分割して段階的に伝える
```
