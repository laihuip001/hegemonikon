# /zet 派生モード

> **定理**: O3 Zētēsis (探求)
> **CCL演算子**: `/zet.{derivative}`

---

## anom — 異常探知

> `/zet.anom` — 「何かおかしい」を構造的に探す

### 発動条件

- 違和感がある
- データや状況に異常値がある
- 「なぜこうなっている」の前の「これは変だ」

### 処理

```
入力: 状況 / データ
  ↓
[1] 正常パターンの定義
[2] 逸脱の検出
  ├─ 統計的異常: 期待値からの逸脱
  ├─ 構造的異常: パターンの崩れ
  └─ 直感的異常: 「なんか変」の言語化
[3] 異常の分類
  ├─ 意味のある異常 (signal)
  └─ 無意味な異常 (noise)
  ↓
出力: [異常リスト, signal/noise 分類]
```

### CCL

```ccl
/zet.anom{data: "X-series 接続パターン"} >> /noe{deeper: true}
```

## hypo — 仮説生成

> `/zet.hypo` — 「なぜそうなっているか」の仮説を生成

### 発動条件

- 現象はわかるが原因が不明
- 説明の候補が必要

### 処理

```
入力: 現象 / 観察
  ↓
[1] 仮説候補の列挙
  ├─ 最も単純な仮説 (Occam)
  ├─ 最も面白い仮説 (Creator向け)
  └─ 最も危険な仮説 (リスク考慮)
[2] 仮説の検証可能性
  └─ 何を観測すれば棄却/支持できるか
[3] 優先順位
  ↓
出力: [仮説リスト, 検証方法, 優先順位]
```

### CCL

```ccl
/zet.hypo{phenomenon: "ズーム伝播"} >> /epi{verify: "仮説検証"}
```

## eval — 問いの評価

> `/zet.eval` — 「この問いは追求する価値があるか」

### 発動条件

- 問いは立てたが、追求すべきか迷っている
- リソース配分の意思決定

### 処理

```
入力: 問い
  ↓
[1] 問いの価値評価
  ├─ 答えが出たら何が変わるか
  ├─ 答えが出なくてもわかることは何か
  └─ 問い自体が目的になっていないか (/tel)
[2] コスト見積もり
  └─ この問いの探求に必要なリソース
[3] GO/PASS 判定
  ↓
出力: [GO/PASS, 根拠]
```

### CCL

```ccl
/zet.eval{question: "全72関係に自然度を付けるべきか"} >> /dia{decide: true}
```

## abduction — 推論的飛躍

> `/zet.abduction` — 観察から最良の説明への逆推論

### 発動条件

- 演繹でも帰納でもなく、「最良の説明」が必要
- 驚きのある観察から理論を生成したいとき

### 処理

```
入力: 驚きの観察
  ↓
[1] 「何が最良の説明か」を問う
[2] 説明候補の生成 (Peircean abduction)
[3] 説明力の評価
  ├─ 単純さ
  ├─ 予測力
  └─ 既存知識との整合
  ↓
出力: [最良の説明, 信頼度, テスト方法]
```

### CCL

```ccl
/zet.abduction{surprise: "A-series が終端"} >> /noe{explain: true}
```

### アンチパターン (全派生共通)

- ❌ 問いを立てることが目的になる → `/tel` で目的確認
- ❌ 仮説を検証せずに信念にする → `/epi` で昇格
