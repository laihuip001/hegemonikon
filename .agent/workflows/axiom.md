# /axiom: 定理群統合ワークフロー

> **Hegemonikón Layer**: 全定理層統合
> **目的**: 問題を本質（O）→ 様態（T）→ 文脈（K）→ 関係（X）の順で多層的に分析
> **参照**: [ousia.md](../../kernel/ousia.md), [tropos.md](../../kernel/tropos.md), [kairos.md](../../kernel/kairos.md), [taxis.md](../../kernel/taxis.md)

---

## 本質

4つの定理層を順に適用することで、問題を多角的に分析する。

```
┌─────────────────────────────────────────────────────┐
│ Level 2a: O-series (Ousia) — 本質                    │
│   「何であるか」を問う                               │
├─────────────────────────────────────────────────────┤
│ Level 2b: T-series (Tropos) — 様態                   │
│   「どのように在るか」を問う                         │
├─────────────────────────────────────────────────────┤
│ Level 2c: K-series (Kairos) — 文脈                   │
│   「どの状況で在るか」を問う                         │
├─────────────────────────────────────────────────────┤
│ Level 2': X-series (Taxis) — 関係                    │
│   「どう関係するか」を問う                           │
└─────────────────────────────────────────────────────┘
```

## 発動条件

| トリガー | 説明 |
|----------|------|
| `/axiom` | 定理群統合ワークフローを起動 |
| `/axiom [問い]` | 特定の問いを全層で分析 |
| 本質的な問い | 「なぜ」「何が」を問う根源的疑問 |
| 多角的分析 | 単一視点では不十分な複雑問題 |

---

## Process

### Step 0: 問いの設定

分析対象の問いを明確化する。

```
入力例: 「Prompt-Lang Generator Skill の設計は正しいか？」
```

### Step 1: O-series（本質分析）

**問い**: この問題の本質は何か？

| カテゴリ | 問い | O-series |
|----------|------|----------|
| 認識 | 何が真実か？ | O1 Noēsis |
| 意志 | 何を望むか？ | O2 Boulēsis |
| 探求 | 何を問うか？ | O3 Zētēsis |
| 行為 | 何をするか？ | O4 Energeia |

**出力**:
```
┌─[Hegemonikón]──────────────────────┐
│ O{N}: 本質分析                     │
│ Q: {問いの本質}                    │
│ A: {純粋な答え}                    │
└────────────────────────────────────┘
```

### Step 2: T-series（様態分析）

**問い**: どのようにアプローチするか？

| 機能 | T-series | 問い |
|------|----------|------|
| 知覚 | T1 | 何を観察したか？ |
| 判断 | T2 | 何を選択するか？ |
| 内省 | T3 | 何を理解したか？ |
| 戦略 | T4 | どう計画するか？ |
| 探索 | T5 | 何を調べるか？ |
| 実行 | T6 | 何を行うか？ |
| 検証 | T7 | 何を確認するか？ |
| 記憶 | T8 | 何を保存するか？ |

**出力**:
```
┌─[Hegemonikón]──────────────────────┐
│ T{N}: 様態分析                     │
│ 発動済み: {T-series リスト}        │
│ 推奨: T{X} {機能名}                │
└────────────────────────────────────┘
```

### Step 3: K-series（文脈分析）

**問い**: どの状況でどう判断するか？

選択公理の組み合わせを特定:

| 軸 | 問い | 値 |
|----|------|-----|
| Tempo | 短期か長期か？ | F / S |
| Stratum | 低次か高次か？ | L / H |
| Agency | 自己か環境か？ | S / E |
| Valence | 接近か回避か？ | + / - |

**出力**:
```
┌─[Hegemonikón]──────────────────────┐
│ K{N}: 文脈分析                     │
│ 選択公理: {Axis A} → {Axis B}      │
│ Kairos: K{N}-{XX}                  │
│ 指針: {行動指針}                   │
└────────────────────────────────────┘
```

### Step 4: X-series（関係分析）

**問い**: 定理間の関係は？

| 対象 | X-series | 分析内容 |
|------|----------|----------|
| O間 | X-O | 純粋定理間の従属 |
| T間 | X-T | 拡張定理間の従属 |
| K間 | X-K | 文脈定理間の従属 |

**出力**:
```
┌─[Hegemonikón]──────────────────────┐
│ X-{N}: 関係分析                    │
│ 検出: {因果連鎖/循環/矛盾/正常}    │
│ 図示: {関係図}                     │
└────────────────────────────────────┘
```

### Step 5: 統合結論

4層の分析を統合し、最終結論を導出。

## 使用例

**例1: 設計検証**
```
/axiom Prompt-Lang Generator Skill の設計は正しいか？

→ O3 Zētēsis: 「何を問うか」= 設計の妥当性
→ T7 Dokimē: 検証モードで評価
→ K7-SS: 自己の長期記録として位置づけ
→ X-T: T4→T6→T7 の連鎖を確認
→ 結論: 設計は妥当、T8 で永続化推奨
```

**例2: 意思決定**
```
/axiom MCP統合を今すぐ実装すべきか？

→ O2 Boulēsis: 「何を望むか」= 完成度 vs 早さ
→ T2 Krisis: 判断モードで評価
→ K3-S+: 長期目標への接近
→ X-O: O2→O4 への委譲
→ 結論: 今日は Plan B、MCP は次回
```
