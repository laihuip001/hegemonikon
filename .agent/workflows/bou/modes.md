# /bou 派生モード

> **定理**: O2 Boulēsis (意志・目的)
> **CCL演算子**: `/bou.{derivative}`

---

## desir — 欲求型意志

> `/bou.desir` — 「何を望んでいるか」から意志を構成する

### 発動条件

- 目的が漠然としている
- 「やりたいこと」はあるが「なぜやりたいか」が不明

### 処理

```
入力: 漠然とした望み
  ↓
[1] 欲求の列挙 (Orexis H3 との連携)
[2] 欲求 → 意志への昇格判定
  ├─ 持続的か (一時的欲求 vs 本質的意志)
  ├─ 行動可能か (実現可能性)
  └─ 価値整合か (Hegemonikón の価値観)
[3] 意志宣言
  ↓
出力: 「私はXを意志する。なぜなら...」
```

### CCL

```ccl
/bou.desir{from: "漠然とした望み"} >> /tel{verify: "目的整合"}
```

### アンチパターン

- ❌ 「欲しい」を即座に「意志」にする → `/ore` と `/bou` は別

## voli — 決断型意志

> `/bou.voli` — 「意志として決定する」宣言的モード

### 発動条件

- 検討は十分。あとは決めるだけ
- 「決断する」瞬間

### 処理

```
入力: 検討済みの選択肢
  ↓
[1] 最終確認 (Pistis H2)
[2] 意志の宣言
  └─ 「私はXを選ぶ。Yは選ばない」
[3] 意志のコミット (不可逆宣言)
  ↓
出力: 意志宣言 + コミットメント
```

### CCL

```ccl
/bou.voli{decision: "Phase C1 実行"} >> /ene{commit: true}
```

### アンチパターン

- ❌ 検討不足のまま voli → `/dia` を先に
- ❌ voli の後に迷う → コミットしたら実行

## akra — 意志の弱さ (アクラシア)

> `/bou.akra` — 「わかっているのにできない」状態の診断

### 発動条件

- 目的は明確だが行動できない
- アクラシア (意志の弱さ) の自覚がある

### 処理

```
入力: 意志 vs 現在の行動の乖離
  ↓
[1] 乖離の構造分析
  ├─ 知識不足: 本当にわかっているか
  ├─ 感情抵抗: 何が怖いか (/pat で精緻化)
  └─ 習慣: 別のアトラクターに引かれている
[2] 介入設計
  ├─ 環境変更 (第零原則: 意志より環境)
  ├─ 小さな一歩 (Energeia の micro化)
  └─ コミットメントデバイス
[3] 意志の再確認
  ↓
出力: [乖離の原因, 介入方法, 再コミット]
```

### CCL

```ccl
/bou.akra{gap: "意志 vs 行動"} >> /pro{diagnose: "何が止めている？"}
```

### アンチパターン

- ❌ アクラシアを意志の問題だけにする → 環境が原因の場合が多い
- ❌ 自己批判に陥る → 構造分析に集中
