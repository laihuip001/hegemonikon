## × 反面教師: 「テーブルだけ」

> **出典**: 2026-02-09 / 1742

### 成果1: バグゼロ + ai_fixer AST化

**Angle: 守る** (Valence)

| | Before | After |
|:--|:-------|:------|
| param=param バグ | 11箇所潜伏、POSTボディ空 | 0箇所、根本原因修正済み |
| ai_fixer | regex — 再実行で再発 | AST — 再実行しても安全 |

**比喩**: 火事の原因がライターだとわかった。火を消すだけでなく、ライターを取り上げた。

---

### なぜこれがダメか

1. **テーブルが全て** — Before→After がテーブルの中に閉じている。物語がない
2. **比喩が添え物** — 「ライターを取り上げた」は悪くないが、**本文の中に溶けていない**。テーブルの下に1行追記しただけ
3. **痛みを感じない** — 「11箇所潜伏」と書いてあっても、**何が辛かったかが伝わらない**。誰がどう困ったかの場面がない
4. **心が動かない** — 読んで「ふーん、直したんだね」で終わる。**これが薄いビジネス書**

### 同じ素材で書き直すなら

> 「`json=json` がコメントアウトされていた。たった1行。だが Jules に送るすべての HTTP POST のボディが、空っぽになっていた。つまり私たちは、**命令を叫びながら一度も口を開いていなかった**。
>
> 原因は ai_fixer の正規表現だった。`x = x` を自己代入と見なして消す — 正しい。だが `func(x=x)` のキーワード引数まで巻き込んでいた。正規表現は文脈を知らない。
>
> regex を AST に置き換えた。木構造を見れば、代入文とキーワード引数の区別は自明だ。火を消しただけではない — ライターの芯を抜いた。」
