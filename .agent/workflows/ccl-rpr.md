# @rpr — 回す (React → Plan → Reflect)

> **CCL**: `C:{/ene >> /dia{retrospective}}_/dox+`
> **俗名**: 回す
> **由来**: DX-012 §13 Pearl 因果階梯分析で発見された HGK 核心構造
> **層対応**: τ (React) → Δ (Plan) → L3近似 (Reflect)

## 目的

実行と反省を **収束するまで** 繰り返し、経験を信念として定着させる。
事前に完璧な計画を立てることより、反復的な「やって、見て、学ぶ」を重視する。

## Pearl 因果階梯との対応

| RPR Phase | Pearl Level | HGK 層 | 認知操作 |
|:----------|:-----------|:-------|:---------|
| **React** | L1 Association | τ (tau) | パターンベースの即時実行 |
| **Plan** | L2 Intervention | Δ (delta) | 「もしXを変えたら？」の判定 |
| **Reflect** | L3 Counterfactual (近似) | Ω近似 | 「何が違えば結果は変わったか？」 |

## CCL 式の分解

```
C:{                       # 収束ループ: 品質が安定するまで繰り返す
  /ene                    # React: τ層の実行 (Energeia)
  >>                      # シーケンス
  /dia{retrospective}     # Plan + Reflect: retrospective モードで振り返り判定
}_/dox+                   # 収束後: 信念として深く記録 (Doxa+)
```

## 発動条件

- 「やってみて学ぶ」タイプのタスク
- 失敗の原因分析が必要なとき
- 反復的改善が必要なとき
- 「なぜうまくいかないか」を問うとき

## 実行手順

### Round N (各反復)

1. **React** (`/ene`): 現在の理解に基づいて実行する
2. **Plan/Reflect** (`/dia{retrospective}`):
   - R1: 実行時の状況・前提・制約を復元
   - R2: 「何を変えていたら」を仮説的に検討
   - R3: 代替結果を予測し、学習を抽出
3. **収束判定**: 新しい学びがなくなれば終了

### 終了後

1. **信念化** (`/dox+`): 反復で得られた教訓を永続的な信念として記録

## 出力テーブル

| Round | React (何をした) | Reflect (何を学んだ) | 変更 (次に何を変える) |
|:------|:----------------|:--------------------|:---------------------|
| 1 | {実行内容} | {学習} | {変更点} |
| 2 | {修正実行} | {追加学習} | {追加変更} |
| ... | ... | ... | ... |
| **収束** | — | **統合的教訓** | — |

## 制約

- 最大ラウンド数: 5 (無限ループ防止)
- 各ラウンドで新しい学びがなければ即終了
- `/dia{retrospective}` の R1-R3 は省略不可
