# 取扱説明書 自動更新ワークフロー

新しい機能やツールをForgeに追加した際は、このワークフローに従って `USER_MANUAL.md` を更新してください。

## トリガー条件

以下のいずれかに該当する変更を行った場合:

1. `forge.ps1` に新しいコマンドを追加した
2. 新しいスクリプト (`.ps1`, `.js`) を追加した
3. 新しいプリセットを追加した
4. Web UIに新機能を追加した
5. 既存機能の動作を大幅に変更した

## 更新手順

### ステップ1: 機能の分類を決める

| 分類 | 更新するセクション |
|------|-------------------|
| CLIコマンド | `## コマンドツール (CLI)` |
| Web画面の機能 | `## Web画面` |
| プリセット | `## プリセット` |
| その他のツール | `## その他のツール` |

### ステップ2: テンプレートに従って追記

以下のテンプレートを使用して、該当セクションに追記します:

```markdown
#### 🆕 `コマンド名` - 機能の名前

**できること**: この機能が何をするか、1行で説明。

**使い方**:
```powershell
実際のコマンド例
```

**イメージ**: 専門用語を使わず、日常の例えで説明。
```

### ステップ3: 更新履歴に追記

`## 更新履歴` セクションに、以下の形式で追記します:

```markdown
### YYYY-MM-DD (vX.X)
- ✨ 追加した機能の説明
```

### ステップ4: インデックスを更新

新しいモジュールを追加した場合は、インデックスも更新します:

```powershell
.\forge.ps1 index
```

## 記述ルール

1. **専門用語を避ける**: 「API」「JSON」「パース」などは使わず、平易な言葉で説明する
2. **例え話を使う**: 「レシピ本」「料理」「工具箱」など、日常的なイメージで説明する
3. **具体例を示す**: 必ずコマンド例や使用シーンを添える
4. **絵文字を活用**: 見出しには絵文字をつけて視認性を高める

## AI (Antigravity) への指示

このワークフローが存在する場合、AIは以下を自動的に行います:

1. 新機能を実装した際、`USER_MANUAL.md` の該当セクションに説明を追記
2. 更新履歴に変更内容を記録
3. 必要に応じて `.\forge.ps1 index` を実行
