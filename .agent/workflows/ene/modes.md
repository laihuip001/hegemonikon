# /ene 派生モード

> **定理**: O4 Energeia (行為・実現)
> **CCL演算子**: `/ene.{derivative}`

---

## flow — フロー実行

> `/ene.flow` — 中断なし集中実行モード

### 発動条件

- 計画は確定。あとは没入して実行するだけ
- フロー状態 (Deep Work) の誘導

### 処理

```
入力: 確定済みタスク
  ↓
[1] 前提条件確認 (Pistis: 確信あり?)
[2] タイムボックス設定 (Chronos)
[3] 中断禁止 — 判断は後回し
  ├─ 疑問が出ても記録のみ
  └─ 完了後にまとめて `/dia`
[4] 実行 → 完了
  ↓
出力: [成果物, 途中の疑問リスト]
```

### CCL

```ccl
/ene.flow{task: "Skill 6つ生成", timebox: "30min"}
```

### アンチパターン

- ❌ 不確実なままフローに入る → `/pis` で先に確認
- ❌ フロー中に方向転換する → 完了後に判断

## prax — 実践的実行

> `/ene.prax` — Praxis (S4) と連携した段階的実行

### 発動条件

- 大きなタスクを段階的に実行する
- 各段階の完了チェックが必要

### 処理

```
入力: 段階的計画 (Praxis出力)
  ↓
[1] ステップ N を実行
[2] チェックポイント
  ├─ 完了条件を満たしたか (/sta)
  └─ 次のステップに進めるか (/euk)
[3] ステップ N+1 へ
  ↓
出力: [各ステップの成果, 全体進捗]
```

### CCL

```ccl
/ene.prax{step: 1, plan: "Phase C1"} >> /sta{checkpoint: true}
```

## pois — 制作的実行

> `/ene.pois` — Poiēsis (制作) モード。成果物を作ることに集中

### 発動条件

- 「作る」ことが目的のとき
- ドキュメント、コード、設計書の生成

### 処理

```
入力: 作るもの (仕様)
  ↓
[1] 材料の確認 (何が必要か)
[2] 制作 — 中間出力なし、最終成果物のみ
[3] 品質チェック
  ├─ 仕様を満たしたか
  └─ 美しいか (Hegemonikón美対)
  ↓
出力: [成果物]
```

### CCL

```ccl
/ene.pois{create: "X-series Skill ファイル", spec: "8項目カバー"}
```

---

## constructive — 建設的実行

> `/ene.constructive` — 既存のものを壊さずに改善する

### 発動条件

- リファクタリング、充実化、改善
- 「壊さずに良くする」制約

### 処理

```
入力: 改善対象 + 改善目標
  ↓
[1] 現状の保存 (git commit)
[2] 改善の適用
  └─ 追加のみ (削除は最小)
[3] 改善の検証
  ├─ 元の機能が壊れていないか
  └─ 改善目標を満たしたか
  ↓
出力: [改善済み成果物, diff]
```

### CCL

```ccl
/ene.constructive{target: "S1 Metron SKILL.md", goal: "v3.0 coverage"}
```

### アンチパターン (全派生共通)

- ❌ 計画なしに実行する → `/bou` → `/mek` → `/ene` の順
- ❌ 実行中に設計を変える → 完了後に再設計
