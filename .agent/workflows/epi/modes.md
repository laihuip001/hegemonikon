# /epi 派生モード

> **定理**: A4 Epistēmē (知識)
> **CCL演算子**: `/epi.{derivative}`
> **注**: 12派生が宣言されているが、Tier 1 として最重要の 4つを実装

---

## prac — 実践的知識

> `/epi.prac` — 「やり方」の知識を確立する

### 発動条件

- 実践で得た知見を知識として定着させたい
- How-to を形式化する

### 処理

```
入力: 実践的な経験/発見
  ↓
[1] 再現性チェック
  └─ 同じ手順で同じ結果が出るか
[2] 条件の明示
  └─ どの条件下で有効か
[3] 知識宣言
  └─ 「条件Cの下で、手順Pは結果Rを生む」
  ↓
出力: [知識宣言, 条件, 再現性スコア]
```

### CCL

```ccl
/epi.prac{experience: "batch enrichment は効率的"} >> /gno{generalize: true}
```

---

## theo — 理論的知識

> `/epi.theo` — 「なぜそうなるか」の理論知識を確立

### 発動条件

- 原理を理解し、形式的に記述したい
- 「なぜ」に対する答えを知識にする

### 処理

```
入力: 理論的仮説
  ↓
[1] 論理的整合性チェック
  └─ 他の確立済み知識と矛盾しないか
[2] 予測力チェック
  └─ この理論から新しい予測ができるか
[3] 反証可能性
  └─ 何を観測すれば棄却できるか
  ↓
出力: [理論宣言, 予測, 反証条件]
```

### CCL

```ccl
/epi.theo{theory: "3層自然度はMarkov blanketに対応する"} >> /sop{verify: "文献検証"}
```

---

## tech — 技術的知識

> `/epi.tech` — ツール/技術の操作知識を確立

### 発動条件

- ツールの使い方を知識として固定したい
- 技術的な Tips を体系化

### 処理

```
入力: 技術的経験
  ↓
[1] 操作手順の記録
[2] エッジケースの特定
[3] 知識宣言
  └─ 「ツールTで操作OをするとR、ただしEの場合はF」
  ↓
出力: [手順, エッジケース, 注意事項]
```

### CCL

```ccl
/epi.tech{tool: "LanceDB", operation: "vector search"} >> /dox{persist: true}
```

---

## pattern — パターン知識

> `/epi.pattern` — 繰り返し出現するパターンを知識として抽出

### 発動条件

- 同じ構造が複数の場所に出現する
- 抽象化すべきパターンの発見

### 処理

```
入力: 複数の類似事例
  ↓
[1] 共通構造の抽出
  └─ 何が同じで何が違うか
[2] パターンの命名
  └─ 適切な名前 = パターンの価値の半分
[3] 適用範囲の定義
  └─ このパターンが有効な条件
  ↓
出力: [パターン名, 構造, 適用範囲]
```

### CCL

```ccl
/epi.pattern{instances: ["S→P→K zoom", "O→H→A drive"]} >> /gno{name_it: true}
```

### アンチパターン (全派生共通)

- ❌ 未検証のまま知識宣言 → 検証 (verify) が必須
- ❌ 全てを知識にする → Doxa のまま保持する価値もある
- ❌ Sophia (知恵) と混同 → 知恵は文脈依存、知識は文脈独立
