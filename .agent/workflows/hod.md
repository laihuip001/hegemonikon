# /hod: 経路定義ワークフロー (Hodos) v2.1

> **正本参照**: [P2 Hodos SKILL.md](file:///home/makaron8426/oikos/.agent/skills/perigraphe/p2-hodos/SKILL.md)
> **目的**: 「どう進めるか」を決定する — 条件空間における経路
> **役割**: 方法論・道筋の選択

---

## Cognitive Algebra

| Operator | Meaning | Output |
|:---------|:--------|:-------|
| `/hod+` | **Deepening** | 詳細経路: 代替経路とトレードオフ |
| `/hod-` | **Reduction** | 即経路: Explore/Exploit の一言 |
| `/hod*` | **Expansion** | メタ経路: 「なぜこの道か」を問う |

---

## 本質

**P2 Hodos** = ὁδός (道・方法) = **経路の定義**

> 「どの道を選ぶか？どう進むか？」

---

## 発動条件

| トリガー | 説明 |
|:---------|:-----|
| `/hod` | 経路定義を開始 |
| `/hod [目標]` | 特定目標への経路を定義 |
| 「どう進める」 | 暗黙トリガー |

---

## Process

```text
入力: 条件空間 + 目標
  ↓
[STEP 1] 経路探索
  ├─ Explore経路: 新しい道を探る
  └─ Exploit経路: 既知の道を辿る
  ↓
[STEP 2] 経路評価
  - 効率性
  - リスク
  - 学習価値
  ↓
出力: 選択された経路
```

## --mode=line (直線経路)

> **CCL**: `/hod.line` = `/hod+{shape=linear}`
> **目的**: シンプルな直線的経路を設計する
> **Scale**: 🔬 Micro

**発動**: `/hod line` または「順番に」「ステップバイステップ」

**プロセス**:

```text
1. 開始点と終了点を定義
   ↓
2. 中間ステップを列挙
   ↓
3. 順序を決定
   ↓
4. 依存関係を確認
   ↓
5. 出力: 直線経路 + 順序
```

**出力形式**:

```text
┌─[P2 Hodos: Line]──────────────────────────────────────┐
│ 開始: {現在地}                                        │
│ 終了: {目標}                                          │
│                                                       │
│ 経路:                                                 │
│   [開始] → Step 1 → Step 2 → Step 3 → [終了]          │
│                                                       │
│ 依存: Step N は Step N-1 完了後                       │
└───────────────────────────────────────────────────────┘
```

## --mode=bran (分岐経路)

> **CCL**: `/hod.bran` = `/hod+{shape=branch}`
> **目的**: 条件分岐を含む経路を設計する
> **Scale**: 🔭 Meso

**発動**: `/hod bran` または「場合分け」「条件によって」

**プロセス**:

```text
1. 分岐点を特定
   ↓
2. 各分岐の条件を定義
   ↓
3. 各経路の結末を確認
   ↓
4. デフォルト経路を設定
   ↓
5. 出力: 分岐図 + 条件
```

**出力形式**:

```text
┌─[P2 Hodos: Bran]──────────────────────────────────────┐
│ 分岐点: {決定ポイント}                                │
│                                                       │
│ 条件A: {条件} → 経路A → {結末A}                       │
│ 条件B: {条件} → 経路B → {結末B}                       │
│ デフォルト: → 経路C → {結末C}                         │
│                                                       │
│ 推奨: 条件{X}の可能性が高い → 経路{X}を準備           │
└───────────────────────────────────────────────────────┘
```

## --mode=cycl (循環経路)

> **CCL**: `/hod.cycl` = `/hod+{shape=cycle}`
> **目的**: 繰り返し/反復の経路を設計する
> **Scale**: 🔭 Meso

**発動**: `/hod cycl` または「繰り返し」「イテレーション」「サイクル」

**プロセス**:

```text
1. サイクルの構成要素を特定
   ↓
2. 開始点とループバック点を定義
   ↓
3. 終了条件を定義
   ↓
4. 各イテレーションの改善点を設定
   ↓
5. 出力: サイクル図 + 終了条件
```

**出力形式**:

```text
┌─[P2 Hodos: Cycl]──────────────────────────────────────┐
│ サイクル:                                             │
│   ┌→ Plan → Do → Check → Act ─┐                       │
│   └───────────────────────────┘                       │
│                                                       │
│ 終了条件: {何を達成したら終わるか}                    │
│ イテレーション目安: {N}回                             │
│ 各回の改善: {何を改善するか}                          │
└───────────────────────────────────────────────────────┘
```

## --mode=bisect (二分探索)

> **CCL**: `/hod.bisect` = `/hod+{strategy=binary}`
> **目的**: 問題空間を二分して効率的に探索する
> **Scale**: 🔬 Micro

**発動**: `/hod bisect` または「半分に」「二分」「バイセクト」

**プロセス**:

```text
1. 探索空間を定義
   ↓
2. 中間点を選択
   ↓
3. 中間点で判定
   ↓
4. 半分を捨てて繰り返し
   ↓
5. 出力: 二分探索手順 + O(log n)
```

**出力形式**:

```text
┌─[P2 Hodos: Bisect]────────────────────────────────────┐
│ 探索空間: [{min}..{max}]                              │
│                                                       │
│ 中間点: {mid}                                         │
│ 判定: {mid で何を確認するか}                          │
│                                                       │
│   結果 < 基準 → 右半分を探索                          │
│   結果 > 基準 → 左半分を探索                          │
│                                                       │
│ 計算量: O(log n) = {ステップ数}                       │
└───────────────────────────────────────────────────────┘
```

## --mode=backcast (逆予測)

> **CCL**: `/hod.backcast` = `/hod+{direction=future→present}`
> **目的**: 望む未来から現在への経路を逆算する
> **Scale**: 🌍 Macro

**発動**: `/hod backcast` または「未来から逆算」「バックキャスト」

**プロセス**:

```text
1. 望む未来状態を詳細に描写
   ↓
2. その状態に至る直前を特定
   ↓
3. 再帰的に「直前」を辿る
   ↓
4. 現在まで到達
   ↓
5. 出力: タイムライン + 最初の一歩
```

**出力形式**:

```text
┌─[P2 Hodos: Backcast]──────────────────────────────────┐
│ 望む未来 (Y年後): {詳細な未来像}                      │
│                                                       │
│ タイムライン:                                         │
│   [Y年後] ← {Y-1年後の状態}                           │
│          ← {Y-2年後の状態}                            │
│          ← ... ← [今]                                 │
│                                                       │
│ 最初の一歩: {今日やるべきこと}                        │
└───────────────────────────────────────────────────────┘
```

## --mode=search (探索戦略)

> **CCL**: `/hod.search` = `/hod+{strategy=bfs|dfs|heuristic}`
> **目的**: 解空間における最適な探索戦略を選択する
> **Scale**: 🔭 Meso

**発動**: `/hod search` または「探索」「どう探す」

**プロセス**:

```text
1. 解空間の構造を把握
   ↓
2. 探索戦略を選択
   ├─ BFS: 幅優先（浅く広く）
   ├─ DFS: 深さ優先（深く狭く）
   └─ Heuristic: ヒューリスティック誘導
   ↓
3. 探索開始点を決定
   ↓
4. 終了条件を定義
   ↓
5. 出力: 探索戦略 + 開始点 + 終了条件
```

**出力形式**:

```text
┌─[P2 Hodos: Search]───────────────────────────────────┐
│ 解空間: {探索対象の構造}                             │
│                                                      │
│ 戦略: {BFS/DFS/Heuristic}                            │
│ 理由: {なぜこの戦略か}                               │
│                                                      │
│ 開始点: {どこから始めるか}                           │
│ 終了条件: {何を見つけたら止まるか}                   │
│                                                      │
│ 予想計算量: O({n})                                   │
└──────────────────────────────────────────────────────┘
```

## --mode=backward (逆算)

> **CCL**: `/hod.backward` = `/hod{direction=reverse}_/tel`
> **目的**: ゴールから逆算して経路を構築する
> **Scale**: 🔭 Meso

**発動**: `/hod backward` または「逆算」「ゴールから」

**プロセス**:

```text
1. ゴール状態を明確に定義
   ↓
2. ゴールに到達する直前の状態を特定
   ↓
3. 再帰的に「直前の状態」を辿る
   ↓
4. 現在地に到達するまで続ける
   ↓
5. 出力: 逆算経路 + マイルストーン
```

**出力形式**:

```text
┌─[P2 Hodos: Backward]─────────────────────────────────┐
│ ゴール: {最終状態}                                   │
│ 現在地: {現在状態}                                   │
│                                                      │
│ 逆算経路:                                            │
│   [ゴール] ← {直前ステップ}                          │
│           ← {その前}                                 │
│           ← ... ← [現在地]                           │
│                                                      │
│ マイルストーン:                                      │
│   1. {中間目標1}                                     │
│   2. {中間目標2}                                     │
│   3. {中間目標3}                                     │
└──────────────────────────────────────────────────────┘
```
