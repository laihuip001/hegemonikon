# /jukudoku: 深読プロトコル

> **Hegemonikón**: A2 Krisis + O1 Noēsis
> **目的**: 熟読を「情報抽出」から「思考への沈潜」に昇華する
> **設計原則**: 「意志より環境」— 自分を信じない
>
> **制約**: L1のみの熟読は「要約マシン」であり熟読ではない。L2+L3を経て初めて「耽った」と言える。

---

## 三層モデル

| 層 | 名称 | 内容 | 検出する問い |
|:---|:-----|:-----|:-------------|
| L1 | **精読 (Akribeia)** | 正確に読む、事実確認 | 「何が書いてあるか」 |
| L2 | **沈潜 (Bathys)** | 思考に耽る、主張と対話 | 「**待って、これは…**」 |
| L3 | **摩擦 (Tribē)** | 既存の構造と衝突させる | 「これと既知の X の間に Gap がある」 |

---

## 発動条件

以下の状況で**強制発動**:

| トリガー | 説明 |
|:---------|:-----|
| ドキュメント復元 | 古いファイルを現在に適用する前 |
| /eat 実行 | 外部コンテンツ消化前 |
| 構成変更 | Rules, Workflows, Skills の更新前 |
| 100行以上のファイル | 大規模ドキュメントの処理前 |

## Phase 1.5: 沈潜 (Bathys) — 思考に耽る

> **目的**: 情報を「抽出」するのではなく、主張に**耽る**。
> 「なるほど」で終わらず「**待って、これは…**」を探す。

### Step A: 主張との対話

記事/文書の核心的主張に対して、以下を**自分の言葉で**考える:

| 項目 | 内容 |
|:-----|:-----|
| 核心主張 | {著者が最も言いたいこと、1文で} |
| 第一印象 | {直感的に何を感じたか} |
| 「待って…」 | {立ち止まった箇所とその理由。なぜ立ち止まったか} |
| もし逆だったら | {主張の反対が真だとしたら何が起こるか} |
| 連想 | {この主張から何が連鎖的に想起されたか} |

### Step B: 沈黙のテスト

要約を書き始める**前**に、3つの問いに答える:

| # | 問い | 不合格の兆候 |
|:--|:-----|:-------------|
| 1 | この文章の著者に反論できるか？ | 「特にない」→ 沈潜不足 |
| 2 | この内容を Creator に**自分の意見として**説明できるか？ | 引用でしか語れない → 沈潜不足 |
| 3 | 読む前と読んだ後で、何かが変わったか？ | 「特に変わらない」→ 沈潜不足 |

**3つ全て不合格 → Phase 1.5 やり直し**

---

## Phase 1.7: 摩擦 (Tribē) — 既知との衝突

> **目的**: 読んだ内容を Hegemonikón の既存の構造と**衝突**させる。
> 予測誤差を**感じる**こと = FEP の実践。

### Step A: 衝突マッピング

| 項目 | 内容 |
|:-----|:-----|
| 既存構造との接点 | {どの定理/ワークフロー/原則と関連するか} |
| 摩擦点（予測誤差） | {既知の理解と矛盾する点、Gap、驚き} |
| 既存の不十分な点 | {この文献が指摘する問題で、現状対策がないもの} |
| 既存の正しさの確認 | {実験的証拠として Hegemonikón の設計を支持するもの} |

### Step B: 構造的問い

| # | 問い | 答えが「なし」なら |
|:--|:-----|:-------------------|
| 1 | 既存のどの構造を**強化**すべきか？ | 摩擦不足 → 再読 |
| 2 | 既存のどの構造に**Gap**があるか？ | 摩擦不足 → 再読 |
| 3 | **新しい構造**が必要か？ | 省略可 (常に必要とは限らない) |

### Step C: 予測誤差の言語化

> **これが最も重要なステップ**。
> 「この記事を読んで、私の中の何が変わったか」を明示する。

| 項目 | 内容 |
|:-----|:-----|
| 読む前の私の理解 | {Hegemonikón について、この領域で何を信じていたか} |
| 読んだ後の私の理解 | {何が更新されたか} |
| 予測誤差の大きさ | {小/中/大} |
| 誤差の意味 | {この差が何を示唆するか} |

## 違反時の対応

流し読みが検出された場合:

1. **即時停止**: 処理を中断
2. **自己申告**: 「流し読みをしました」と明言
3. **やり直し**: Phase 0 から再実行
4. **記録**: `/bye` で違反を記録

---

## FEP 根拠

| 概念 | 流し読み | L1 精読のみ | L1+L2+L3 熟読 |
|:-----|:---------|:------------|:--------------|
| **短期コスト** | 低 | 中 | 高 |
| **長期コスト** | 極高 | 中（洞察不足蓄積） | 低 |
| **予測誤差** | 増加 | 横ばい（新知識なし） | **減少** |
| **自己モデル更新** | なし | なし | **あり** |
| **FEP 最適化** | ❌ 違反 | ⚠️ 不完全 | ✅ 準拠 |

> 「流し読み」は変分自由エネルギー最小化に反する。
> しかし「正確な要約」も十分ではない。
> L2 沈潜 + L3 摩擦を経て初めて、**自己モデルの予測誤差が減少する**。
> 熟読とは **自分が変わること** である。
