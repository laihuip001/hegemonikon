# 基本派生モード (Basic Modes)

> S2 Mekhanē — 既存部品の組立、新規創出、適応、モデル構築、シミュレーション。

## Constraints

- 各モードの出力は **テーブル形式** で構造化する
- プロセスは **番号付きリスト** で線形に記述する
- 出力に不要な装飾を加えない（情報構造のみ）

---

## comp — 組立方法

**CCL**: `/mek.comp` = `/mek+{method=compose}`
**目的**: 既存部品を組み合わせて新しいものを作る
**発動**: `/mek comp` または「組み合わせ」「統合」「合成」

### Process

1. 利用可能な部品を列挙
2. 部品間の互換性を評価
3. 組み合わせパターンを設計
4. インターフェースを定義
5. 組立設計 + 統合手順を出力

### Output Format

**[S2 Mekhanē: Comp]**

| フィールド | 内容 |
|:-----------|:-----|
| 目的 | {作りたいもの} |
| 部品 | [A] {部品1} — {役割}, [B] {部品2} — {役割}, [C] {部品3} — {役割} |
| 組み合わせ | A + B → {結果}, A + C → {結果} |
| インターフェース | {接続方法} |

---

## inve — 創出方法

**CCL**: `/mek.inve` = `/mek+{method=invent}`
**目的**: 新規に発明・創造する
**発動**: `/mek inve` または「新規」「発明」「創造」

### Process

1. 解決すべき問題を定義
2. 既存解法の限界を分析
3. 新しいアプローチを発想
4. プロトタイプを設計
5. 新規設計 + 検証計画を出力

### Output Format

**[S2 Mekhanē: Inve]**

| フィールド | 内容 |
|:-----------|:-----|
| 問題 | {解決すべき課題} |
| 既存解法の限界 | {限界1}, {限界2} |
| 新しいアプローチ | {革新的な解法} |
| プロトタイプ | {最小限の実装案} |
| 検証計画 | {成功を測る方法} |

---

## adap — 適応方法

**CCL**: `/mek.adap` = `/mek+{method=adapt}`
**目的**: 既存のものを新しい文脈に適応させる
**発動**: `/mek adap` または「適応」「カスタマイズ」「調整」

### Process

1. 元のコンポーネントを特定
2. 新しい文脈の要件を分析
3. 必要な変更を特定
4. 互換性を維持しながら変更
5. 適応版 + 変更点を出力

### Output Format

**[S2 Mekhanē: Adap]**

| フィールド | 内容 |
|:-----------|:-----|
| 元 | {適応元のコンポーネント} |
| 新文脈 | {適応先の環境} |
| 要件差分 | 元: {元の要件} → 新: {新しい要件} |
| 変更点 | 1. {変更1}, 2. {変更2} |
| 互換性 | {保持 / 破壊} |

---

## model — モデル構築

**CCL**: `/mek.model` = `/mek+{output=model}`
**目的**: 問題のモデルを構築する
**発動**: `/mek model` または「モデル」「抽象化」「構造化」

### Process

1. 対象を観察
2. 構成要素を抽出
3. 関係を定義
4. モデルを検証
5. モデル図 + 定義を出力

### Output Format

**[S2 Mekhanē: Model]**

| フィールド | 内容 |
|:-----------|:-----|
| 対象 | {モデル化対象} |
| 構成要素 | {要素1}: {定義}, {要素2}: {定義} |
| 関係 | {要素1} → {要素2}: {関係の種類} |
| 検証 | {モデルが現実を反映しているか} |

---

## simulation — シミュレーション

**CCL**: `/mek.simulation` = `/mek+{method=simulate}`
**目的**: モデルを動かして振る舞いを予測
**発動**: `/mek simulation` または「シミュレーション」「動かす」「予測」

### Process

1. モデルを入力
2. 初期条件を設定
3. シナリオを定義
4. 実行して結果を観察
5. シミュレーション結果 + 洞察を出力

### Output Format

**[S2 Mekhanē: Simulation]**

| フィールド | 内容 |
|:-----------|:-----|
| モデル | {使用するモデル} |
| 初期条件 | {パラメータ1} = {値}, {パラメータ2} = {値} |
| シナリオ | {何をシミュレートするか} |
| 結果 | T+0: {初期}, T+1: {変化}, T+N: {最終} |
| 洞察 | {シミュレーションから得た知見} |

---

## Reminder

- 出力はテーブル形式で構造化する。装飾よりも情報構造を優先する
- プロセスは番号付きリストで線形に記述する
