# 派生モード

> Noēsis の6派生 + 推論チェーン3変種。問題特性に応じて自動提案される。
>
> **制約**: 出力形式テーブルに従うこと。普遍洞察(nous)は適用範囲を明示すること。

---

## nous (普遍的洞察)

> `/noe.nous` — 普遍的・原理的な洞察を得る (Macro)

発動: `/noe nous` または「普遍的」「原理」「本質」

| 項目 | 内容 |
|:-----|:-----|
| 問い | {分析対象} |
| 普遍的洞察 | 抽出された原理 |
| 根拠 | なぜ普遍的と言えるか |
| 適用範囲 | この原理が成り立つ条件 |

## scaffold (骨格駆動)

> `/noe.scaffold` — Exploit 優先: 骨格を先に固め、創発を後から注入する (PGH型)

発動: `/noe scaffold` または「骨格から」「安定重視で深掘り」

| 項目 | 内容 |
|:-----|:-----|
| Phase構造 | 低エントロピー骨子 → 高エントロピー創発 → 低エントロピー整流 |
| 適用場面 | 既知の構造を洗練する場合 (前提が正しいとき) |
| 対比 | 標準 /noe = Explore 優先 (前提破壊から)、scaffold = Exploit 優先 (前提保存から) |
| 出典 | Prompt R&D Lab #37 確率勾配ハンドオフ (PGH) |

> FEP 的に: scaffold = 精度重み付け (Precision weighting) が高い状態での推論。
> 標準 /noe = 精度重み付けが低い状態での自由探索。

---

## recursive (再帰的議論)

> `/noe --mode=recursive` — エージェント間の再帰的議論で収束 (旧 /syn 系)

発動: `/noe recursive` または「再帰的に議論」「何度もやりとり」

| 項目 | 内容 |
|:-----|:-----|
| 方法 | 複数視点を定義 → 各視点で評価 → 前の視点の出力を次に渡す → 収束まで繰り返す |
| 対比 | council = 並列評価→統合 (余積)、recursive = 逐次議論→収束 (随伴) |
| 適用場面 | 視点間の相互作用が重要な場合 |
| 出典 | Prompt R&D Lab #15 協調型エージェント |

*/noe サブモジュール — v2.1 scaffold + recursive 追加 (2026-02-10)*
