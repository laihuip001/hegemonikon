# 派生モード

> Noēsis の6派生 + 推論チェーン3変種。問題特性に応じて自動提案される。
>
> **制約**: 出力形式テーブルに従うこと。普遍洞察(nous)は適用範囲を明示すること。

---

## nous (普遍的洞察)

> `/noe.nous` — 普遍的・原理的な洞察を得る (Macro)

発動: `/noe nous` または「普遍的」「原理」「本質」

| 項目 | 内容 |
|:-----|:-----|
| 問い | {分析対象} |
| 普遍的洞察 | 抽出された原理 |
| 根拠 | なぜ普遍的と言えるか |
| 適用範囲 | この原理が成り立つ条件 |

---

## phro (実践的判断)

> `/noe.phro` — 文脈に応じた実践的判断を導出する (Meso)

発動: `/noe phro` または「実践的」「この場合は」「具体的に」

| 項目 | 内容 |
|:-----|:-----|
| 文脈 | {具体的状況} |
| 判断 | この文脈での最善の行動 |
| 理由 | なぜ適切か |
| 代替案 | 他に考えられる選択肢 |

---

## meta (メタ認知)

> `/noe.meta` = `/noe*` — 認知プロセス自体を分析・評価する

発動: `/noe meta` または「なぜそう考えた」「思考の構造」

| 項目 | 内容 |
|:-----|:-----|
| 分析対象 | どの思考プロセスを分析するか |
| 前提 | 暗黙の前提リスト |
| バイアス | 検出されたバイアス |
| 信頼性 | HIGH/MED/LOW |
| 改善点 | より良い思考のために |

---

## separate (問題分離)

> `/noe.separate` — 複雑な問題を独立した部分問題に分離

発動: `/noe separate` または「分けて考える」「切り分ける」

| 項目 | 内容 |
|:-----|:-----|
| 問題 | 元の複雑な問題 |
| 部分問題 | P1, P2, P3... + 独立度(HIGH/MEDIUM/LOW) |
| 依存関係 | P1 → P2 → P3 |
| 推奨順序 | 解決すべき順番 |

---

## align (調整・整合)

> `/noe.align` — 複数の要素・視点の整合性を取る

発動: `/noe align` または「整合させる」「揃える」

| 項目 | 内容 |
|:-----|:-----|
| 対象 | 整合させるべき要素 |
| 不整合 | 矛盾する要素のペア |
| 整合基準 | 何を基準に揃えるか |
| 調整アクション | 具体的な調整ステップ |
| 整合後 | 期待される状態 |

---

## metalearning (メタ学習)

> `/noe.metalearning` — 学習プロセス自体を学習・改善する (Macro)

発動: `/noe metalearning` または「学び方を学ぶ」

| 項目 | 内容 |
|:-----|:-----|
| 学習対象 | 何を学んできたか |
| 効果的パターン | 良かった学習方法 |
| 非効率パターン | 改善すべき学習方法 |
| 学習戦略 | 抽出された学習原則 |
