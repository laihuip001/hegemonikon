# /ore 派生モード

> **定理**: H3 Orexis (欲求)
> **CCL演算子**: `/ore.{derivative}`
> **注**: 11派生が宣言されているが、Tier 1 として最重要の 5つを実装

---

## targ — ターゲット欲求

> `/ore.targ` — 特定の対象に向けた欲求の同定

### 発動条件

- 「Xが欲しい」の X を明確にしたい
- 漠然とした欲求を具体化する

### 処理

```
入力: 漠然とした欲求
  ↓
[1] 対象の具体化
  ├─ 何が欲しいか (具体名)
  ├─ いつまでに欲しいか (時間)
  └─ どの程度欲しいか (強度)
[2] 対象の実在性チェック
  └─ 存在する対象か、理想化された幻か
  ↓
出力: [対象, 強度, 実在性]
```

### CCL

```ccl
/ore.targ{vague: "もっと良くしたい"} >> /bou{concrete: true}
```

---

## acti — 行動欲求

> `/ore.acti` — 「やりたい/やりたくない」の行動的欲求

### 発動条件

- 行動への欲求/抵抗を感じている
- 「やりたい」とは何をやりたいのかを分解する

### 処理

```
入力: 行動的衝動
  ↓
[1] 行動の分解
  ├─ 表層: 直接やりたいこと
  └─ 深層: その行動で満たされる本当の欲求
[2] 行動可能性
  └─ 今すぐできるか、条件が必要か
  ↓
出力: [行動, 深層欲求, 実行可能性]
```

### CCL

```ccl
/ore.acti{impulse: "コードを書きたい"} >> /tel{check: "目的に合っているか"}
```

## stat — 状態欲求

> `/ore.stat` — 「こうありたい」の状態的欲求

### 発動条件

- 行動ではなく状態への欲求
- 「もっと○○でありたい」

### 処理

```
入力: 状態の理想
  ↓
[1] 現状とのギャップ分析
[2] 状態は行動で達成できるか
  ├─ 達成可能 → /ene で実現
  └─ 環境依存 → 環境変更の検討
  ↓
出力: [理想状態, ギャップ, 達成方法]
```

### CCL

```ccl
/ore.stat{ideal: "信頼される存在"} >> /bou.voli{commit_to: "行動で示す"}
```

## jtbd — Jobs-to-be-Done

> `/ore.jtbd` — 欲求を「片づけたい仕事」として再定義

### 発動条件

- 欲求の本質を機能的に理解したい
- 「なぜそれが欲しいか」を仕事の言語で

### 処理

```
入力: 欲求
  ↓
[1] JTBD フレーム変換
  └─ 「状況Sにおいて、Xのために、Yしたい」
[2] 代替手段の列挙
  └─ 同じ Job を果たす別の方法は
  ↓
出力: [JTBD 定義, 代替手段リスト]
```

### CCL

```ccl
/ore.jtbd{want: "もっとSkillを充実させたい"} >> /zet.eval{value: "この仕事の価値は"}
```

## bias — 欲求バイアス検出 ⚠️

> `/ore.bias` — 欲求が認知を歪めていないか診断する
> **構造的に最重要**: H-series (動機層) はバイアスの発生源。X-HK3/4 の射を持つ Orexis は特に危険。

### 発動条件

- 欲求が強い状態で判断しようとしている
- 「欲しいから正しい」と思い始めている
- X-HK3 (欲求→好機認識) または X-HK4 (欲求→目的書換) が活性化している

### 処理

```
入力: 現在の欲求 + 判断対象
  ↓
[1] バイアスの種類特定
  ├─ X-HK3 好機バイアス: 「欲しいから今がチャンスに見える」
  ├─ X-HK4 目的バイアス: 「欲しいから目的を書き換えた」
  └─ 確証バイアス: 「欲しいから都合の良い情報だけ集めた」
  ↓
[2] 反実仮想テスト
  └─ 「もし欲求がなかったら、同じ判断をするか？」
  ↓
[3] 浄化判定
  ├─ CLEAN: バイアスなし、判断は妥当
  ├─ BIASED: バイアスあり → /dia.epo で判断停止
  └─ UNCERTAIN: 判断保留 → 追加検証
  ↓
出力: [CLEAN/BIASED/UNCERTAIN, バイアス種別, 浄化提案]
```

### CCL

```ccl
# 欲求が好機認識を歪めていないか
/ore.bias{desire: "新しいフレームワークを使いたい", judgment: "今がチャンス"}

# 全通過フィルター: 欲求→バイアスチェック→判定
/ore >> /ore.bias >> /dia{filtered: true}

# Epochē と連携
/ore.bias{result: "BIASED"} >> /dia.epo{suspend: true}
```

### アンチパターン

- ❌ バイアス検出を面倒がってスキップ → **H-series で最も危険な省略**
- ❌ 全てを BIASED にする → バイアスがないケースも認める
- ❌ バイアス=悪と決めつける → バイアスは信号。浄化の対象であり除去の対象ではない

---

### アンチパターン (全派生共通)

- ❌ 欲求=目的にする → Orexis ≠ Telos
- ❌ 欲求→好機宣言 → X-HK3 バイアス。必ず `/dia.epo`
- ❌ 全ての欲求を叶えようとする → 優先順位が必要
