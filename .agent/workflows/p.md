# /pure: 純粋定理ワークフロー

> **Hegemonikón Layer**: Level 2a 純粋定理
> **目的**: メタ認知的な問い（何を知っているか、何を望むか、何を問うか、何をするか）に対し、具体的制約（時間・階層）を排除した純粋な答えを導く。

---

## 発動条件

- `/pure` コマンド実行時
- ユーザーから「我々」「私たち」を主語とした本質的な問いかけがあった時
- 既存の `/think` (M4-S) や `/plan` (M4-F) で解決できない、価値観レベルの対立が発生した時

---

## Process

### Step 1: 問いの分類 (Input Analysis)

入力された問いを4つのカテゴリに分類する：

| カテゴリ | 本質的問い | 対応スキル |
|----------|------------|------------|
| **認識** | Knowing (知) | **P1 Noēsis** |
| **意志** | Wanting (意) | **P2 Boulēsis** |
| **探求** | Asking (問) | **P3 Zētēsis** |
| **行為** | Doing (行) | **P4 Energeia** |

### Step 2: 純粋推論 (Pure Inference)

選択されたスキルを発動し、Processing Logic を実行する。
ここで重要なのは「具体化しない」こと。「XXというツールを使う」などの具体的手段ではなく、「XXという状態を目指す」という抽象的理想を維持する。

### Step 3: 拡張定理への委譲判断 (Delegation)

純粋な答えが出た後、現実に落とし込むためのパスを提示する。

- **Fast Path** (Tempo: Fast): M1, M2, M5, M6 へ
- **Slow Path** (Tempo: Slow): M3, M4, M7, M8 へ

---

## 出力形式

```
┌─[Hegemonikón]──────────────────────┐
│ P{N} {Name}: 純粋推論完了          │
│ Q: {問いの本質}                    │
│ A: {純粋な答え}                    │
│ Delegation: → M{X}(F) / M{Y}(S)    │
└────────────────────────────────────┘
```
