# 調査モード

> 各モードは調査の深さとスケールを制御する。
>
> **制約**: モード選択後、出力形式に従った構造化出力を返すこと。

---

## surf (表層調査)

> `/sop.surf` — 広く浅く概要を把握する (Micro Scale)

発動: `/sop surf` または「ざっくり」「概要」「さらっと」

出力形式:

| 項目 | 内容 |
|:-----|:-----|
| トピック | {調査対象} |
| 調査時間 | 5-10分 |
| 概要 | 主要ポイント 3件 |
| 深掘り候補 | さらに調べるべきトピック |
| 確信度 | LOW — 表層のみ |

---

## deep (深層調査)

> `/sop.deep` — 特定のトピックを徹底的に調査する (Macro Scale)

発動: `/sop deep` または「徹底的に」「詳しく」「深掘り」

出力形式:

| 項目 | 内容 |
|:-----|:-----|
| トピック | {調査対象} |
| 調査時間 | 30-60分 |
| 詳細分析 | セクション別の詳細内容 |
| 不確実性 | 確認できなかった点 |
| 根拠 | URL 3件以上 |
| 確信度 | HIGH/MEDIUM |

---

## prag (実践調査)

> `/sop.prag` — 実際に使える情報・手順を調査する (Meso Scale)

発動: `/sop prag` または「どうやるか」「手順」「実装方法」

出力形式:

| 項目 | 内容 |
|:-----|:-----|
| 目的 | {実現したいこと} |
| 手順 | ステップ 3-5件 |
| サンプル | コード例 |
| 注意点 | 落とし穴 |

---

## track (調査進捗追跡)

> `/sop.track` — 調査の進捗状態を追跡し変化を記録する (Meso Scale)

発動: `/sop track` または「調査進捗」「どこまで調べた」

出力形式:

| 項目 | 内容 |
|:-----|:-----|
| 調査テーマ | {テーマ} |
| 完了 | 完了した論点 |
| 進行中 | 進行中の論点 |
| 未着手 | 未着手の論点 |
| 進捗率 | N% |
| 次のアクション | 次に調査すべきこと |

---

> **制約リマインダ**: 各モードの出力形式テーブルに従って構造化出力を返すこと。
