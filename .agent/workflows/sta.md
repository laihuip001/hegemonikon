## 派生モード

> **v2.0 新機能**: 基準の性質に応じた3つの評価モード

| 派生 | 適用場面 | 基準特性 |
|:-----|:---------|:---------|
| **norm** | 規範・標準に基づく基準 | 業界標準、ベストプラクティス |
| **empi** | 経験・実績に基づく基準 | 過去の成功例、数値実績 |
| **rela** | 相対・比較に基づく基準 | 競合比較、ベンチマーク |

### 自動提案の出力形式

> **「/sta だけで自動的に派生が提案される」** — これが美しい

```text

│ 理由: {rationale}                                  │

│                                                    │
│ → このまま {derivative} で設定？ [y/代替/n]         │
└────────────────────────────────────────────────────┘
```

```python
from mekhane.fep.derivative_selector import select_derivative
```

## --mode=pareto (80/20 分析)

> **CCL**: `/sta.pareto` = `/sta{rule=80/20}_/met`
> **目的**: 重要な20%を特定し、80%の成果を得る
> **Scale**: 🔬 Micro

**発動**: `/sta pareto` または「パレート」「80/20」

### プロセス

```text
1. 全項目の列挙
   ↓
2. 各項目の貢献度を推定
   ↓
3. 累積貢献度でソート
   ↓
4. 上位20%を「Vital Few」として抽出
   ↓
5. 出力: Vital Few リスト + 残りは「Trivial Many」
```

### 出力形式

```text
┌─[S3 Stathmos: Pareto 80/20]──────────────────────────┐
│ 対象: {分析対象}                                     │
│                                                      │
│ Vital Few (上位20%):                                 │
│   1. {項目} — {貢献度}%                              │
│   2. {項目} — {貢献度}%                              │
│   3. {項目} — {貢献度}%                              │
│                                                      │
│ Trivial Many (残り80%): {要約}                       │
│                                                      │
│ 推奨: {Vital Few に集中する具体的アクション}         │
└──────────────────────────────────────────────────────┘
```
