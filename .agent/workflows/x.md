# /x: 関係層ワークフロー v5.1

> **Hegemonikón Layer**: X-series (Taxis)
> **目的**: 24定理間の従属関係を定義・参照

---

## 本質

X-series は「関係層」として機能する。
6シリーズが互いに接続し、36の関係を定義する。

```text
┌─────────────────────────────────────────────────────────────┐
│                    X-series: 関係層                          │
│                                                             │
│         ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐     │
│         │ O │───│ S │───│ H │───│ P │───│ K │───│ A │     │
│         └─┬─┘   └─┬─┘   └─┬─┘   └─┬─┘   └─┬─┘   └─┬─┘     │
│           │       │       │       │       │       │       │
│           └───────┴───────┴───────┴───────┴───────┘       │
│                     6 × 6 = 36 関係                         │
└─────────────────────────────────────────────────────────────┘
```

## --mode=toc (制約理論)

> **CCL**: `/x.toc` = `/x_/sta.bottleneck`
> **目的**: システムのボトルネックを特定し、全体最適を図る
> **Scale**: 🌍 Macro

**発動**: `/x toc` または「ボトルネック」「制約」「TOC」

**プロセス**:

```text
1. システム全体を俯瞰
   ↓
2. 制約（ボトルネック）を特定
   ↓
3. 制約を最大限活用
   ↓
4. 他をすべて制約に従属
   ↓
5. 制約を解消 → 新しい制約を発見
   ↓
6. 出力: ボトルネック + 5集中ステップ
```

**出力形式**:

```text

│   1. 特定: {制約は何か}                              │
│   2. 活用: {制約を最大限使う}                        │
│   3. 従属: {他を制約に合わせる}                      │
│   4. 解消: {制約を取り除く}                          │
```
