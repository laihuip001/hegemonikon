# /x: 関係層ワークフロー

> **Hegemonikón Layer**: Level 2' X-series (Taxis)
> **目的**: 定理間の従属関係（因果・抑制・促進・循環）を分析・適用
> **参照**: [taxis.md](../../kernel/taxis.md)

---

## 本質

X-series は「関係層」として機能する。
他のシリーズ間の依存関係・連鎖を定義する。

| サブシリーズ | 数 | 対象 |
|-------------|-----|------|
| X-O | 4 | O-series（純粋定理）間の従属 |
| X-T | 8 | T-series（拡張定理）間の従属 |
| X-K | 12 | K-series（文脈定理）間の従属 |

---

## 発動条件

| トリガー | 説明 |
|----------|------|
| `/x` | 関係層ワークフローを起動 |
| 処理フロー設計 | Workflow の順序を設計する際 |
| 循環検出 | 処理が循環している可能性を分析 |
| 矛盾検出 | 複数の Kairos が矛盾している可能性 |

---

## Process

### Step 1: 対象シリーズの選択

| コマンド | 対象 |
|----------|------|
| `/x o` | X-O（O-series 間の関係） |
| `/x t` | X-T（T-series 間の関係） |
| `/x k` | X-K（K-series 間の関係） |
| `/x` | 全体を俯瞰 |

### Step 2: 関係の可視化

選択されたサブシリーズの従属関係を図示。

### Step 3: 応用

- **因果連鎖**: A → B → C の連鎖を追跡
- **循環検出**: A → ... → A の循環を検出
- **矛盾検出**: 相反する Kairos の組み合わせを警告

## 使用例

**例1: Workflow 設計**
```
/x t
→ X-T の処理フローを表示
→ /plan は T2→T4、/code は T4→T6→T7 を使用
```

**例2: 矛盾検出**
```
/x k
→ K1-FL + K4-HS の矛盾を検出
→ 「短期×低次」と「長期×高次」は両立困難
```
