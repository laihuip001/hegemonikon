> Hegemonikón 定理ワークフローの使い方と選び方

## シリーズコマンド一覧 (Δ層)

| コマンド | シリーズ | 日本語名 | 主な用途 |
|:---------|:---------|:---------|:---------|
| `/ax` | 全定理統合 | 公理 | 多層分析 |
| `/o` | Ousia | 本質 | メタ認知的問い |
| `/s` | Schema | 様態 | 戦略・計画 |
| `/h` | Hormē | 傾向 | 動機・衝動 |
| `/p` | Perigraphē | 境界 | 環境・制約 |
| `/k` | Kairos | 文脈 | タイミング・目的 |
| `/a` | Akribeia | 精密 | 検証・精度 |
| `/x` | Taxis | 関係 | 依存関係 |

---

## ユースケース別選択ガイド

### 🎯 Case 1: 「何をすべきかわからない」

**推奨**: `/o` → `/noe` (深掘り)

### 🎯 Case 2: 「どう進めるか決めたい」

**推奨**: `/s` → `/plan`

### 🎯 Case 3: 「今やるべきか判断したい」

**推奨**: `/k` → `/pri`, `/why`

### 🎯 Case 4: 「検証・確認したい」

**推奨**: `/a` → `/dia`, `/vet`, `/fit`

### 🎯 Case 5: 「全体像を把握したい」

**推奨**: `/ax`

---

## フローチャート

```text
問いの性質は？
  ├─ 本質・抽象 → /o (/noe, /bou, /zet, /ene)
  ├─ 戦略・計画 → /s (/plan)
  ├─ 動機・衝動 → /h (/bye, /boot)
  ├─ 環境・制約 → /p (/dev, /sop)
  ├─ タイミング → /k (/pri, /why)
  ├─ 検証・精度 → /a (/dia, /vet, /fit)
  ├─ 依存関係 → /x
  └─ 複合・不明 → /ax
```
