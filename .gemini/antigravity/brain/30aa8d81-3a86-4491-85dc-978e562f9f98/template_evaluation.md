## 2. 重要な発見

### 2.1 「簡易版で劣化」は誤解だった

パプ君の出力を確認した結果、**簡易版でも2618行の詳細レポートを生成**していた。

実際の構造:

- エグゼクティブサマリー ✅
- 位置バイアスの実証的知見 ✅
- 最適プロンプト長の実測データ ✅
- Perplexity Sonar特有の処理特性 ✅
- 用途別テンプレート（コピペ可能版）✅
- 根拠リンク（20+ソース）✅

### 2.2 Claude特化の追加知見（ハイブリッド版）

ハイブリッド版はClaude固有の以下の情報を追加で取得:

1. **XML + Markdown ハイブリッドが最適**
   - トークン効率: Markdown単独より -5～-10%
   - 精度向上: +45-55%

2. **Lost in the Middle 問題**
   - Claude にも U字形バイアスが存在
   - 冒頭と末尾が優先、中盤 -25～-40% 精度低下

3. **Claude 4.5 世代の確定**
   - Opus 4.5: 複雑推論で +15-25% 優位
   - Sonnet 4.5: コスト効率（90-95%精度で30-40%コスト削減）
   - 1M コンテキスト: Beta（Sonnet のみ）

4. **段落形式の優位性** (2025年10月以降)
   - 箇条書きより段落（prose）が1.5倍詳細な説明を生成

## 3. テンプレート設計の結論

### 3.1 「劣化」ではなく「差分」

| 観点 | 簡易版 | ハイブリッド版 |
|------|--------|--------------|
| Perplexity最適化 | ✅ 十分 | ✅ 十分 |
| Claude特化 | ❌ 含まれない | ✅ 含まれる |
| 依頼書作成時間 | 短い | 長い |

### 3.2 推奨

**用途に応じて使い分けるべき**:

- **Perplexity調査のみ**: 簡易版で十分
- **Claude処理最適化も必要**: ハイブリッド版を使用

### 3.3 /zet テンプレートの最終形

現在の `zet.md` は **冒頭凝縮 + 詳細維持のハイブリッド** になっている。これは正しい方向。

**ただし、Claude特化のセクションを追加すべき**:

```markdown
## 【Claude処理最適化】（Final Consumer向け）

- XML タグ（役割定義）+ Markdown（内容）を推奨
- 重要情報は冒頭に配置（Lost in the Middle 対策）
- 段落形式を優先（箇条書きより詳細な説明を生成）
```
