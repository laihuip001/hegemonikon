# Perplexity調査結果: LLM向け依頼書最適設計

> **調査日**: 2026-01-28
> **ソース**: Perplexity Deep Research
> **時間範囲**: 2025年7月〜2026年1月

---

## エグゼクティブサマリー

| 項目 | 最適値/推奨 |
|------|------------|
| プロンプト長 | **500-2,000トークン**（日本語: 1,500-6,000文字） |
| 冒頭優先順位 | フォーマット指定 → タスク定義 → 時間制約 → スタイル |
| 時間制約表現 | **「過去X日」**（相対日付）が最効果的 |
| Deep Research | 複雑問題時に推奨（精度+95%、処理3-5分） |

---

## 1. 位置バイアスの実態

**結論**: 「冒頭優先」は過度に単純化。モデル依存である。

| モデル | バイアス傾向 |
|--------|-------------|
| Llama3.1-8B | 冒頭優先（従来通り） |
| Qwen2.5-7B | **後段優先**（異例） |
| DeepSeek-7B | **後段優先**（異例） |

## 2. Perplexity冒頭に置くべき要素（優先順位）

### 優先度1: 出力フォーマット指定

```
「構造化テーブル（4列: モデル, 文脈制限, 料金, 強み）を返す。
その後、2-3文の結論を formal トーンで記述。」
```

### 優先度2: タスク定義（1-2文）

### 優先度3: 時間制約

```
「2025年7月以降の公開情報のみ。
2024年以前は歴史的背景として参考程度。」
```

### 優先度4: スタイル・トーン

---

## 3. 時間制約の効果的な表現

| 表現方法 | 効果 | 推奨度 |
|---------|------|--------|
| 「最新情報をお願いします」 | 低 | ❌ |
| 「2025年7月〜1月の情報のみ」 | 中 | ⚠️ |
| **「過去6ヶ月以内の研究」** | **高** | ✓ 推奨 |
| **「2024以前は参考程度」** | **高** | ✓ 推奨 |

---

## 4. Perplexity特有機能

### 4.1 時間予算指定

```
"Spend 3 minutes analyzing…"
→ Deep Research 自動起動
```

### 4.2 引用強化（幻覚低減 +20-30%）

```
「各出典にURLを付記、または'わかりません'と答えよ」
```

### 4.3 Sub-Query 自動生成

```
「Consider these dimensions: [A] 技術的 [B] 経済的 [C] 規制的」
→ 3つの並列検索実行
```

---

## 5. 失敗パターン

- ❌ 超長プロンプト（5,000+トークン）→ 中盤情報喪失
- ❌ 「最新の情報をお願いします」→ フィルタ非適用
- ❌ 特殊マーカー過多（`<goal>`等）→ 2025年中盤でブロック
