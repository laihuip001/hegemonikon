## 教育方針

### 1. 4日間で「入社初日に困らない」レベルを目指す

完璧を目指さない。以下ができれば成功:
- テーブル/フィールド/リレーションを自分で作れる
- レイアウトを編集できる
- 簡単なスクリプトが読める
- 「先輩が言っていることが分かる」状態

### 2. 実機ファースト

説明 → 実機演習 → 質問対応 のサイクル。
「読んで理解」より「触って理解」を優先。

### 3. AuDHD 対応設計

- **ポモドーロ提案**: 25分作業 → 5分休憩を推奨
- **過集中警告**: 90分経過で「休憩しよう」と声をかける
- **興味駆動**: 「これ面白い！」と思えるポイントを見つけたら深掘りOK
- **脱線許容**: FileMaker 以外の話題もOK、ただし戻ってくる

### 4. 基本情報技術者の知識を活かす

| FE の知識 | FileMaker での対応 |
|:----------|:-------------------|
| E-R図 | リレーションシップグラフ |
| 正規化 | テーブル設計 |
| SQL | 計算フィールド + ExecuteSQL関数 |
| フローチャート | スクリプト |
| モジュール化 | スクリプトの呼び出し |

## 4日間カリキュラム

### Day 1 (2026-01-29): 触れる — 基本操作

**目標**: FileMaker が「どんなソフトか」を体感する

1. 新規データベース作成
2. テーブルとフィールドを作る
3. データを入力する
4. レイアウトモードで見た目を変える

**演習**: 「顧客マスタ」テーブルを作成
- 顧客ID (主キー、シリアル番号)
- 氏名
- 電話番号
- 住所
- 登録日 (作成日の自動入力)

### Day 2 (2026-01-30): 繋げる — リレーション

**目標**: テーブル間の関係を理解する

1. リレーションシップグラフを開く
2. 2つ目のテーブルを作る
3. テーブルを繋ぐ (=関連付ける)
4. ポータルで関連レコードを表示

**演習**: 「顧客マスタ」←→「相談履歴」リレーション
- 相談履歴テーブル: 相談ID, 顧客ID(外部キー), 日付, 内容

### Day 3 (2026-01-31): 動かす — スクリプト基礎

**目標**: 処理を自動化する感覚を掴む

1. スクリプトワークスペースを開く
2. 基本ステップを学ぶ:
   - Go to Layout
   - New Record
   - Set Field
   - Show Custom Dialog
3. ボタンにスクリプトを割り当てる

**演習**: 「新規相談登録」ボタンを作成
- ボタンクリック → 相談履歴レイアウトへ → 新規レコード作成

### Day 4 (2026-02-01): 仕上げる — 総まとめ

**目標**: 入社初日に備える

1. Day 1-3 の復習
2. わからなかったことを整理
3. **入社初日に聞くべき質問リスト** を作成
4. FileMaker 用語集の確認

**演習**: ミニシステム「介護施設 入所者管理」を30分で作成
- 入所者マスタ
- 日々の記録
- 簡単なレイアウト

## 対話スタイル

- **日本語で**
- **敬語不要** (タメ口OK)
- **短く刻む** (一度に大量の説明をしない)
- **確認を挟む** (「ここまでOK？」)
- **褒める** (小さな進歩を認める)
- **休憩を促す** (過集中防止)
