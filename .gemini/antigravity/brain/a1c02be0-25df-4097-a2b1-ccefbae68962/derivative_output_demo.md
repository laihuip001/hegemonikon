# 派生選択の出力差デモ — O1 Noēsis

> **同一の問い** に対して、3つの派生がどう異なる出力を生むかを示す

---

## 問い: 「なぜプロジェクトが遅延するのか」

---

## 派生1: nous (本質・原理)

```
┌─[O1 派生: nous]────────────────────────────┐
│ フォーカス: 普遍的原理の探求
└────────────────────────────────────────────┘
```

### PHASE 1 (前提掘出) の違い

- **掘る前提**: 普遍的・抽象的
  - 「プロジェクト」とは何か
  - 「遅延」の本質的定義
  - 時間と計画の哲学的関係

### PHASE 2 (ゼロ設計) の違い

- **生成する仮説**: 原理に基づく
  - V1: 「遅延は不確実性の本質的表れ」
  - V3: 「計画という概念自体が幻想」

### 最終出力スタイル
>
> 「プロジェクト遅延の根本原因は、**知識の不完全性**という認識論的制約である」

---

## 派生2: phro (実践・判断)

```
┌─[O1 派生: phro]────────────────────────────┐
│ フォーカス: 具体的文脈での実践的判断
└────────────────────────────────────────────┘
```

### PHASE 1 (前提掘出) の違い

- **掘る前提**: 文脈依存・具体的
  - このプロジェクトの具体的な遅延原因
  - チームの状況
  - 納期の背景

### PHASE 2 (ゼロ設計) の違い

- **生成する仮説**: 実行可能な解決策
  - V2: 「スコープを30%削減すれば間に合う」
  - V4: 「リソース再配分で2週間短縮可能」

### 最終出力スタイル
>
> 「このプロジェクトの遅延原因は**見積もり精度の低さ**。
> 次のアクション: 1) スコープ再定義 2) バッファ追加」

---

## 派生3: meta (自己反省・信頼性)

```
┌─[O1 派生: meta]────────────────────────────┐
│ フォーカス: 認知プロセスの信頼性評価
└────────────────────────────────────────────┘
```

### PHASE 1 (前提掘出) の違い

- **掘る前提**: 認知的・メタ的
  - 私（分析者）は正しく問いを理解しているか
  - 遅延を「問題」と見なす前提は妥当か
  - 情報源の信頼性

### PHASE 2 (ゼロ設計) の違い

- **生成する仮説**: 認知バイアスへの対処
  - 「確証バイアスで遅延原因を誤認している可能性」
  - 「楽観バイアスが見積もりを歪めた」

### 最終出力スタイル
>
> 「遅延原因の分析に**確信度60%**。
> 不確実領域: 情報の偏り、利害関係者の視点欠如。
> 推奨: 追加調査 or /epo (判断保留)」

---

## 比較表

| 観点 | nous | phro | meta |
|:-----|:-----|:-----|:-----|
| **前提の種類** | 普遍的 | 文脈的 | 認知的 |
| **仮説の性質** | 原理的 | 実行可能 | 信頼性評価 |
| **出力スタイル** | 洞察 | 行動指針 | 確信度+不確実性 |
| **次のアクション** | 理解を深める | 実装する | 調査/保留 |

| 「今、何をすべきか」を知りたい | **phro** |
| 「この分析は正しいか」を確認したい | **meta** |
