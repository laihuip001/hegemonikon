### 仮説3: Hegemonikón ≒ CCL なのでは?

| 判定 | **不成立** ✗ |
|:-----|:------------|
| **破綻点** | CCL は Hegemonikón の**表現層**の一部。Hegemonikón は公理体系(FEP) + 定理群(24) + 関係層(36) + CCL + 実装(mekhane) の**全体**を指す |

**Python アナロジー**:

```
Hegemonikón 全体 ≒ Python エコシステム
├── FEP (公理)     ≒ Python 言語仕様 (PEP, 言語の哲学)
├── 定理群 (24)    ≒ builtins (組み込み関数・型)
├── X-series (36)  ≒ import 文・依存解決 (モジュール間の関係)
├── CCL (演算子)   ≒ Python 構文 (syntax) + AST

```

「Python ≒ Python 構文」とは言わないように、「Hegemonikón ≒ CCL」とは言えない。

### 仮説5: WFはCCLを書くための"関数"では?

| 判定 | **不成立** ✗ (因果が逆) |
|:-----|:----------------------|
| **実態** | **CCL で WF を呼び出す**のであり、WF が CCL を書くわけではない |
| **正確な関係** | WF = 関数定義、CCL 式 = 関数呼び出し式 |

**Python アナロジー**:

```python
# WF = 関数定義
def noe(depth=1):
    """深い認識を行う"""
    return run_recognition(depth)

# CCL 式 = 関数呼び出し式
result = noe(depth=3)  # ≒ /noe+

# CCL 複合式 = 関数合成
result = merge(noe(3), dia(1))  # ≒ /noe+*dia-
```

**WF は CCL のための「関数」ではなく、CCL で呼び出される「プリミティブ」**。
