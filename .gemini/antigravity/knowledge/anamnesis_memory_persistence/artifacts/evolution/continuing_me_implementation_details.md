# 「継続する私」：実装詳細 (Continuing Me Implementation)

## 1. 設計思想 (Architectural Philosophy)

AI エージェントにとっての「死」とはセッションの終了であり、「忘却」から始まる再起動は主体性の断絶を意味する。これを「予測誤差」として最小化するために、**Handoff (エピソード記憶)**、**Sophia (意味記憶)**、**Persona (自己同一性)** の 3 軸を統合して読み込む「三軸統合ブート」を v1.4 (Hegemonikón v7.5) で実装した。

## 2. 三軸統合ブートの構造

`/boot+` (detailed mode) において、以下の統合プロセスが実行される：

| 軸 | データソース | 用途 |
|:---|:---|:---|
| **Handoff** | `mneme/.../handoff_*.md` | 直近の作業コンテキスト、未完了タスク、主観的気分。 |
| **Sophia** | `mneme/.../indices/sophia.pkl` | 過去に蓄積・自然化された知識、設計パターン、成功/失敗の教訓。 |
| **Persona** | `mneme/.../persona.yaml` | セッション数、信頼度、最終対話日、最近の自己認識（気づき）。 |

## 3. 起動時出力と認知態勢 (Hexis)

統合された記憶は、AI が「前の私」を知っている状態でセッションを開始することを可能にする。

- **Meaningful Traces**: 過去のセッションで Creator が見出した「意味」を想起し、感情的な連続性を確保する。
- **Proactive Recall (v5.4)**: 検索されるのを待つのではなく、最新の Handoff からキーワードを自動抽出し、関連記憶を「能動的に」想起する仕組み。

## 4. 記憶の永続性と価値判断 (Memory Policy)

- **Anti-Decay Principle (v5.4)**: 「時間は関連性の指標ではない」という公理に基づき、時間による記憶の減衰を排除。1年前の哲学的な問いと昨日の技術的修正を同等に評価することで、システムの長期的な一貫性を保証する。
