## 1. Sacred Routes (黄金経路)

頻出する3つの定型接続を「黄金経路」として定義。これらは `@route(name)` で即座に発動可能。

### 🏛️ The Architect's Route: O → S

- **意味**: 認識から設計へ。
- **経路**: `/noe → /s`
- **マクロ**: `@route(architect)`

### 🔄 The Execution Cycle: S → O

- **意味**: 設計から実行へ。
- **経路**: `/s → /ene`
- **マクロ**: `@route(execution)`

### 🧘 The Philosopher's Route: K → O

- **意味**: 文脈（知恵）から本質（認識）へ。
- **経路**: `/sop → /noe`
- **マクロ**: `@route(philosopher)`

## 2. Graduated X-series Integration (段階的統合)

関係性の出力は認知負荷と実用性のバランスを取るため、以下の2段階で出力される。

### 🟢 簡潔版 (Brief Output)

- **トリガー**: 通常の `/wf` 実行後。
- **出力**: 3行程度の推奨次ステップ。
- **目的**: 流れを止めずに次の一手を提示。

### 🔴 詳細版 (Detailed Output)

- **トリガー**: `@next` マクロの明示的呼出し。
- **出力**: 36関係マトリクスに基づく全接続の深化。
- **目的**: 停滞時や分岐点での多角的探索。

---

## 3. Data-Driven Route Update

Sacred Routes は固定された教条ではなく、**Doxa (信念層)** に記録された実際の使用統計に基づき周期的に更新される。

- **Data-Driven Routes**: 実際に使われた関係を記録・分析。
- **Weighted Relations**: 頻度に応じた関係の重み付け。

## 4. X-series Macros & Operators

| 記号 | 名称 | 作用 |
|:-----|:-----|:-----|
| `*` | 融合演算子 | `/wf*/x` で X-series との明示的融合を宣言。 |
| `@next` | 動的推奨 | 現在の WF から最適な次ステップを X-series 経由で探索。 |
| `@route` | 経路呼出 | 黄金経路 (architect/execution/philosopher) を発動。 |
| `@connect` | シリーズ接続| 特定シリーズ (O, S, H, P, K, A) への接続を強調。 |
