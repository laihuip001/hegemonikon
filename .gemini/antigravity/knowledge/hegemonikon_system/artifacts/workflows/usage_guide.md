## 2. どれを使えばいいか: TPO 早見表

### 💡 状況別 推奨ワークフロー

| こんな時 | 使う WF | 理由 |

| Hub | 問い | いつ使う |

## 3. Hub vs 個別定理: いつどちらを使うか

### Hub を使う場合

```
/o  → 認識→意志→探求→行為 の4定理を巡回（4項振動）
```

**使う場面**: 問題が複雑で、どの定理が適切か自分でもわからない時

### 個別定理を使う場合

```
/noe → O1 Noēsis だけを実行
```

**使う場面**: やりたいことが明確な時

### 判断フロー

```
迷っている？
  ├─ Yes → Hub を使う（/o, /s 等）
  └─ No  → 個別定理を使う（/noe, /bou 等）
```

## 5. 派生（Derivative）の選び方

各定理には **3つの派生** がある（計72派生）。

### 例: O1 Noēsis の派生

| 派生 | 意味 | いつ使う |
|:-----|:-----|:---------|
| `nous` | 本質直観 | 抽象的な原理を掴みたい |
| `phro` | 実践判断 | 具体的な状況で判断したい |
| `meta` | メタ認識 | 自分の認識を振り返りたい |

### 派生の指定方法

```
/noe             → 自動選択（FEP が状況判断）
/noe@nous        → 強制的に nous モード
/noe --mode=meta → 強制的に meta モード
```

**推奨**: 最初は自動選択に任せ、慣れたら手動指定

## 7. X-series: 次に何をすべきか

実行後、X-series が次の WF を推奨する。

```
/noe 実行後 → 推奨: /s (認識後に設計)
/bou 実行後 → 推奨: /zet (意志後に問いを立てる)
/ene 実行後 → 推奨: /dox (活動後に記録)
```

**使い方**: 出力の「次の推奨」を参考にするか、`/x` で全関係を確認

## 8. Cognitive Algebra（CCL）との組み合わせ

詳細は [Cognitive Algebra 入門](./cognitive_algebra_introduction_20260129.md) を参照。

### 最低限覚えておくべき演算子

| 演算子 | 意味 | SEL 強制 (minimum_requirements) |
|:---:|:---|:---|
| `+` | **深化 (MUST)** | 5-Whys, 根拠提示, ステップ完全実行を強制 |
| `-` | **縮約 (MAY)** | 1行サマリー、最小限の応答を許可 |
| `!` | **全展開 (MUST)** | 全派生ワークフローの並列/同時思考を強制 |
| `^` | **メタ分析 (MUST)** | 前提の疑義、高次元からの俯瞰を強制 |
| `*` | **融合** | X-series 関係層による知識統合を強制 |

> **SEL (Semantic Enforcement Layer)**: 全ての `/` コマンドは演算子に基づき、ワークフローフロントマターで定義された「最低要件」を遵守しなければならない。遵守率は `/vet` で監査される。

## 9. トラブルシューティング

### Q: どの WF を使えばいいかわからない

→ `/bou-` で「今日何をしたいか」を1行で出力

### Q: Hub と個別定理の違いがわからない

→ Hub = 4定理の巡回、個別 = 1定理のみ。迷ったら Hub。

### Q: 派生が多すぎて覚えられない

→ 覚えなくていい。自動選択に任せる。

### Q: CCL が複雑すぎる

→ まず `+`, `-`, `_` の3つだけ覚える。
