# Claude / Gemini モデル選択ガイド

> **目的**: Antigravity IDEにおいて、タスク特性に応じて最適なAIモデルを選択する。
> **根拠**: Perplexity調査レポート「Claude vs Gemini 性格比較」

---

## モデル性格サマリー

| 特性 | Claude (Opus/Sonnet) | Gemini (3 Pro/Flash) |
|------|---------------------|----------------------|
| **設計思想** | Constitutional AI（安全性最優先） | マルチモーダル最適化（適応性重視） |
| **推論スタイル** | 決定論的（一貫性高） | 非決定論的（複数案並列） |
| **ルール解釈** | 厳格・形式的 | 柔軟・文脈適応的 |
| **MBTI類型** | ISTJ（責任感・原則的） | ENTJ（実用的・多才） |
| **強み** | 精度、長文一貫性、倫理判断 | 速度、マルチモーダル、創造性 |

---

## タスク別推奨モデル

### Claude を選ぶべき場面

| カテゴリ | 具体例 |
|----------|--------|
| **精密実装** | リファクタリング、アーキテクチャ設計、型安全なコード |
| **長文処理** | 法的文書、技術ドキュメント、研究論文 |
| **規制・監査対象** | セキュリティ要件、コンプライアンス、品質保証 |
| **倫理的判断** | ガイドライン遵守、ポリシー策定 |

### Gemini を選ぶべき場面

| カテゴリ | 具体例 |
|----------|--------|
| **探索的タスク** | ブレインストーミング、MVP開発、プロトタイピング |
| **マルチモーダル** | 画像分析、UI/UX設計、図解生成 |
| **高速処理** | 大量バッチ、初期調査、トリアージ |
| **文化的多様性** | 非英語圏コンテンツ、地域固有の文脈 |

## Hegemonikón ワークフロー対応

| ワークフロー | 推奨モデル | 理由 |
|--------------|------------|------|
| `/plan` | Claude | 計画は一貫性と網羅性が重要 |
| `/do` | Claude/Gemini | 複雑な実装→Claude、プロトタイプ→Gemini |
| `/rev` | Claude | 検証は厳格さが重要 |
| `/ask` | Gemini | 情報収集は速度と多様性が有効 |
| `/think` | Claude | 深い思考は決定論的推論が有効 |

---

## 注意事項

> [!WARNING]
> **Model Decision モード使用時の注意**
> 
> 同じルールでもモデルにより解釈が異なる:
> - **Claude**: 曖昧な条件 → 安全側に解釈（ルール適用しがち）
> - **Gemini**: 曖昧な条件 → 文脈で判断（スキップしがち）
> 
> `model_decision` を使うルールは、トリガー条件を明確に記述すること。
