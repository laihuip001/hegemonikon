# エラーメッセージ評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- `_request` メソッドにおいて、APIエラー発生時に `resp.raise_for_status()` を使用しているため、APIが返す詳細なエラーメッセージ（JSONボディ）が例外に含まれず、呼び出し元で原因特定が困難である（ログには出力されるが、ライブラリ利用者には不親切）。
- `_request` メソッドのエラーログ出力において、`body[:200]` とレスポンス本文を切り詰めているため、重要なデバッグ情報が欠落する可能性がある。
- `poll_session` メソッドの `TimeoutError` において、タイムアウト発生時の直前のセッション状態（State）が含まれていないため、どのフェーズでスタックしたかが不明瞭である。
- `RateLimitError` のメッセージが "Rate limit exceeded" のみであり、`retry_after` の値がある場合でもメッセージ文字列に含まれないため、ユーザーへの案内として不十分である。

## 重大度
- Medium
