# エラーメッセージ評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- `_request` メソッド内の `raise_for_status()` は標準的なHTTPエラーのみを返し、APIからの詳細なエラーメッセージ（body）を例外に含めないため、ユーザーが原因を特定しづらい。
- ログ出力時の `body[:200]` による切り詰めは、長いエラーメッセージの重要な部分を隠してしまうリスクがある。
- `poll_session` の `TimeoutError` メッセージはコンテキスト（期待される完了時間や推奨アクション）が不足しており、ユーザーへの共感性に欠ける。

## 重大度
- High
