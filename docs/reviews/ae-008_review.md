# シンプリシティ評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **`synedrion_review` メソッドの責務過多**: HTTPクライアント内にドメイン固有のビジネスロジック（パースペクティブのロード、フィルタリング、プロンプト生成）が含まれています。これは関心の分離（Separation of Concerns）に違反し、クライアントを `mekhane.ergasterion.synedrion` に強く結合させています。
- **ブロッキングI/Oの混入**: `synedrion_review` 内の `PerspectiveMatrix.load()` は同期処理の可能性が高く、非同期メソッド内でイベントループをブロックする恐れがあります。
- **不要なCLIコード**: ライブラリファイル内にテスト用の `main()` 関数と `argparse` ロジックが含まれており、ファイルサイズと認知負荷を増大させています。
- **過剰なコメントとレビュー参照**: 過去のレビューID（`cl-003`, `th-003` など）や「自己代入を削除」といった修正履歴コメントが散乱しており、コードの可読性を損なっています。これらはGitの履歴に残すべき情報です。
- **レガシーコードの残留**: `parse_state` 関数が「Legacy alias」として残されていますが、新規リファクタリングであれば削除すべきです。
- **エラー処理の複雑さ**: `batch_execute` において、例外発生時に `JulesSession` オブジェクトを擬似的に作成してエラー情報を詰めるロジックは、データ構造の意味を曖昧にし、理解を困難にしています。
- **セッション管理の複雑性**: `_request` メソッド内で、共有セッションの有無に応じた一時セッションの生成・破棄ロジックが混在しており、メソッドの複雑度を上げています。
