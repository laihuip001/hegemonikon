# シンプリシティ評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- `synedrion_review` メソッドにおける重大な結合度の高さ: クライアントクラス内にビジネスロジック（`PerspectiveMatrix` のロード、フィルタリング、480の視点生成など）が直接実装されています。これは単一責任の原則に違反しており、クライアントを不必要に複雑にしています。ドメインロジックは別のサービス層に分離すべきです。
- 不要なCLIロジック: `if __name__ == "__main__":` および `main()` 関数が含まれていますが、これらはライブラリファイルには不要であり、テストや使用例は別ファイルに分けるべきです。
- レガシーコードの残存: `parse_state` という「Legacy alias」が残されており、APIの表面積を不必要に広げています。
- `_request` メソッド内のセッション管理: 一時的なセッション生成とクリーンアップのロジックが混在しており、メソッドの責任範囲が曖昧になっています。
