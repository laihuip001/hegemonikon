# 比喩一貫性の詩人 レビュー

## 対象ファイル
`hermeneus/src/dispatch.py`

## 判定
発言（要改善）

## 発見事項
- **比喩混在 (Medium)**: モジュール名 `dispatch` (物流/工場) と内部処理 `format_ast_tree` (庭園/自然) が混在しています。「木」から「ワークフロー」を「抽出 (extract)」して「計画 (plan)」を作るという、自然物と工業製品のメタファーが衝突しています。Hermēneus（解釈）ドメインに合わせて `interpret` や `translate` への統一、あるいは `tree` 表記の抽象化が望まれます。
- **命名規則不統一 (Low)**: `format_ast_tree` (verb_noun_noun) と `extract_workflows` (verb_noun) で命名の粒度が異なります。AST は本質的に木構造であるため、`format_ast` または `render_structure` と簡潔にするか、`extract_workflows_from_ast` と対称性を保つべきです。
- **動詞不統一 (Low)**: `dispatch` (配送/指令) というモジュール名に対し、実態は `parse` (解析) と `plan` (計画作成) です。動詞が実態と乖離しており、また `extract` (抽出) という物理的な操作用語も混ざっています。
