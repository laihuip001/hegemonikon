# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/fep/tests/test_attractor_advisor.py`

## 判定
発言（要改善）

## 発見事項
- (Medium) 比喩混在: `SeriesAttractor` (自然/物理: Attractor) が `diagnose` (医療: 診断) を行い、その結果が `SuggestResult` (社会: 提案) となっている。また `_build_diagnosis` (工場: build) という表現も混入している。物理現象としての「アトラクタ」なら `converge` や `sense`、医療なら `examine`、工場なら `inspect` に統一すべき。現状は Nature, Medical, Factory, Social が混在している。
- (Low) 動詞不統一: `advisor.recommend` と `attractor.suggest` で類似の「提案」行為に異なる動詞が使われている。`propose` や `guide` などに統一を検討されたし。
- (Low) 命名規則不統一: `diagnose()` メソッドが `SuggestResult` を返している。`DiagnosisResult` か、あるいはメソッド名を `suggest_based_on_diagnosis()` などにするなど、行為と返り値の整合性を取るべき。

## 重大度
Medium
