# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/pks/push_dialog.py`

## 判定
発言（要改善）

## 発見事項
- **比喩混在 (Medium)**
  - 「Push」（通信/配送）と「Nugget」（採掘）と「Gnosis/Anamnesis」（神秘主義/哲学）が混在しています。
  - `PushDialog` という機械的な名前に対し、扱う対象が `KnowledgeNugget`（金塊）であり、背後にある `GnosisIndex`（霊知）との落差が大きいです。
  - "Push" された "Nugget" について "Dialog" する、という状況設定が比喩的に散漫です。

- **命名規則不統一 (Low)**
  - メソッド名が動詞で統一されておらず、品詞が混在しています。
    - `why()` (副詞/疑問詞)
    - `deeper()` (形容詞比較級)
    - `related()` (形容詞)
  - Pythonのメソッド命名慣習（動詞開始）に反しており、直感的なインターフェース（CLIコマンドとの対応）を優先した結果、コードとしての整合性が損なわれています。

- **抽象度の不統一 (Low)**
  - `related()` メソッド内で `mekhane.anamnesis.index` を直接 import しており、対話層（高レベル）と検索実装層（低レベル）が混在しています。

## 修正案
比喩を「採掘（Mining）」または「探求（Exploration）」に統一し、メソッド名を動詞化することを提案します。

```python
class KnowledgeInspector:  # "PushDialog" -> "Inspector" (調査官/鑑定士)
    ...
    def explain_reason(self, nugget: KnowledgeNugget) -> str:  # why -> explain_reason
        ...

    def dig_deeper(self, nugget: KnowledgeNugget, question: str) -> str:  # deeper -> dig_deeper
        ...

    def explore_related(self, nugget: KnowledgeNugget, k: int = 5) -> list[KnowledgeNugget]:  # related -> explore_related
        ...
```
