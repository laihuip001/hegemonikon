# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/pks/feedback.py`

## 判定
発言（要改善）

## 発見事項

- **Metaphor Mixing (Medium)**
  - `PushFeedback` (Cybernetics) / `nugget_title` (Mining)
  - 制御システム（Control System）の比喩を基調とするこのモジュールに、知識採掘（Mining）の用語 `nugget` が混入しています。
  - フィードバックループという機械的な構造において、処理対象が「金塊（Nugget）」である必要はなく、より抽象的な「信号（Signal）」や「対象（Target）」であるべきです。
  - 提案: `nugget_title` → `target_signal_id` / `content_id`

- **Naming Inconsistency (Low)**
  - `FeedbackCollector` (Archive) / `adjust_threshold` (Cybernetics)
  - `Collector` は静的な収集・保管を示唆しますが、実態は `adjust_threshold` を通じてシステムの挙動を動的に変える `Controller` または `Regulator` です。
  - 提案: `FeedbackCollector` → `FeedbackController` / `FeedbackLoop`

- **Verb Inconsistency (Low)**
  - `record` (Archive) / `persist` (System) / `get_stats` (Generic)
  - `record` は記録係の所作ですが、`persist` はシステムの状態保存です。また、`get_stats` は `adjust_threshold` のような強い操作に対して弱すぎます。
  - 提案: `record` → `input_signal` / `receive`, `get_stats` → `compute_metrics` / `analyze_state`

## 重大度
Medium
