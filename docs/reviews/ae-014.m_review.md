# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/anamnesis/gnosis_chat.py`

## 判定
発言

## 発見事項
- (Medium) 同一モジュール内で異なる比喩体系（工場と庭園が混在）: モジュール名 `anamnesis` (想起) や `Gnosis` (知識), `Mneme` (記憶) といった「認知・哲学」の比喩に対し、`Indexer`, `Reranker`, `_build_context`, `_chunk_text` といった「工場・機械」の比喩が混在している。また、ファイル発見に `discover` (庭園・探求) という動詞も用いられている。
- (Low) 関連クラス間で命名規則の不統一: `KnowledgeIndexer` (対象+役割) に対し、`Reranker` (役割のみ)、`ConversationHistory` (対象+機能)、`GnosisChat` (製品名) となっており、構成要素の命名パターンが一致していない。
- (Low) 動詞の活用不統一（get/fetch/retrieve が混在）: 検索・取得に関して `_retrieve` (メソッド), `fetch_k` (変数), `_get_embedder` (内部メソッド呼び出し) が混在している。また、ファイル発見に `discover` を用いる一方で、検索フラグには `search` を用いている。
- (Low) 抽象度の不統一（Manager と helper が同列）: `KnowledgeIndexer` は静的メソッドのみのユーティリティ（実質的な関数群）として実装されているが、`Reranker` や `ConversationHistory` は状態を持つオブジェクトとして実装されており、クラスとしての扱い（抽象度）が不揃いである。

## 重大度
Medium
