# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/dendron/checker.py`

## 判定
発言（要改善）

## 発見事項
- **比喩混在 (Medium)**: モジュール名は `dendron`（樹木・ギリシャ語）であり、`root` や `tree` といった自然界の比喩が使われているが、クラス名やメソッド名は `Checker`, `validate`, `exempt`, `Proof`, `Quality` といった工場・法務・官僚的な比喩が支配的であり、世界観が衝突している（機械的な検査員が生物的な森を歩いている）。
- **命名規則不統一 (Low)**: `check_file_proof` / `check_dir_proof`（対象の状態確認）に対し、`check_functions_in_file` / `check_variables_in_file`（内部要素の探索）と、前置詞 `in_file` の有無による命名の揺れがある。
- **動詞不統一 (Low)**: `validate` という動詞を使用しているメソッド群において、戻り値の契約が不統一である（`validate_parent` と `_validate_level` は `(bool, str)` を返すが、`_validate_purpose_quality` は `Optional[str]` を返す）。
- **抽象度の不統一 (Low)**: ディレクトリツリー全体を巡回・管理する `check` メソッド（Manager層）と、単一の文字列操作を行う `_parse_level` や `_validate_level`（Helper層）が同一クラス内のフラットなメソッドとして並列に扱われている。

## 重大度
Medium
