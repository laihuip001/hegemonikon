# 比喩一貫性の詩人 レビュー

## 対象ファイル
`hermeneus/src/dispatch.py`

## 判定
発言（要改善）

## 発見事項
- **比喩混在 (Medium)**: モジュール名 `hermeneus`（解釈・伝令の神）は「解釈・理解」の比喩世界を示唆しますが、ファイル名および関数名 `dispatch`（配送・発送）は機械的・物流的な比喩であり、世界観が混在しています。また、実際の処理は AST のパース・構造表示・計画提示（解釈プロセス）であり、実行を委譲する「Dispatch（発送）」という行為とは乖離しています。`interpreter` や `planner`、あるいは `analyze` といった用語の方が `hermeneus` の解釈的性質に合致します。
- **動詞の活用不統一 (Low)**: `extract_workflows`（採掘・抽出のメタファー）と `format_ast_tree`（整形・事務のメタファー）で、データ処理に対する姿勢が不揃いです。
- **命名規則不統一 (Low)**: 同階層の `cli.py` が `compile`, `execute` 等の標準的な操作動詞を採用しているのに対し、`dispatch` は浮いており、抽象度が揃っていません。

## 重大度
Medium
