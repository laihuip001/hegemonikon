# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/dendron/checker.py`

## 判定
発言（要改善）

## 発見事項
- **比喩の不統一 (Medium)**: クラス名 `DendronChecker` において、`Dendron`（樹木/有機的）と `Checker`（検査機/工学的）の比喩が混在しています。`Dendron` というドメイン名を採用するなら、その管理者や観察者を指す言葉（例: `Arborist` や `Overseer`）が相応しいでしょう。逆に工学的な検証を強調するなら `ProofValidator` などにすべきです。現状は「工場と庭園の混在」状態です。
- **命名規則の不統一 (Low)**: `check_file_proof`（対象物指定）に対し、`check_functions_in_file`（対象物+場所指定）や `check_variables_in_file` と命名規則が揺れています。引数で `path` を取っているため、`check_functions`, `check_variables` で統一するか、あるいは `scan_file_for_functions` のように動作を明示すべきです。
- **抽象度の不統一 (Low)**: `validate_parent` は `check_file_proof` の内部で使用されるヘルパーメソッドですが、`check` や `check_file_proof` と同列の public メソッドとして公開されています。これは「管理者（check）」と「作業員（validate_parent）」が同列に並んでいる状態です。`_validate_parent` としてカプセル化すべきです。
- **動詞の不統一 (Low)**: 検証動作として `check`（確認する）と `validate`（妥当性を保証する）が混在しています。特に `check_file_proof` は `FileProof` オブジェクトを *生成* するメソッドであり、単なる `check` 以上のことを行っています（`generate_proof` や `assess_file` が実態に近い）。

## 重大度
Medium
