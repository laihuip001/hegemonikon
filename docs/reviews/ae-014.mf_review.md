# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/symploke/boot_integration.py`

## 判定
発言（要改善）

## 発見事項
- **動詞の不統一 (Low)**:
  - 公開APIである `get_boot_context` が一般的な `get` を使用しているのに対し、内部ヘルパーや依存モジュール（`load_handoffs` 等）は具体的な `load` で統一されている。この不一致は予測可能性を低下させる。
  - `extract_dispatch_info` が `extract` を使用しており、データ取得系の動詞体系（load/get）と混在している。

- **抽象度の不統一 (Medium)**:
  - システム全体の起動を司る `get_boot_context`（Manager/Orchestrator）と、特定タスクのヘルパーである `extract_dispatch_info`（Worker/Helper）が同列のパブリック関数として定義されている。
  - `extract_dispatch_info` は「テスト容易性のために分離」された内部ロジックであり、モジュールの主要インターフェースとして公開すべきではない（`_extract_dispatch_info` 推奨）。

- **命名規則の不統一 (Low)**:
  - `postcheck_boot_report` の `postcheck` は造語的な響きがあり、他の標準的な動詞（load, print, generate）と馴染まない。`validate` や `verify` のような標準的な用語が望ましい。

## 重大度
Medium
