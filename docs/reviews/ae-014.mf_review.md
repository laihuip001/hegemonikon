# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/fep/tests/test_attractor_advisor.py`

## 判定
発言（要改善）

## 発見事項
- **比喩の混在 (Medium):**
    - 「物理学/力学系」（Attractor, Oscillation, Convergence, Series, Basin, Energy）と「医療/コンサルティング」（Diagnosis, Advisor, Recommend, Suggest, Client）の比喩体系が混在している。
    - 特に `attractor.diagnose()` は、物理的実体（アトラクタ）が人間的・専門的行為（診断）を行う不自然な擬人化である。「観測者」や「分析器」が診断を行う形に責務を分離すべきである（例: `AttractorAnalyzer.analyze(attractor)`）。
    - 解決策案: `Attractor` は `check_stability()` や `measure_convergence()` を行い、`Advisor` が `diagnose()` する形に整理する。
- **命名規則の不統一 (Low):**
    - `SuggestResult` (動詞+名詞) が、`Recommendation` や `OscillationDiagnosis` (名詞) と並列で定義されている。`SuggestionResult` または `DiagnosisResult` に統一すべきである。
    - 解決策案: `SuggestResult` を `SuggestionResult` にリネームする。
- **動詞の不統一 (Low):**
    - `suggest`, `recommend`, `diagnose` が類似の文脈（結果の提示）で使用されているが、`Attractor` は `suggest` し、`Advisor` は `recommend` するという使い分けが、抽象度の違いによるものか比喩の違いによるものか不明瞭である。
    - 解決策案: `Attractor` は `output_state()` や `report()` とし、`Advisor` は `recommend()` に統一する。

## 重大度
Medium
