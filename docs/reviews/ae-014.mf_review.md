# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/pks/pks_cli.py`

## 判定
発言（要改善）

## 発見事項
- **(Metaphor Mix)**: `PKSEngine` (Factory/Mechanical), `nuggets` (Mining), `serendipity` (Garden), `Push` (Logistics), `Attractor` (Physics) が混在しています。特に「工場（Engine, Push）」と「鉱山（Nugget, Deepened）」と「庭園（Serendipity）」の対立が顕著です。
- **(Naming Inconsistency)**: クラス名のプレフィックスが不統一です。`PKSEngine`, `PKSBacklinks` に対し、`PushDialog` は `Push` を使用しています。
- **(Verb Inconsistency)**: `search_and_push` という動詞が `cmd_backlinks` (擬似バックリンク表示) で使用されていますが、バックリンクは「プッシュ（押し出し）」されるものではなく「取得・表示」されるべきものです。動詞 `push` の意味が過剰に拡張されています。

## 重大度
Medium

## 修正コード提案
比喩を「鉱山（Mining）」または「知識の庭園（Garden）」に寄せるか、あるいは「サイバネティクス（Feedback/Control）」に統一することを提案します。現状の `Engine` (Factory) は汎用的すぎます。

### 提案1: クラス名の統一 (Naming)
`PushDialog` -> `PKSDialog` または `KnowledgeDialog`

### 提案2: 動詞の適正化 (Verb)
`search_and_push` -> `search_and_present` または `retrieve_knowledge`
(バックリンク表示のコンテキストにおいて `push` は不適切)

### 提案3: 比喩の統一 (Metaphor)
- `PKSEngine` -> `PKSExcavator` (Mining) / `PKSGardener` (Garden)
- `nuggets` -> `resources` (Factory) / `fruits` (Garden) -> 現状 `nugget` がドメイン用語として定着しているなら `Excavator` への寄せが妥当。
