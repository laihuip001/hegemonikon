# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/anamnesis/export_chats.py`

## 判定
発言（要改善）

## 発見事項
- **比喩の混在 (Medium)**
  - `mekhane` (機械/工場) ドメインにおいて、データの取得に `collect` (採集/庭園) が使用されている (`scroll_and_collect_messages`)。
  - 同クラス内の `extract_conversation_list` や `AntigravityChatExporter` (輸出業者) と合わせ、工場のメタファーである `extract` (抽出/採掘) に統一すべきである。
  - 提案: `scroll_and_collect_messages` -> `scroll_and_extract_messages`

- **動詞の不統一 (Low)**
  - 保存メソッドにおいて、対象の指定方法が揺れている。
  - `save_individual` (形容詞: 個別の) vs `save_single_chat` (動詞句: 単一のチャットを保存)。
  - `save_individual` は全件を個別に保存する処理であり、`save_split_archives` や `save_each_chat` など、動作を明確にする動詞句が望ましい。

- **抽象度の不統一 (Low)**
  - `AntigravityChatExporter` クラスが、高レベルの進行管理 (`export_all`) と低レベルの DOM 操作 (`scroll_and_collect_messages` 内の生 JS 文字列) を同列に扱っている。
  - `Exporter` (輸出業者) が自ら `scroll` (現場作業) を行うのは、マネージャーと作業員の混同である。

## 重大度
Medium
