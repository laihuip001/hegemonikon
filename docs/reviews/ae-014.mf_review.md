# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/pks/feedback.py`

## 判定
発言（要改善）

## 発見事項
- **Collector (収集者) と Regulator (調整者) の役割混在 (Medium)**
  - クラス名は `FeedbackCollector`（収集者）ですが、メソッド `adjust_threshold`（閾値調整）という能動的な操作（制御・機械のメタファー）を含んでいます。収集者は「集める」までが責務であり、「どう使うか（調整）」は別の管理者（Manager/Controller）の役割であるべきです。
  - 提案: `adjust_threshold` を `calculate_threshold` （算出）という計算的な名前に変更し、収集者の役割（Observation）に留める。

- **Mining (採掘) と Cybernetics (制御) のメタファー混在 (Medium)**
  - データクラス `PushFeedback` 内のフィールド `nugget_title` は「金塊（Nugget）」という採掘のメタファーですが、クラス全体は `Feedback`、`Threshold`、`Reaction` というサイバネティクス（制御理論）の用語で構成されています。
  - 提案: `nugget_title` を `item_title` に変更し、ドメインを中立化して制御用語との衝突を避ける。

- **動詞の対義語不統一 (Low)**
  - 保存メソッドが `persist` であるのに対し、読み込みが `_load` になっています。`persist` の対義語は `restore`、`load` の対義語は `save` または `dump` が一般的です。
  - 提案: `persist` を `save` に変更し、`_load` と対にする。
  - 取得メソッド `get_stats` は弱いため、計算・報告のニュアンスを持つ `report_stats` に変更する。

## 重大度
Medium
