# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/dendron/models.py`

## 判定
発言（要改善）

## 発見事項
- (Medium) `ProofLevel` ドキュメント内の「マトリクス構造」: `Dendron` (樹木) モジュールにおいて、階層構造を「マトリクス」(数学/グリッド) と表現しており、比喩が混在している。「階層構造」や「レイヤー」の方が自然。
- (Low) `CheckResult` の命名不統一: `FileProof`, `FunctionProof` 等の `Proof` 接尾辞を持つクラス群を集約しているが、ここだけ `Check` (動作/検査) と `Result` (結果) になっている。`ProofReport` や `ProofSummary` の方が「証明」の比喩で統一される。
- (Low) `ProofStatus` 内の `ORPHAN`: `EXEMPT`, `INVALID`, `MISSING` (論理/法的) と並んで `ORPHAN` (生物/家族) が使用されており、比喩体系が混在している。論理的には `UNREFERENCED` や `ISOLATED` が適切。

Medium
