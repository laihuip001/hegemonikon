# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/pks/push_dialog.py`

## 判定
発言

## 発見事項
- [Medium] 比喩の混在: モジュール内で「鉱山 (Nugget)」「サイバネティクス (Push)」「対話 (Dialog, why)」「空間 (deeper)」の比喩が混在しており、メンタルモデルが不安定である。特に `KnowledgeNugget` (鉱山) と `PushDialog` (対話) の不整合が顕著。
- [Low] 命名規則の不統一: メソッド名 `why`, `deeper`, `related` が動詞句 (verb_noun) ではなく、形容詞や副詞、あるいは DSL コマンドのような命名になっている。Python の標準的な `get_reason`, `explore_topic`, `find_related` といった形式から逸脱している。
- [Low] 抽象度の不統一: 同一クラス内で、高レベルなプロンプト構築 (View層) と低レベルなインデックス検索 (Data層, `GnosisIndex` の直接利用) が混在しており、関心の分離が不十分である。

## 重大度
Medium
