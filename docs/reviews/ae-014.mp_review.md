# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/pks/pks_cli.py`

## 判定
発言（要改善）

## 発見事項
- **比喩混在 (Medium)**
    - **Factory vs Garden vs Mining vs Consultant**: 以下の比喩体系が無秩序に混在しており、ドメインの世界観が統一されていません。
        - **Factory (機械)**: `PKSEngine`, `cmd_push` (production implied), `generate`
        - **Garden (庭園/有機)**: `deepened` (feedback reaction), `enable_serendipity`
        - **Mining (採掘)**: `nuggets` (knowledge units), `search_and_push` (extraction), `proactive_push`
        - **Consultant (諮問)**: `suggest`, `dialog`, `report`
    - 特に `nuggets` (採掘) と `engine` (機械) の組み合わせや、Feedbackにおける `used` (道具) と `deepened` (成長) の混在が顕著です。

- **抽象度の不統一 (Medium)**
    - **Manager vs Helper**: CLI (`cmd_feedback`) が `engine` (Manager層) をバイパスして、内部実装である `_feedback` (Private Helper) に直接アクセスしています (Line 167)。これは `Engine` を単なるコンテナとして扱っており、抽象化の階層が崩れています。

- **命名規則不統一 (Low)**
    - **CLI vs Internal API**: コマンド名と内部メソッド名の対応が非対称です。
        - `cmd_infer` (動詞) -> `auto_context_from_input` (名詞句/抽出)
        - `cmd_suggest` -> `suggest_questions` (一貫しているが、処理として `search_and_push` も行っている)

- **動詞不統一 (Low)**
    - **Search vs Push**: 知識の取得操作に対して、文脈によって `search` (能動的探索), `push` (受動的供給), `get` (取得) が使い分けられていますが、CLIの視点（ユーザーへのPush）と内部実装の視点（DBからのSearch）が混ざっています。

## 重大度
Medium
