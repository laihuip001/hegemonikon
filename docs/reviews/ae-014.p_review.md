# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/fep/tests/test_attractor_advisor.py`

## 判定
発言（要改善）

## 発見事項
- **[Medium] 比喩の混在 (Nature vs Factory/Medical/Consulting)**:
    - `SeriesAttractor` (Physics/Nature) のコンテキストで `_build_diagnosis` (Factory: "build") が使用されています。"Derive", "Infer", "Formulate" など、自然/科学的な動詞が好ましいです。
    - `diagnose` (Medical), `attractor/oscillation` (Physics), `advisor/recommend` (Consulting) と、複数の比喩体系が混在しており、ドメインの世界観が統一されていません。
- **[Low] 命名規則の不統一**:
    - `SuggestResult` (Verb+Noun) と `Recommendation` (Noun) が混在しています。`Suggestion` と `Recommendation`、あるいは `SuggestResult` と `RecommendResult` のように統一すべきです。
- **[Low] 動詞の活用不統一**:
    - `suggest` (Attractor) と `recommend` (Advisor) という類義語が、異なるクラスの主要動作として使い分けられていますが、その境界が曖昧です。
    - `diagnose` メソッドの結果に `interpretation` プロパティがあり、型は `OscillationDiagnosis` です。`diagnosis` プロパティにするか、メソッドを `interpret` にするなど、動詞と名詞の対応を整理すべきです。
