# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/pks/pks_cli.py`

## 判定
発言（要改善）

## 発見事項
- **比喩混在 (Medium)**:
  - 鉱業 (`nuggets`): 「知識の塊」として使用。
  - 工業/機械 (`PKSEngine`): 「処理エンジン」として使用。本来エンジンは動力を生むもので、採掘や精製を行うものではない。
  - 物流/サイバネティクス (`Push`, `Feedback`): 「プッシュ配信」「フィードバック制御」。
  - 物理学/力学系 (`Attractor`): 「文脈推論」に使用。
  - 神秘主義/哲学 (`Gnōsis`): 「知識検索」に使用。
  - これらが同一モジュール内で混在しており、「一つのドメインには一つの比喩世界」の原則に反する。特に「エンジンがナゲットをプッシュする」という表現は不自然。

- **命名規則不統一 (Low)**:
  - `PushDialog` のメソッド (`why`, `deeper`, `related`) は形容詞や副詞的で会話調だが、`PKSEngine` のメソッド (`search_and_push`, `proactive_push`) は動詞句で機能的。クラス間で命名のトーンが統一されていない。

- **動詞不統一 (Low)**:
  - `get_stats` (明示的な取得) と `auto_context_from_handoff` (暗黙的な取得/抽出) が混在している。統一感に欠ける。

## 重大度
Medium
