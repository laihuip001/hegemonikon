# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/anamnesis/gnosis_chat.py`

## 判定
発言（要改善）

## 発見事項
- **比喩体系の混在 (Medium)**:
    - **工場 (Factory)**: `_build_context` (構築), `_generate` (生成), `_chunk_text` (加工), `pipeline` 的な処理フロー。
    - **図書館 (Library)**: `index_knowledge` (索引), `_retrieve` (検索/取得), `ConversationHistory` (記録)。
    - **探検/自然 (Nature/Garden)**: `discover_knowledge_files` (発見), `root` (根/起点), `oikos` (生態系)。
    - データの取り扱いにおいて、「構築する(Build)」のか「発見する(Discover)」のか「検索する(Retrieve)」のか、世界観が統一されていない。

- **動詞の不統一 (Low)**:
    - データの取得操作に対し、`retrieve` (`_retrieve`), `fetch` (`fetch_k`), `search` (`search_knowledge`), `discover` (`discover_knowledge_files`), `get` (`_get_embedder`) が混在している。
    - 特に `retrieve` (メソッド名) と `fetch` (変数名) の揺らぎは認知負荷を高める。

- **抽象度の不統一 (Low)**:
    - **Manager/Engine**: `GnosisChat`, `KnowledgeIndexer` (実質的な機能集合)
    - **Helper/Component**: `Reranker`, `ConversationHistory`
    - これらがフラットに並列定義されており、モジュールの主役と脇役の階層構造が不明瞭。

## 重大度
Medium
