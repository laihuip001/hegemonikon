# 比喩一貫性の詩人 レビュー

## 対象ファイル
`hermeneus/src/dispatch.py`

## 判定
発言（要改善）

## 発見事項
- **比喩の混在** (Medium): モジュール内で「工場/物流」（Dispatch, Workflow, Plan）と「庭園/自然」（Tree, Oscillation, Fusion, Colimit）の比喩が混在している。特に `extract_workflows` (工場的) と `format_ast_tree` (庭園的) が同居している点が不調和を生んでいる。
- **抽象度の不統一** (Low): `dispatch` (Manager/Entry point) と `format_ast_tree`/`extract_workflows` (Helpers) が同一階層に定義されており、抽象度が揃っていない。
- **命名規則の不統一** (Low): 関連する AST クラスにおいて、`Workflow` (ビジネス/IT) と `Oscillation`/`Fusion` (物理/自然) という異なるドメインの命名が混在している。
