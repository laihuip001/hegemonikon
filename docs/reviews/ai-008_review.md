# 自己矛盾検出者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **並行性設定の矛盾**: `__init__` で受け取った `max_concurrent` 引数が、`__aenter__` 内の `aiohttp.TCPConnector` の初期化で無視されている。コネクタの上限がクラス定数 `MAX_CONCURRENT` (60) に固定されているため、ユーザーがより高い並行数を指定しても物理的な接続数で制限される。
- **非同期設計との矛盾**: `synedrion_review` メソッドがバッチを順次実行（`await batch_execute(...)`）しており、前のバッチが完了するまで次のバッチが開始されない。これは `asyncio` のノンブロッキングな特性と、既に実装されているグローバルセマフォ (`_global_semaphore`) によるレート制限の利点を殺しており、スループットを不必要に低下させている。
- **成功判定の論理矛盾**: `JulesResult.is_success` プロパティが、例外が発生していない限り `True` を返すが、これにはセッション状態が `FAILED` や `CANCELLED` の場合も含まれる。「成功」という言葉の意味と実際の動作が矛盾している。
- **「沈黙」検出の無効性**: `synedrion_review` 内の `SILENCE` チェックが `str(r.session)` を対象としているが、`JulesSession` オブジェクトの文字列表現には LLM の出力内容が含まれないため、このチェックは常に偽となる（機能していない）。
- **リソース管理の矛盾**: `_session` プロパティが、アクセスされるたびに新しい `ClientSession` を作成するが、クリーンアップの仕組みがない。`__aenter__` / `_request` で採用されているリソース管理パターンと矛盾しており、誤用によるリソースリークの温床となっている。
