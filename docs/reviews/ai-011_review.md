# 過剰最適化検出者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **ビジネスロジックの過剰な結合**: `synedrion_review` メソッド（433-514行目）は、汎用的なAPIクライアントに「Synedrion v2.1」や「480の直交視点」といった特定のビジネスロジックを埋め込んでいます。これはクライアントの責務を超えており、再利用性を低下させる過剰な最適化（特定のワークフローへの特化）です。
- **ハードコードされた並行性制限**: `MAX_CONCURRENT = 60` が「Ultra plan」を前提としてハードコードされています。利用プランや環境に応じた設定可能性が排除されており、柔軟性を欠いています。
- **複雑なセマフォ管理**: `_global_semaphore` や `batch_execute` における `use_global_semaphore` フラグによる内部的なレート制限は、クライアントの状態管理を複雑にしています。レート制限は呼び出し元や専用のミドルウェアで管理する方がシンプルです。
- **動的インポートによる結合隠蔽**: `synedrion_review` 内での `mekhane.ergasterion.synedrion` の動的インポートは、設計上の結合度が高いことを示唆しており、循環参照を回避するための対症療法的な複雑さです。
