# 再試行ロジック評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- `with_retry` デコレータによるエクスポネンシャルバックオフ実装において、ジッター（ランダムな揺らぎ）が欠如しています。`MAX_CONCURRENT=60` の高並列環境下では、再試行タイミングが同期し、サーバーへの負荷が集中する「Thundering Herd」問題を引き起こすリスクがあります。
- `create_session` メソッド（POSTリクエスト）に対して `aiohttp.ClientError` での無条件な再試行が適用されています。Idempotency-Key（冪等性キー）の仕組みが見当たらないため、レスポンス受信時のタイムアウト等で再試行が発生した場合、セッションが重複して作成される可能性があります。
- `poll_session` 内で独自のバックオフ・リトライロジックを実装していますが、内部で呼び出している `get_session` も `with_retry` で装飾されています。これにより `RateLimitError` 発生時の挙動が二重の再試行構造となり、意図しない待機時間の増大や複雑なエラー伝播を引き起こす可能性があります。
