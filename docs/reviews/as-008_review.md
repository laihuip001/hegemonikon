# コネクションプール評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **コンテキストマネージャ未使用時のプール不在**: `JulesClient` を `async with` ブロック内で使用しない場合、`_request` メソッドはリクエストごとに新しい `aiohttp.ClientSession` を作成し、直後に破棄しています。これによりコネクションプーリングが機能しません。
- **ポーリングとバッチ処理における非効率**: `poll_session` や `batch_execute` は多数のリクエストを行いますが、永続的なセッションがない場合、TCP接続の確立・切断が頻発します。これはレイテンシの増加と、システムレベルでのポート枯渇（Port Exhaustion）のリスクを招きます。
- **構成の適用範囲**: `__aenter__` 内での `TCPConnector` の設定（`limit`, `keepalive_timeout`）自体は適切ですが、これはコンテキストマネージャ使用時にしか適用されません。クラスの設計として、永続的なセッションの利用をより強く推奨するか、内部で管理する仕組みが必要です。

## 重大度
- High
