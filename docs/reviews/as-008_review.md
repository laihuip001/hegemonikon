# コネクションプール評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- `_request` メソッド内で、コンテキストマネージャー（`async with`）が使用されていない場合、リクエストごとに新しい `aiohttp.ClientSession` が作成され、即座に閉じられています。これにより、TCP接続の再利用（Keep-Alive）が行われず、パフォーマンスの低下やリソースの浪費（ポート枯渇など）を引き起こす可能性があります。
- `batch_execute` メソッドで多数のタスクを実行する際、コンテキストマネージャーを使用していないと、各タスクが独自のセッションを作成・破棄することになり、接続プールの利点が失われます。
- ドキュメンテーションの `Usage` 例では `client = JulesClient(...)` といったコンテキストマネージャーを使用しない方法が示されており、このままでは非効率な接続管理を助長する可能性があります。

## 重大度
- High
