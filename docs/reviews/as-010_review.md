# シグナルハンドリング評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **シグナルハンドラの不在**: `signal` モジュールや `loop.add_signal_handler` を用いた明示的な中断シグナル（SIGINT, SIGTERM）の処理が実装されていません。
- **リモートセッションの孤立化リスク**: `synedrion_review` などの長時間実行プロセスが中断された際、実行中のJules APIセッションに対してキャンセルを送信するクリーンアップ処理（Graceful Shutdown）が存在しません。これにより、クライアントが停止してもサーバー側で処理が継続し、リソース（およびコスト）が無駄になる可能性があります。
- **構造化された並行処理の欠如**: `asyncio.gather` を使用しており、Python 3.11+ の `asyncio.TaskGroup` のような、例外発生時やキャンセル時に子タスクを安全に終了させる仕組みが明示的ではありません。

## 重大度
- Medium
