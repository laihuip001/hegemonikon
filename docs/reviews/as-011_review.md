# 非同期イテレータ評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- `poll_session` メソッドは `while` ループを使用してポーリングを行っており、完了または失敗までブロックします。これにより、呼び出し元は中間状態（例: `PLANNING` -> `IMPLEMENTING`）を観測できません。非同期ジェネレータ（`async generator`）を使用して状態遷移を `yield` すべきです。
- `batch_execute` メソッドは `asyncio.gather` を使用して全タスクの完了を待ちます。これにより、一部のタスクが早く完了しても、最も遅いタスクが完了するまで結果を取得できません。`asyncio.as_completed` または非同期ジェネレータを使用して、完了した順に結果を `yield` するのが望ましいです。
- `synedrion_review` メソッドは全ての結果をメモリ上のリスト `all_results` に蓄積してから返します。大量のパースペクティブを処理する場合、メモリ効率が悪く、呼び出し元の応答性も低下します。非同期イテレータとして実装し、結果を逐次 `yield` すべきです。
- ファイル全体で `async for` の使用が見られず、ストリーミング処理の利点が活かされていません。
