# 非同期イテレータ評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **`poll_session` の状態監視が不透明**: 現在 `poll_session` は `while` ループを使用し、最終的な完了状態のみを返却しています。中間状態（`PLANNING`, `IMPLEMENTING` など）の遷移を呼び出し元が把握できません。`async generator` (`yield`) を使用して状態遷移をストリーム配信する設計が望ましいです。
- **`batch_execute` の非同期イテレーション欠如**: `asyncio.gather` を使用しているため、バッチ内の全てのタスクが完了するまで結果が得られません。`asyncio.as_completed` 等と組み合わせた非同期ジェネレータとして実装することで、完了したタスクから順次処理（ストリーミング処理）が可能になり、応答性が向上します。
- **`synedrion_review` のバッチ処理**: 固定サイズのバッチごとに `await` しており、バッチ内の最も遅いタスクに律速されます。非同期イテレータを使用すれば、全タスクを通してスムーズな並行処理と進捗報告が可能になります。

## 重大度
- Medium
