# メンタルモデル穴検出者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **内部エンドポイントへの暗黙的な依存**: `BASE_URL` が `https://jules.googleapis.com/v1alpha` にハードコードされており、特定のネットワーク環境（Google内部/VPN等）または非公開APIへのアクセス権限を前提としているが、これに関するドキュメントがない。
- **入力フォーマットの暗黙知**: `source` パラメータが `sources/github/owner/repo` という特定の形式を要求しているように見えるが、バリデーションや形式の定義がなく、ユーザーが推測する必要がある。
- **不透明な設定オプション**: `create_session` メソッドの `automation_mode` 引数はデフォルトで `"AUTO_CREATE_PR"` だが、他にどのような値が許容されるか（例: `"AUTO_COMMIT"`, `"DRY_RUN"` 等）がコードからもドキュメントからも不明。
- **ハードコードされたプラン制限**: `MAX_CONCURRENT = 60` が "Ultra plan limit" とコメントされているが、ユーザーがどのプランを利用しているか確認するロジックはなく、全てのユーザーに対してこの制限が適用される前提となっている。
- **隠れたモジュール結合**: `synedrion_review` メソッド内で `mekhane.ergasterion.synedrion` を動的にインポートしており、クライアントライブラリが特定のリポジトリ構造に強く依存している。これがライブラリとして配布される場合、外部依存関係として明示されていないと動作しない。
- **出力構造の脆弱な仮定**: `get_session` メソッドにおいて、`outputs[0]` にプルリクエスト情報が含まれると仮定しているが、出力が空の場合や複数ある場合の考慮が不足している（ただし `if outputs:` チェックはある）。
