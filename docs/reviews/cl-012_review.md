# コンテキストスイッチ検出者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **Synedrionモジュールへの強い結合と隠蔽された依存**: `synedrion_review` メソッド内で `mekhane.ergasterion.synedrion` を動的にインポートしており、トップレベルの依存関係として明示されていません。このメソッドのロジック（特にパースペクティブのフィルタリングやプロンプト生成）を理解するには、`mekhane/ergasterion/synedrion/prompt_generator.py` や `perspectives.yaml` を参照する必要があり、頻繁なコンテキストスイッチが発生します。
- **暗黙的なデータコントラクト**: `Perspective` オブジェクトの属性（`id`, `name`, `theorem`, `domain_id`, `axis_id`）にアクセスしていますが、これらの型定義やドキュメントはこのファイル内に存在しません。開発者は `ergasterion` パッケージの内部実装を確認しないと利用可能な属性を把握できません。
- **内部CLIによる責務の混合**: ファイル末尾に `main` 関数と `argparse` を用いたCLIロジックが含まれています。ライブラリとしての機能と、テスト/実行用のスクリプト機能が混在しており、本来のCLIエントリポイントやテストコードとの役割分担が不明瞭です。
- **構成値の断片化**: `MAX_CONCURRENT = 60` や `BASE_URL` などがハードコードされており、これらの値の根拠や変更を行うために、外部の設定ファイルではなくこのソースコード自体を確認・修正する必要があります。

## 重大度
- High
