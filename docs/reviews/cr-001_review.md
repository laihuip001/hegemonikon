# LGTM拒否者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 判定
発言（要改善）

## 発見事項
- **重大なパフォーマンス欠陥 (High)**: `_request` メソッド内で、コンテキストマネージャを使用していない場合、リクエストごとに `aiohttp.ClientSession` を新規作成・破棄しています。これは `aiohttp` の推奨するコネクションプーリングの利点を完全に殺しており、特にバッチ処理時のパフォーマンスを著しく低下させます。
- **幻覚の疑いがあるAPIエンドポイント (Critical)**: `https://jules.googleapis.com/v1alpha` というエンドポイントがハードコードされていますが、これは既知の公開されたGoogle APIではありません。実在しないAPIへのリクエストは必ず失敗します。
- **不要なコメントの残存 (Low)**: 複数の箇所（L280, L297, L437, L501）に `# NOTE: Removed self-assignment` という自動生成ツール由来と思われるコメントが残置されています。これらはコードの可読性を損なうノイズであり、削除すべきです。
- **一貫性のないリトライロジック (Medium)**: `create_session` 等では `@with_retry` デコレータを使用している一方、`poll_session` では独自のリトライ・バックオフロジックを実装しています。ロジックが分散しており、保守性を低下させています。
- **CLIテストの実装不備 (Low)**: `main` 関数内のCLIテストで `print` 文が多用されていますが、プロダクションコードの一部として配置されている以上、適切に `logging` モジュールを使用すべきです。

## 重大度
Critical
