# コラボレーション障壁検出者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
1. **ビジネスロジックの混入 (`synedrion_review`)**:
   - `synedrion_review` メソッドは特定のレビュー手法（Synedrion v2.1）に強く依存しており、汎用的なAPIクライアントとしての責務を超えています。
   - `mekhane.ergasterion.synedrion` への依存は、このクライアントを単体で利用したい開発者にとって不要な結合を生み出し、認知負荷を高めます。
   - **改善案**: このメソッドを `JulesReviewService` や別のオーケストレーション層に移動し、`JulesClient` は純粋なAPI通信に専念させるべきです。

2. **ハードコードされた設定値**:
   - `BASE_URL`, `MAX_CONCURRENT` (60), `POLL_INTERVAL` などが定数としてハードコードされています。
   - 開発環境や異なる契約プランを持つメンバーが、コードを変更せずにこれらの値を調整する方法がコンストラクタで一部提供されていますが、`BASE_URL` は変更できません。
   - **改善案**: 全ての設定値をコンストラクタ引数または環境変数から注入可能にし、デフォルト値を定数として持つ形式に変更することで、テストや異なる環境での再利用性を高めるべきです。

3. **コンテキスト依存のコメント ("Insider References")**:
   - "cl-003 fix", "th-003 fix", "ai-009" といった、特定のレビューIDへの参照が散見されます。
   - これらの背景を知らない新規参画者にとっては意味不明なノイズとなり、「文脈を知っている人しか触れない」という心理的障壁を作ります。
   - **改善案**: レビューIDだけでなく、具体的に「何が問題でどう直したか」を簡潔に記述するか、Gitのコミットログに委ねてコードからは削除することを推奨します。
