# 非同期コラボ評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **部族的な知識（Tribal Knowledge）への依存**: コメント内で `cl-003`, `th-003`, `ai-004` といったレビューIDが文脈なしに参照されています（例: `See cl-003 review`, `th-001 fix`）。これらは共有されたドキュメントへのリンクや要約がない限り、新規または外部のリモートメンバーにとって意味不明な「暗号」となり、非同期での自律的な理解を妨げます。
- **排他的な専門用語（Jargon）**: "Hegemonikón", "Symplokē", "Synedrion", "PerspectiveMatrix" といったプロジェクト固有の用語が、定義や用語集への参照なしに使用されています。これにより、チームの「内部」と「外部」の知識格差が広がり、新規参画者のオンボーディングコストが高まります。
- **隠蔽された依存関係**: `synedrion_review` メソッド内で `mekhane.ergasterion.synedrion` を動的にインポートしており、静的な解析や一見しての依存関係の把握を困難にしています。これはコードの理解可能性（Readability）を下げ、レビューの認知負荷を高めます。
- **ビジネスロジックとクライアントの結合**: `MAX_CONCURRENT = 60` ("Ultra plan limit") や `https://jules.googleapis.com/v1alpha` のハードコーディングは、クライアントライブラリに特定のビジネス契約や環境を強制しています。これにより、異なる環境（開発、ステージング、別プラン）でのテストや利用が困難になり、チーム間の協力を阻害します。
