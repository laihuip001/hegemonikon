# レビュー疲労検出者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **順次バッチ処理による負荷の偏り**: `synedrion_review` メソッドにおいて、`batch_execute` をループ内で `await` して呼び出しているため、現在のバッチ内の全てのタスクが完了するまで次のバッチが開始されません。これにより、バッチ内の最も遅いタスクが完了するまでの間、並行実行スロットがアイドル状態となり、リソース使用効率が低下します。
- **バースト的な負荷発生**: バッチの切り替わり時に一斉にリクエストが発生し、その後徐々に減衰するというバースト的な負荷パターン（Stop-and-Wait）が形成されており、安定したスループットが得られません。これは下流システムや監視者に対して不均一な「疲労（負荷）」を与えます。
- **スライディングウィンドウの欠如**: `asyncio.Semaphore` を使用したグローバルな並行数制御はあるものの、タスク投入自体がバッチ単位で区切られているため、常に最大並行数 (`MAX_CONCURRENT`) を維持するスライディングウィンドウ方式になっていません。
