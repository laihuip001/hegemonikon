# レビュー疲労検出者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **順次バッチ処理による負荷バイアス (Load Bias)**: `synedrion_review` メソッドにおいて、パースペクティブ（視点）のリストが固定順序で、かつ `MAX_CONCURRENT` (60) 単位の順次バッチとして処理されています。もしパースペクティブがドメイン順などに並んでいる場合、レビュープロセスの中断や、ログ監視者の疲労（Review Fatigue）により、リスト後方のドメイン（例：ユーザビリティやエッジケースなど）が体系的に見過ごされるリスクがあります。
- **アラート疲労 (Alert Fatigue)**: `with_retry` および `poll_session` におけるレート制限やリトライのログが `WARNING` レベルで出力されます。大規模な並列実行時にレート制限にかかると、大量の警告ログが発生し、重要なエラーメッセージが埋もれる可能性があります。
- **認知的負荷 (Cognitive Load)**: 低レイヤーのHTTP通信制御（リトライ、セッション管理）と、高レイヤーのビジネスロジック（Synedrionレビューのオーケストレーション）が同一クラスに混在しており、単一責任の原則（SRP）に違反しています。これによりコードの可読性が低下し、レビュアーの負担が増大しています。

## 重大度
