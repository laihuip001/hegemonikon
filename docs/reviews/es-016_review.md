# レビュー疲労検出者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **負荷バイアス (Load Bias)**: `synedrion_review` メソッド（L591-667）において、パースペクティブ（視点）のバッチ処理が `for` ループによる逐次実行 (`await self.batch_execute(batch_tasks)`) として実装されています。これにより、リストの後方に位置するパースペクティブは実行開始が大幅に遅れ、システム全体のタイムアウトや中断が発生した場合に体系的に無視されるリスクがあります。これはレビューの網羅性に深刻なバイアスをもたらします。
- **アラート疲労 (Alert Fatigue)**: `poll_session`（L427）や `with_retry`（L154）デコレータにおいて、再試行やレート制限（RateLimitError）のたびに `logger.warning` が出力されます。高並行（`MAX_CONCURRENT=60`）環境下では、一時的なネットワーク障害やAPIの揺らぎで短時間に数百行の警告ログが発生し、真に重要なエラーが埋もれる「アラート疲労」を引き起こします。
- **ログの冗長性**: `SessionState.from_string`（L89）や `poll_session`（L456）での `UNKNOWN` ステート検出時にも、即座に警告ログが出力されます。ポーリング間隔（5秒）ごとにこれが発生するため、状態が安定するまでログが汚染され続けます。
