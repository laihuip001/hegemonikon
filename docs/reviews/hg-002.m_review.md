# 予測誤差審問官 レビュー

## 対象ファイル
`mekhane/symploke/boot_integration.py`

## 判定
発言（要改善）

## 発見事項
- **副作用非明示 (High):** `sys.path.insert(0, ...)` がモジュールレベルで実行され、インポート時にグローバルな Python パスを変更している。これはインポート順序やテスト環境への予測不能な影響を与える。
- **副作用非明示 (High):** `get_boot_context` という取得（Query）を示唆する関数内で、`urllib.request.urlopen` を使用して `http://localhost:5678/webhook/session-start` (n8n) に POST リクエストを送信している。これは典型的な Command-Query 分離違反であり、予測誤差が大きい。
- **型予測困難 (Medium):** `get_boot_context`, `extract_dispatch_info`, `_load_projects`, `_load_skills` などの主要関数が、構造が明示されていない汎用的な `dict` を返却している。呼び出し元は実装を読まないとキー（`primary`, `alternatives`, `formatted` 等）を知ることができない。
- **例外の発生が関数シグネチャから不明 (Medium):** `extract_dispatch_info` や `get_boot_context` 内の n8n 呼び出し部分などで `except Exception: pass` により例外が握りつぶされている。失敗時に何が起きるか（デフォルト値が返るのか、単に無視されるのか）がシグネチャからは読み取れず、呼び出し元でのハンドリングが困難。

## 重大度
High
