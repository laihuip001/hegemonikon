# 予測誤差審問官 レビュー

## 対象ファイル
`mekhane/dendron/checker.py`

## 判定
発言（要改善）

## 発見事項
- (High) `check_functions_in_file` / `check_variables_in_file`: `SyntaxError` 等を捕捉し空リストを返却しているため、解析失敗と「関数なし」の区別がつかず、呼び出し元の予測を裏切る挙動となっている。
- (Medium) `_validate_purpose_quality`: 名前は `validate` (検証) だが戻り値が `Optional[str]` (エラーメッセージ)。通常 `validate` は例外送出か Boolean を想起させるため、型予測が困難。
- (Medium) `validate_parent`: 戻り値が `tuple[bool, str]` だが、シグネチャからは `is_valid` なのか `check_result` なのか直感的に予測しにくい。

## 重大度
High
