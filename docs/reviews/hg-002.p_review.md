# 予測誤差審問官 レビュー

## 対象ファイル
`mekhane/dendron/checker.py`

## 判定
発言（要改善）

## 発見事項
- **AST解析失敗時の静かなる無視 (High)**: `check()` メソッドおよび `check_functions_in_file` 内で `SyntaxError` や `UnicodeDecodeError` が発生した際、エラーを記録せずに処理をスキップしています（`tree = None` や `return []`）。これにより、構文エラーのあるファイルが「問題なし」として扱われる（検査対象から漏れる）という、設計者が意図しない「驚き」を引き起こします。
- **読み込みエラー時の偽陽性判定 (Medium)**: `check_dir_proof` メソッドにおいて、`PROOF.md` の読み込みで `OSError` が発生した場合、`ProofStatus.OK` を返しています。読み込み不可能であるにもかかわらず「合格」と判定されるのは予測不能な振る舞いです。
- **検証メソッドへの隠れたI/O副作用 (Medium)**: `validate_parent` という名前は純粋な値の検証を想起させますが、`self.root` が設定されている場合にファイルシステムへのアクセス（`exists()`）を行います。メソッド名から I/O 副作用が予測できません。

## 重大度
High
