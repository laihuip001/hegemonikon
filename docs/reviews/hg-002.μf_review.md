# 予測誤差審問官 レビュー

## 対象ファイル
`mekhane/dendron/models.py`

## 判定
発言（要改善）

## 発見事項
- **`CheckResult.coverage` の意味的範囲 (Medium)**
  - `CheckResult` クラスはファイル、関数、変数の統計情報を含んでいるが、`coverage` プロパティはファイル数のみに基づいて計算されている。名前から「全要素のカバレッジ」と誤認される可能性が高い。
  - **提案**: `file_coverage` にリネームするか、関数・変数のカバレッジを含めた総合的な指標として再定義する。また、各層のカバレッジを個別のプロパティ（`function_coverage`, `variable_coverage`）として提供することを検討すべきである。

- **`CheckResult.is_passing` の判定基準 (Medium)**
  - `is_passing` は CI の合否判定を行う重要なプロパティだが、現在の実装ではファイルの証明有無（`files_missing_proof`, `files_invalid_proof`）のみをチェックしており、関数や変数の不備（例: `functions_missing_purpose`, `short_name_violations`）を無視して `True` を返す。これは「合格」という名前の期待値と乖離している。
  - **提案**: `is_file_passing` にリネームしてスコープを明確にするか、他の層のチェック結果も含めるようにロジックを修正する。もし段階的な導入（Phase 1）のために意図的に除外しているなら、その旨を docstring だけでなくプロパティ名にも反映させる（例: `is_file_check_passing`）ことが望ましい。
