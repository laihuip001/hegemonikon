# 信念状態一貫性評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **セッションライフサイクルの不整合**: `__aenter__` によるコネクションプーリングをサポートしつつ、`_session` プロパティは常に新規セッションを作成し、管理されていない（リソースリークの可能性）。また `_request` はメソッド呼び出しごとに使い捨てセッションを作成するフォールバックを持つが、これらが混在している。
- **並行性制御の競合**: `synedrion_review` メソッドは手動でバッチ分割（チャンク処理）を行っているが、内部で呼び出す `batch_execute` はセマフォによる並行数制御を持っている。この二重制御はスライディングウィンドウの利点を殺し、バッチ境界での待ち時間を発生させている。
- **エラー処理戦略の矛盾**: `create_session`/`get_session` は `with_retry` デコレータで指数バックオフを行うが、`poll_session` は独自にループ内でバックオフを実装している。これにより、`poll_session` 呼び出し時には二重の再試行ループが発生し、意図しない長時間待機になる可能性がある。
- **データ構造と使用法の不整合 (Critical)**: `create_session` メソッド内で `JulesSession` をインスタンス化する際、必須引数である `source` がコメントアウト（`Removed self-assignment`）されており、実行時に `TypeError` が発生する状態になっている。
- **戻り値パターンの不統一**: 単体操作メソッド（`create_session`, `poll_session`）は例外を送出するが、バッチ操作メソッド（`batch_execute`）は `JulesResult` オブジェクトを返す Result パターンを採用している。
- **レイヤー違反**: インフラストラクチャ層（L2）であるはずのクライアント内に、ドメイン層（L1/L3）の知識を要する `synedrion_review` メソッド（「480の直交する視点」など）が実装されている。
- **沈黙判定の論理不整合**: `synedrion_review` における沈黙判定が `str(r.session)` に "SILENCE" が含まれるかどうかに依存しているが、`JulesSession` オブジェクトには出力テキストが含まれていないため、この判定は機能しない可能性が高い。

## 重大度
- Critical
