# 支配二分法評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **外部制約の内部ハードコード (MAX_CONCURRENT)**: `MAX_CONCURRENT = 60` として "Ultra plan limit" とコメントされていますが、これはユーザーの契約状況（外部の事実）をコード内の定数（内部の制御）として固定してしまっています。契約プランが異なるユーザーや、プラン内容の変更に対して脆弱です。これは「我々に委ねられているもの（設定）」と「我々に委ねられていないもの（外部APIの制限）」の混同です。
- **列挙型による外部状態の固定化 (SessionState)**: APIが返しうる状態を `Enum` で定義していますが、APIの進化（外部の変化）により新しい状態が追加された場合、`UNKNOWN` として扱われます。`poll_session` メソッドは `UNKNOWN` が3回続くと例外を投げますが、これが「待機すべき一時的な状態」なのか「終了すべき新しい状態」なのかをクライアント側で勝手に判断（支配）しようとしています。未知のものに対する謙虚さ（Epistemic Humility）はありますが、その扱いが硬直的です。
- **タイムアウトのデフォルト値**: `DEFAULT_TIMEOUT = 300` (5分) は、生成系AIのタスク（外部の事象）がその時間内に終わるという内部的な期待を課しています。タスクの複雑性によっては容易に破られる前提です。
