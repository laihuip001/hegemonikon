# 階層的予測評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **抽象化レベルの混在**: `JulesClient`（インフラストラクチャ層/APIクライアント）が `synedrion_review`（ドメイン層/ビジネスロジック）を実装しており、関心の分離に違反している。クライアントが上位層の `mekhane.ergasterion.synedrion` に依存している（動的インポートによる回避が見られるが、構造的な循環依存のリスクがある）。
- **エラー処理の一貫性の欠如**: 単一操作（`create_session`など）は例外を送出するが、バッチ操作（`batch_execute`）はエラーを `JulesResult` オブジェクトにラップして返す。これにより、呼び出し元は異なるエラー処理戦略を使い分ける必要がある。
- **ポリシーとメカニズムの混同**: `MAX_CONCURRENT = 60` のような具体的なビジネスプランの制約（ポリシー）が、汎用的なクライアント実装（メカニズム）にハードコードされている。
- **依存関係の逆転**: 下位レイヤーであるはずの API クライアントが、上位レイヤーの概念である「Synedrionレビュー」の詳細（パースペクティブ生成など）を知りすぎている。

- High
