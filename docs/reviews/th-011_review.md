# JTB知識評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- `parse_state` 関数が未知の状態を `SessionState.IN_PROGRESS` にマッピングしている点。もし API が新しい終端状態（例: `CANCELLED`, `EXPIRED`）を返した場合、ポーリングがタイムアウトまで無駄に継続するリスクがある。これは「未知の状態は進行中である」という正当化が不十分な信念に基づいている。
- `get_session` メソッドでの PR URL 抽出ロジック (`outputs[0]`) が、出力リストの構造に対して脆弱である可能性。常に最初の要素が PR 情報を含んでいるという仮定に基づいている。
- `MAX_CONCURRENT = 60` がハードコードされており、「Ultra plan」という特定のコンテキストに依存している。環境変数や設定での上書きが考慮されていない。
- `v1alpha` エンドポイントを使用しており、API の仕様変更に対する耐性が低い（これは alpha 版の性質上やむを得ないが、リスクとして認識すべき）。
