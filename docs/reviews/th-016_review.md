# ホメオスタシス評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **リトライロジックのジッター欠如**: `with_retry` および `poll_session` のバックオフ処理にランダムな遅延（ジッター）が含まれていません。高い並行数（`MAX_CONCURRENT=60`）でレート制限に達した場合、全てのリクエストが同時に再試行され、「サンダーバード（thundering herd）」問題を引き起こし、システムの安定性を損なうリスクがあります。
- **キャンセル時の孤立セッション**: `batch_execute` で `asyncio.gather` を使用していますが、クライアント側で処理が中断（キャンセル）された際、サーバー側で実行中のセッションをキャンセルするクリーンアップ処理（Graceful Shutdown）が実装されていません。これによりサーバーリソースが無駄に消費され続け、外部システムとの整合性が保たれない可能性があります。
- **設定のハードコード**: `BASE_URL` や `MAX_CONCURRENT` などの重要設定がクラス内にハードコードされており、`SymplokeConfig` (`mekhane/symploke/config.py`) を利用していません。環境変化への適応性（可塑性）が低く、設定の一元管理という観点で恒常性が欠如しています。

## 重大度
- Medium
