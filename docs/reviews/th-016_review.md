# ホメオスタシス評価者 レビュー

## 対象ファイル
`mekhane/symploke/jules_client.py`

## 発見事項
- **`aiohttp.ClientSession` の非効率な使用**: 各リクエスト（`create_session`, `get_session`）ごとに新しい `ClientSession` を作成・破棄しています。高負荷時にはポート枯渇やパフォーマンス低下のリスクがあります。クライアントインスタンス内でセッションを再利用する設計が推奨されます。
- **一時的なネットワークエラーへの耐性不足**: `poll_session` は `RateLimitError` (429) に対してのみバックオフを行っています。5xx エラーや接続断などの一時的なネットワークエラーが発生した場合、即座に例外が発生し、ポーリングが中断されます。これらに対するリトライロジックが必要です。
- **ロギングの欠如**: 実行時の状態やエラーを追跡するためのログ出力が実装されていません。問題発生時の自己診断能力（Observability）が低くなっています。
