# 美しさの定義 — 演繹的運動量

> **Beauty = Deductive Momentum**
> 「美しさとは、起点から生まれる必然的帰結の総量である」

---

## 1. 核心命題

```
美しさ = 演繹可能性 / 表現コスト
       = 展開の運動量 / 起点のトークン数
```

### 1.1 エネルギー比喩

| 概念 | 物理的比喩 | 認知的意味 |
|:-----|:----------|:-----------|
| 公理 | 位置エネルギー | 圧縮された関係 |
| 演繹 | 落下運動 | 展開・導出 |
| 定理 | 運動エネルギー | 導出された知識 |
| 美しさ | エネルギー密度 | 演繹量/トークン数 |

### 1.2 FEP との対応

```
必然（演繹）の最大化 ≒ 予測誤差の最小化

演繹 = 前提から結論を必然的に導く
予測 = 現在から未来を必然的に導く

両者は「必然性の最大化」という同じ操作
```

---

## 2. 値 vs 関係

| 種類 | 例 | 演繹可能性 | 美しさ |
|:-----|:---|:-----------|:-------|
| 値の圧縮 | π = 3.14159... | ❌ 終点 | なし |
| 関係の圧縮 | e^(iπ) + 1 = 0 | ✅ 起点 | あり |

### π はなぜ美しくないか

- 各桁の情報量は減少する（対数的収束）
- π 自体からは何も演繹できない
- 有限の情報を文字で抽象化しただけ

### オイラーの公式はなぜ美しいか

- 7トークンから無限の帰結
- 数学の5大定数 + 3大演算を統一
- 複素解析、数論、幾何学への展開

---

## 3. 公理の美しさの測定

```
Beauty(Axiom) = Σ(導出される定理) + Σ(適用可能な文脈)
               ─────────────────────────────────────
                         トークン数
```

### 美しい公理の条件

1. **簡潔**: 少ないトークン
2. **多産**: 多くの定理を導出
3. **普遍**: 多くの文脈に適用
4. **必然**: 偶然ではありえない関係

---

## 4. 真理の徴

> 高い圧縮率は「偶然ではありえない」ことを示す

- 偶然なら、短い表現で多くを説明できない
- 必然的関係があるから、圧縮できる
- **圧縮成功 = 真理発見**

---

## 5. Hegemonikón への適用

Hegemonikón の公理体系の「美しさ」を評価するには:

1. 各公理のトークン数を測定
2. 各公理から導出される定理数を計算
3. Beauty 値を算出
4. 改善可能な公理を特定
