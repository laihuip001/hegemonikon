## 抽象: ベイズ更新としての認知行為

全ての認知行為は、FEP のベイズ更新と同型の 3 相構造を持つ:

```
P(θ|x) ∝ P(x|θ) · P(θ)

[前] Prior       →  [核] Likelihood  →  [後] Posterior
文脈・前感情         認知操作            統合・確信・記録
```

**CCL に翻訳すると:**

```
[Prior]      → [Likelihood]  → [Posterior]
/pro, /s-, /kho   各マクロ固有    /pis, /dox-, ~(/h*/k), ~(/noe*/dia)
```

| 相 | FEP | CCL 要素 | 役割 |
|:---|:----|:---------|:-----|
| **Prior** | 事前分布 P(θ) | `/pro` (前感情), `/s-` (方向性), `/kho` (場), `/bou-` (目的) | 「何を見るか」を決める認知の準備 |
| **Likelihood** | 尤度 P(x\|θ) | 各マクロ固有の認知操作 | 世界を観測し、予測誤差を計算する |
| **Posterior** | 事後分布 P(θ\|x) | `/pis` (確信度), `/dox-` (記録), `~(/h*/k)` (動機×文脈), `~(/noe*/dia)` (直観×判定) | 更新された信念を評価・記録する |

## 具体 1: @nous の昇華 — 「問う」とは何か

**Before** (Prior 欠如):

```
@nous = R:{F:[×2]{/u+*^/u^}}_/dox-
```

問いの核だけがある。何について問うか、問いの質はどうか — 不在。

**After** (3 相完備):

```
@nous = /pro_/s-_R:{F:[×2]{/u+*^/u^}}_~(/noe*/dia)_/pis_/dox-
```

| 相 | 要素 | 人の認知で何が起きるか |
|:---|:-----|:---------------------|
| Prior | `/pro_/s-` | 「何が引っかかるか」を感じ、問いの方向を定める |
| Likelihood | `R:{F:[×2]{/u+*^/u^}}` | 再帰的に深く自問する (核は変わらない) |
| Posterior | `~(/noe*/dia)_/pis_/dox-` | 問いの質を直観×判定で検証し、確信度を測り、記録する |

**教訓**: 人は「いきなり問う」のではない。問う前に「何が分からないか」を感じ、問うた後に「良い問いだったか」を評価している。

## 具体 2: 密度判定の 3 段階基準

マクロが「認知の営み」として十分かどうかの判定基準:

| 判定 | 条件 | 例 |
|:-----|:-----|:---|
| **✅ 十分** | 3 相全てが存在し、核に相応の複雑さがある | @build: `/bou-_/chr_/kho_/s+_/ene+_V:{/dia-}_I:[✓]{/dox-}` |
| **🔧 微調整** | 核は適切だが Prior か Posterior が欠けている | @dig (旧): `/s+~(/p*/a)_/ana_/dia*/o+` — Posterior なし |
| **⚡ 要昇華** | Prior も Posterior も欠け、核だけが裸で存在する | @learn (旧): `/dox+_*^/u+_/bye+` — Prior も Posterior もなし |

**診断手順**:

1. **Prior を探す**: `/pro`, `/s-`, `/kho`, `/bou-` のいずれかがあるか？
2. **Posterior を探す**: `/pis`, `/dox-`, `~(/h*/k)`, `~(/noe*/dia)` のいずれかがあるか？
3. **どちらも欠ければ ⚡**、片方だけなら 🔧、両方あれば ✅

## 具体 3: 新マクロ設計チェックリスト

新しいマクロを作るとき、以下の 3-step で認知の営みとしての完全性を担保する:

### Step 1: 核を定義する

「このマクロは何をするのか？」— 認知操作の本質を CCL で表現する。this is Likelihood.

```
例: @search の核 → /sop~(/zet*/noe)
    (知恵を用いて探索×認識で調べる)
```

### Step 2: Prior を付与する

「この行為の前に、人は何をしているか？」

| 行為の性格 | 推奨 Prior |
|:-----------|:-----------|
| 直感的 | `/pro` (前感情) |
| 分析的 | `/s-` (方向性) |
| 環境依存 | `/kho` (場の把握) |
| 目的駆動 | `/bou-` (目的確認) |

### Step 3: Posterior を付与する

「この行為の後に、人は何をしているか？」

| 行為の結果 | 推奨 Posterior |
|:-----------|:--------------|
| 判断を含む | `~(/noe*/dia)` (直観×判定の振動) |
| 全体像に関わる | `~(/h*/k)` (動機×文脈の融合) |
| 確信度が重要 | `/pis` (確信度測定) |
| 永続化が必要 | `/dox-` (信念記録) |

### 完成形の検証

```
✅ /pro_/sop~(/zet*/noe)_/pis_/dox-
   Prior → Likelihood → Posterior — 認知の営みとして完全
```
