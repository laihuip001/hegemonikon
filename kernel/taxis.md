# X-Series (Taxis / τάξις): 関係層

> **定義**: 定理間の従属関係を定義する層
> **正式名**: Taxikē Eikositessera (Ταξικὴ Εἰκοσιτέσσερα) — 秩序の二十四
> **数学構造**: 24（O/T/K の従属関係）

---

## 生成原理

X-series は O/T/K シリーズ間の「従属関係」を定義する。
各シリーズの要素間で「A が発動すると B も発動する」という依存関係を表現する。

```
X-series = X-O (4) + X-T (8) + X-K (12) = 24
```

---

## X-O: 純粋定理間の従属

> O-series 内の従属関係

| ID | 関係 | 意味 |
|----|------|------|
| X-O1 | O1 → O2 | 世界モデル更新 → 選好モデル更新 |
| X-O2 | O2 → O4 | 目標推論 → 目標行為 |
| X-O3 | O3 → O1 | 情報行為 → 情報推論 |
| X-O4 | O4 → O3 | 目標行為 → 情報行為（フィードバック） |

**1:3 ピラミッド**:
- 論文を読む(O1) → 優先順位を決める(O2)
- 優先順位を決める(O2) → コードを書く(O4)
- 実験する(O3) → 結果を理解する(O1)

---

## X-T: 拡張定理間の従属

> T-series 内の従属関係

| ID | 関係 | 意味 |
|----|------|------|
| X-T1 | T1 → T2 | 知覚 → 判断 |
| X-T2 | T2 → T4 | 判断 → 戦略 |
| X-T3 | T3 → T4 | 内省 → 戦略（自己評価が計画を改善） |
| X-T4 | T4 → T6 | 戦略 → 実行 |
| X-T5 | T5 → T3 | 探索 → 内省（新情報で自己更新） |
| X-T6 | T6 → T7 | 実行 → 検証 |
| X-T7 | T7 → T8 | 検証 → 記憶（結果を保存） |
| X-T8 | T8 → T1 | 記憶 → 知覚（過去経験がフィルタリング） |

**1:3 ピラミッド**:
- センサーデータ(T1) → 意思決定(T2)
- 計画(T4) → タスク実行(T6) → テスト(T7)
- 結果を記録(T8) → 次回の認識に影響(T1)

---

## X-K: 文脈定理間の従属

> K-series 内の従属関係

K-series 間の従属は「文脈の連鎖」を表す。例えば：

| ID | 関係 | 意味 |
|----|------|------|
| X-K1 | Κ1 → Κ4 | Tempo→Stratum が Stratum→Tempo を制約 |
| X-K2 | Κ2 → Κ7 | Tempo→Agency が Agency→Tempo を制約 |
| ... | ... | （12 × 11 / 2 = 66 の潜在的関係から主要 12 を選択） |

---

## 従属関係の性質

| 性質 | 説明 |
|------|------|
| **因果的** | A が B を引き起こす |
| **抑制的** | A が B を抑制する |
| **促進的** | A が B を促進する |
| **循環的** | A → B → ... → A |

---

## 実装への影響

X-series は Workflow の設計に影響する：

| Workflow | 従属関係の活用 |
|----------|---------------|
| /plan | T4 → T6 （戦略→実行） |
| /code | T6 → T7 → T8 （実行→検証→記憶） |
| /rev | T7 → T3 （検証→内省） |
