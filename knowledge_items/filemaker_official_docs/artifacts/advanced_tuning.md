# FileMaker 高度なチューニングテクニック

**出典**: Performance_Tuning_Techniques.pdf (エムシス提供)

## 1. データベース管理システムとしての FileMaker

FileMaker の「盲点」を理解することが、メンテナンス性とパフォーマンスの維持に不可欠。

## 2. スキーマ構造とパフォーマンス

### テーブルの正規化と分割

- **非正規化のリスク**: フィールド数が多すぎる「巨大なテーブル」はレコード読み込み時のキャッシュ消費を増大させる。
- **解決策 (テーブル分割)**:
  - 頻繁にアクセスするフィールドと、稀にしか使わないフィールド（メモやログなど）を別テーブルに分ける。
  - これにより、キャッシュ効率が向上し、ネットワークトラフィックが減少する。

### リレーションシップとキャッシュ

- リレーションシップの複雑性はキャッシュサイズに直結する。
- 不要なリレーションパスを整理し、評価コストを削減する。

## 3. WebDirect の最適化

- WebDirect はサーバー側で UI を生成するため、クライアント（ブラウザ）とのやり取りを最小化する必要がある。
- スクリプトトリガの使用を控えめにし、サーバーの負荷を監視する。

## 4. JavaScript の活用

- Web ビューア内で JavaScript を実行することで、FileMaker 本体の負荷を抑えつつ高速な処理を実現できる。
- **全文検索**: JavaScript によるクライアントサイド検索の導入。
- **UI工夫**: 郵便番号検索 (PostFinder) など、外部APIやライブラリを活用した高速なUI実装。
