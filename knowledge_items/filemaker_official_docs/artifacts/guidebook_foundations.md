# FileMaker ガイドブック重要要旨

**出典**: cfg_ba_V3.pdf (基礎編), cfg_in_V3.pdf (実践編) - 第3版 (2025)

## 1. データモデリング

- **概念設計**: 業務プロセスからエンティティを抽出する。
- **リレーションシップの基本**:
  - 主キー (PK) と 外部キー (FK) による 1:多 の接続。
  - 自然キーよりもサロゲートキー（UUIDなど）の使用が推奨される。

## 2. リレーションシップグラフの管理

- **テーブルオカレンス (TO)**: グラフ上の「別名」。同じベーステーブルでも用途（一覧、詳細、フィルタ用）に応じて複数のTOを作成する。
- **パスの制約**: ループの禁止。パスが一意に定まるように「アンカー・ブイ」方式などでグラフを整理する。

## 3. スクリプトと自動化

- **構造化**: If/End If や Case による分岐、Loop による繰り返し。
- **デバッグ**: スクリプトデバッガを用いたステップ実行と変数監視。
- **トリガ**: OnRecordLoad や OnObjectEnter などのイベント駆動開発。

## 4. セキュリティ設計

- **アカウント管理**: OAuth (Google/Microsoft/Amazon) 連携による多要素認証。
- **アクセス権セット**: テーブル・フィールドレベルでの閲覧・編集制限。
- **通信**: SSL/TLS によるサーバー・クライアント間の暗号化。

## 5. モバイル・Web展開

- **FileMaker Go**: iOSデバイス固有の機能（カメラ、位置情報、署名）の活用。
- **WebDirect**: プラットフォーム非依存のブラウザ展開。
