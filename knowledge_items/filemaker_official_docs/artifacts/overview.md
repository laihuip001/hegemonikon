## 主要トピック

### 1. パフォーマンス設計

**出典**: DesignPerformanceJP.pdf

| カテゴリ | 内容 |
| :--- | :--- |
| レイアウト設計 | 画面サイズ、オブジェクト数、条件付き書式の最適化 |
| スキーマ設計 | テーブル・フィールド・リレーションシップ構造 |
| スクリプト最適化 | サーバーサイド処理、キャッシュ活用 |

**パフォーマンス評価の3観点**:

1. カスタムAppがどれだけ速く立ち上がるか
2. ユーザがどれだけ迅速に業務を完了できるか
3. 必要な情報を取り出すのにどれだけ時間がかかるか

### 3. リレーションシップ

**出典**: next_knowledge.pdf, cfg_ba_V3.pdf

| 概念 | 説明 |
| :--- | :--- |
| テーブルオカレンス (TO) | リレーションシップグラフ上のテーブルの分身/別名 |
| 主キー (PK) | レコードを一意に識別するフィールド |
| 外部キー (FK) | 別テーブルの主キーを参照するフィールド |
| アドオンテーブル | 関連データ表示を自動設定 |

**制約**: 同じTO間に複数のリレーションパスは不可

## MICKS業務システムへの適用

| 公式ドキュメントの教え | MICKS での実装 |
| :--- | :--- |
| TO を分けて別パスを作る | Anchor-Buoy パターン（System ハブ） |
| グローバルフィールドで状態管理 | `G_UI_*`, `G_SCH_*` パターン |
| スクリプト引数でTO参照ミス回避 | `Get(スクリプト引数)` 活用 |
| ポータルフィルタ | 検索条件グローバル参照 |
| 文字化け対策 | UTF-8 with BOM (U-BOM) 必須 |
