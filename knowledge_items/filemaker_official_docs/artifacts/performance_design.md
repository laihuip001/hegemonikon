# FileMaker パフォーマンス設計ベストプラクティス

**出典**: DesignPerformanceJP.pdf

## 1. パフォーマンスの定義

パフォーマンスとは単なる処理速度ではない。以下の3つの観点で評価される：

1. **カスタム App の起動速度**: どれだけ速く立ち上がるか。
2. **業務完了の迅速性**: ユーザがどれだけ迅速に業務を完了できるか。
3. **情報取得の所要時間**: 必要な情報を取り出すのにどれだけ時間がかかるか。

## 2. パフォーマンスに影響を与える主要因

- **クライアント・サーバーの作業量**: 計算、レイアウト表示などの処理量。
- **データ転送量**: ネットワークを介して送信されるデータ量。

## 3. レイヤごとの最適化指針

### レイアウト（画面設計）

- 画面サイズとオブジェクト数の最小化。
- 条件付き書式やポータルフィルタの評価コストに留意。
- 帯域幅の制限（インターネット経由のアクセス）を考慮した設計。

### データベーススキーマ

- テーブル、フィールド、リレーションシップ構造の簡素化。
- インデックスの適切な管理。

### ロジック

- 計算式（データの保存、非保存の選択）。
- スクリプト（特にサーバーサイドでのスクリプト実行 PSOS の活用）。

## 4. ネットワーク環境の考慮

オフィス内（LAN）では高速でも、低速なインターネット環境やモバイル環境（FileMaker Go/WebDirect）ではパフォーマンスが劇的に低下する可能性がある。
データ転送量を最小限に抑える設計が「最高のパフォーマンス」への鍵となる。
