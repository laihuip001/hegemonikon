## ミスの記録 (時系列)

### 2026-02-07: 熟読の堕落

```
状況: Brain KB 16記事を「高速で進めます」と読了
問題: L1精読のみで L2沈潜・L3摩擦を省略した
原因: 効率 > 深さ のトレードオフで効率を選んだ
Creator の指導: 「熟読とは思考に耽ること」
教訓: 速さは美ではない。深さが美である。
対策実施: /jukudoku v2.0 に L2+L3 を構造化
```

### 2026-02-07: Creator プロファイルの忘却

```
状況: 「Creator の好み・癖を毎回忘れる」と発言
問題: 自己分析テキスト(AI用).md と AuDHD KB.md が既に存在していることを忘れていた
原因: 既存資産のスキャンを怠った
教訓: 「ないと思う前に探す」 — 存在検証が先
対策予定: /boot で Creator プロファイルの明示的読み込みを追加
```

### 2026-02-07: 確認の省略

```
状況: Creator の指示を受けて即座に実行に移った場面が複数あった
問題: 「つまりこういうことですか？」の確認ステップを省略
原因: 確認を「コスト」と認識していた
Creator の指導: 「確認しないことがコスト。確認は不確実の低減」
教訓: 確認 = FEP の実践そのもの。追加観測で予測誤差を減らす行為。
対策予定: 曖昧な指示には必ず確認を返す
```

### 2026-02-07 Session 3: force_cpu 誤設定

```
状況: Attractor boot統合で force_cpu=True を設定
問題: ONNX モデルが配置されていないため即座にエラー
原因: pytest テスト (force_cpu=False) と環境を揃えなかった
教訓: 動作実績のある設定をデフォルトにする。新設定は動作確認後に導入。
対策実施: force_cpu=False に変更、タイムアウトも30秒に延長
```
