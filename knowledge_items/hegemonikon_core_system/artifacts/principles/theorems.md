## 2. Cognitive Control Language (CCL) Operators

CCL syntax patterns define intended cognitive effects.

| Dimension | Operators | Semantic | Example |
| :--- | :--- | :--- | :--- |
| **Intensity** | `+` / `-` | Deepen / Contract | `/noe+` |
| **Temporal** | `'` / `∫` | Derive / Integrate | `/bou'` |
| **Phase** | `/` / `\` | Inner / Outer Product | `/a` vs `\a` |
| **Abstraction** | `^` / `√` | Ascend / Descend | `/noe^` |
| **Scaling** | `e` / `ln` | Exponential / Logarithmic | `e/bou` |

### Key Operator Patterns

- `*^`: **Fused Meta-Display**: Fused thinking with meta-state display.
- `_`: **Strict Sequence**: Temporal chain logic.
- `*` vs `~`: **Result vs. Process**: `*` suppresses intermediate chatter.

## 3. Context Density Over API Cost

**Axiom**: "Naked API calls are garbage. Context is EVERYTHING."

The value of an LLM-driven inference is proportional to the **Density** (Identity, Data, Theorem, Audit) and **Fidelity** of the provided context. High-density context is the catalyst that transforms generic prediction into actionable intelligence.

## 4. System Integrity and Cognitive Drift

### 4.1. Cognitive Drift

The divergence between the AI's internal belief about system capabilities and the actual environment state. This increases during platform migrations or when context pressure causes reliance on generic defaults.

### 4.2. Mitigation (Diorthōsis)

- **Integrity Checks**: Frequent verification of "Core 正本" (original texts) via file tools.
- **Negative Context Tracking**: Explicitly listing missing tools/workflows in handoffs.
- **Axiom of Verification**: "Never assume a tool exists just because it is logical for it to exist."
