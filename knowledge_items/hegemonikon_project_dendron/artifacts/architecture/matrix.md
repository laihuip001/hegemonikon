# Existence Purpose Tensor (EPT) Architecture

Dendron employs a 3D tensor to verify codebase integrity, mapping proofs across granularity, logic, and time.

## 1. The Three Dimensions

1. **Depth (L0–L3)**:
    - **L0 (Kairos)**: Directory (Context/Placement).
    - **L1 (Ousia/Schema)**: File (Essence/Structure).
    - **L2 (Hormē/Perigraphē)**: Function/Class (Motivation/Boundary).
    - **L3 (Akribeia)**: Variable/Token (Precision).
2. **Meta Layer (Normal Forms)**:
    - **Surface (NF1)**: Existence proof (PROOF declarations).
    - **Structure (NF2)**: Dependency relations (Import/Call graphs).
    - **Function (NF3)**: Functional redundancy (Semantic similarity/MECE).
    - **Verification (BCNF)**: Indispensability (Deletion/Ablation testing).
3. **Temporal Axis (The 2M-Matrix)**:
    - **Purpose (未来/目標/意図)**: Teleological intent (Goal-oriented). What the component aims to achieve.
    - **Reason (過去/根拠/原因)**: Historical background and logical premises. Why the component was born.

## 2. Tensor Logic

A complete existence proof spans **32 cells** (4 Depth x 4 Meta x 2 Temporal).

```text
                        Purpose (目的)              Reason (理由)
                    ┌────────────────────┐     ┌────────────────────┐
                    表層  構造  機能  実証       表層  構造  機能  実証
              ┌────┬────┬────┬────┬────┐ ┌────┬────┬────┬────┬────┐
    L0 (Dir)  │ P00│ P01│ P02│ P03│    │ R00│ R01│ R02│ R03│
              ├────┼────┼────┼────┤    ├────┼────┼────┼────┤
    L1 (File) │ P10│ P11│ P12│ P13│    │ R10│ R11│ R12│ R13│
              ├────┼────┼────┼────┤    ├────┼────┼────┼────┤
    L2 (Func) │ P20│ P21│ P22│ P23│    │ R20│ R21│ R22│ R23│
              ├────┼────┼────┼────┤    ├────┼────┼────┼────┤
    L3 (Var)  │ P30│ P31│ P32│ P33│    │ R30│ R31│ R32│ R33│
              └────┴────┴────┴────┘    └────┴────┴────┴────┘

    P = Purpose (主役)    R = Reason (背景)
```

**Existence Formula**:

```text
Existence Error = Existence - Purpose
```

Complexity in the proof matrix is seen as a reflection of the system's "Value"—ensuring that every component is mathematically and philosophically justified.
