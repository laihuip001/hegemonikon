# Transformations Reference

プロンプト品質向上のための変換ルール集。

---

## 1. 曖昧語 → 具体化変換

| 曖昧語 | 変換後 | 例 |
|:---|:---|:---|
| 適切に | [条件A]なら[処理X]、[条件B]なら[処理Y] | 「適切に要約」→「300字以内、主要論点3つ抽出」 |
| うまく | [成功基準]を満たす形で | 「うまく説明」→「専門用語なし、中学生が理解可能」 |
| 高品質な | [指標]が[閾値]以上 | 「高品質な文章」→「文法エラー0件、可読性スコア70以上」 |
| 必要に応じて | [トリガー条件]を満たした場合のみ | 「必要に応じて詳細化」→「ユーザーが『詳しく』と言った場合のみ」 |
| できるだけ | 優先度[N]として、[制約]の範囲内で最大化 | 「できるだけ短く」→「100字以下を目標、最低限の情報は保持」 |
| 適宜 | [判断基準]に基づき選択 | 「適宜改行」→「1文が50字を超えたら改行」 |
| なるべく | [閾値]を目標とし、[最低ライン]は必達 | 「なるべく早く」→「目標2秒、最大5秒」 |
| いい感じに | [評価軸]で[スコア]以上 | 削除（使用禁止） |
| ちゃんと | [チェックリスト]を全て満たす | 「ちゃんと確認」→「項目A,B,Cを全てチェック」 |
| しっかり | [深度レベル]で実行 | 「しっかり分析」→「3階層まで原因分析」 |

## 2. 冗長表現 → 簡潔化変換

| 冗長表現 | 簡潔形 |
|:---|:---|
| 〜することができます | 〜可能 |
| 〜を行う必要があります | 〜必須 |
| 〜の可能性があります | 〜の恐れ / 要検証 |
| 〜と考えられます | 〜と推定（確信度X%） |
| 〜ではないでしょうか | 〜の可能性（要検証） |
| 〜していただけますと幸いです | 〜してください |
| 〜させていただきます | 〜します |
| 〜という形になります | 〜です |
| 〜の方向で進めていきたいと思います | 〜で進行 |
| 〜に関しましては | 〜について |
| 基本的には〜 | 〜（例外: [例外条件]） |

---

## 3. 確信度表現マッピング

| 確信度 | 表現パターン | 使用場面 |
|:---|:---|:---|
| 90-100% | 断定（「〜である」「〜だ」） | 検証済み事実、定義 |
| 70-89% | 「〇〇の可能性が高い」 | 強い推論、多数派見解 |
| 50-69% | 「〇〇と推定される」 | 合理的推論、限定的根拠 |
| 30-49% | 「要検証: 〇〇の可能性」 | 仮説段階、根拠薄弱 |
| 0-29% | 「不明（情報不足）」 | 判断不能、データなし |

## 4. 指示動詞の強度マッピング

| 強度 | 動詞 | 意味 |
|:---|:---|:---|
| **必須** | 〜せよ / 〜すること / MUST | 違反は失敗 |
| **強く推奨** | 〜すべき / SHOULD | 違反は警告 |
| **推奨** | 〜が望ましい / RECOMMENDED | ベストプラクティス |
| **任意** | 〜してもよい / MAY | オプション |
| **禁止** | 〜してはならない / MUST NOT | 違反は失敗 |

## 5. 数量表現の具体化

| 曖昧表現 | 具体化パターン |
|:---|:---|
| 多くの | N個以上（Nを明示） |
| 少数の | N個以下（Nを明示） |
| ほとんど | X%以上（90%以上推奨） |
| 一部 | X%程度（10-30%推奨） |
| 頻繁に | N回/[期間] |
| まれに | N回/[期間]以下 |
| 長い | X[単位]以上 |
| 短い | X[単位]以下 |
| 大きい | X[単位]以上 |
| 小さい | X[単位]以下 |

## 6. 条件分岐の標準形式

### Before（曖昧）
```
状況に応じて適切に対応してください。
```

### After（明確）
```
以下の条件分岐に従って対応:

IF [条件A]:
  THEN [処理X]
ELIF [条件B]:
  THEN [処理Y]
ELSE:
  THEN [デフォルト処理Z]
```

## 7. 優先度表現の標準化

| 表現 | 標準形 | 処理順序 |
|:---|:---|:---|
| 最優先 | P0（Critical） | 即時対応 |
| 重要 | P1（High） | 24時間以内 |
| 通常 | P2（Medium） | 今週中 |
| 低優先度 | P3（Low） | 余裕時 |
| 検討 | Backlog | 未定 |

---

## 8. 範囲表現の明確化

| 曖昧表現 | 明確化 |
|:---|:---|
| 〜など | 列挙 + 「これに限る」or「例示であり他も含む」 |
| 〜等 | 同上 |
| 〜その他 | 含まれるものを明示 or 「上記以外全て」 |
| 〜を含む | 必須要素 vs 任意要素を区別 |
| 〜に関する | 具体的なスコープを定義 |

---

## 9. エラーハンドリング表現

### Before（曖昧）
```
エラーが発生したら適切に対応してください。
```

### After（明確）
```
エラーハンドリング:

1. [エラータイプA]:
   - 検出条件: [条件]
   - 対応: [具体的なアクション]
   - 出力: 「[エラーメッセージ]」

2. [エラータイプB]:
   - 検出条件: [条件]
   - 対応: [具体的なアクション]
   - 出力: 「[エラーメッセージ]」

3. 想定外エラー:
   - 対応: [デフォルトアクション]
   - 出力: 「予期せぬエラーが発生しました。[コンテキスト情報]」
```

## 10. 時間表現の標準化

| 曖昧表現 | 標準形 |
|:---|:---|
| すぐに | X秒/分以内（数値を明示） |
| 後で | [具体的な期限] |
| そのうち | Backlog（期限未定を明示） |
| 定期的に | X[単位]ごと |
| 随時 | [トリガー条件]発生時 |

## 変換チェックリスト

プロンプト生成後、以下を確認:

```
□ 「適切」「うまく」「いい感じ」等の曖昧語: 0件
□ 全ての基準に数値または条件分岐あり
□ 確信度表現が統一されている
□ 指示動詞の強度が明確
□ 範囲表現が具体的
□ エラーハンドリングが定義済み
□ 時間表現が数値化されている
```
