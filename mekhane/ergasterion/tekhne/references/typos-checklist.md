# Evaluation Checklist
# ====================
# Týpos 生成物の品質確認用チェックリスト

## 簡易評価（5問セット）

### 1. ドメイン適合性
- [ ] 指定ドメインに応じて @constraints が変化しているか
- [ ] ドメイン固有の評価軸が @rubric に含まれているか
- [ ] 高リスクドメイン（医療/法律）で免責注記があるか

### 2. 構文正確性
- [ ] `typos.py parse` でエラーなし
- [ ] YAML インデントが正しい
- [ ] コロン後にスペースあり

### 3. 実用性
- [ ] @examples の入出力が現実的
- [ ] @format が具体的（「JSON形式」ではなく実際のスキーマ）
- [ ] @constraints が具体的（「適切に処理」→「〇〇の場合は△△する」）

### 4. 再利用性
- [ ] 共通部分が @mixin で切り出されているか（optional）
- [ ] ドメイン別テンプレートを @extends しているか（optional）

### 5. 改善ループ
- [ ] @rubric が定義されているか
- [ ] 評価軸が定量的か（1-5, 0-10 など）
- [ ] 閾値や合格基準があるか

---

## 詳細評価表

| カテゴリ | 項目 | 合格基準 | チェック方法 |
|:---------|:-----|:---------|:-------------|
| 構造 | 必須ディレクティブ | 5種全てあり | grep |
| 構造 | @constraints 数 | 3個以上 | count |
| 構造 | @examples 数 | 1個以上 | count |
| 品質 | 曖昧語 | 0件 | 禁止語リスト照合 |
| 品質 | 定量的基準 | @rubric に数値あり | pattern match |
| 形式 | YAMLパース | エラーなし | typos.py parse |
| 形式 | メタデータ | version, name, owner | exist check |
