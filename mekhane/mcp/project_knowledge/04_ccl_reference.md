# CCL (Cognitive Control Language) — リファレンス

## 基本演算子

| 演算子 | 名称 | 意味 | 例 |
|:-------|:-----|:-----|:---|
| `/` | ワークフロー | WF 呼び出し | `/noe` |
| `+` | 詳細化 | 深い版を実行 | `/noe+` |
| `-` | 簡略化 | 軽い版を実行 | `/noe-` |
| `~` | 振動 | 交互に実行 | `/noe~/dia` |
| `~*` | 収束振動 | 合意するまで交互実行 | `/noe~*/dia` |
| `>>` | 逐次 | 順番に実行 | `/noe >> /dia` |
| `_` | 連結 | 続けて実行 | `/noe_/dia` |
| `*` | 並列 | 同時に実行 | `/noe*/dia` |
| `{}` | グループ | ブロック化 | `{/noe_/dia}` |
| `()` | 部分式 | 優先順位 | `(/noe+~*/dia)` |
| `\` | コリミット | 複数を統合 | `\(/noe, /dia)` |

## 制御構造

| 構造 | 意味 | 例 |
|:-----|:-----|:---|
| `F:[×N]{...}` | N回繰り返し | `F:[×3]{/eat+}` |
| `F:[list]{...}` | リスト各要素に適用 | `F:[/s,/dia]{L:[x]{x~x+}}` |
| `C:{...}` | 条件: 真なら実行 | `C:{/dia+_/ene+}` |
| `V:{...}` | 検証: 通れば続行 | `V:{/dia}` |
| `I:[✓]{...}` | 通過時の処理 | `I:[✓]{/dox-}` |
| `E:{...}` | 失敗時の処理 | `E:{/ene{_limbo/}}` |
| `R:{...}` | 再帰 | `R:{F:[×2]{/u+}}` |
| `L:[x]{...}` | ラムダ束縛 | `L:[x]{x~x+}` |
| `*^` | 融合 (Fusion) | `*^/u+` |
| `@` | マクロ展開 | `@build` |

## 派生 (+/-/無印)

| 派生 | 深度 | 適用BC |
|:-----|:-----|:-------|
| `/noe-` | L1 Quick | Always-On + BC-3/10/14 |
| `/noe` | L2 Standard | + BC-8/9/15 |
| `/noe+` | L3 Deep | 全 BC |

## 使用例

```
# 本質を問う (深い版)
/noe+

# 計画を立てて実行
@plan_@build

# 論文を3本消化
F:[×3]{/eat+~(/noe*/dia)}

# 判定して、通れば記録、落ちれば保留
C:{V:{/dia+}_/ene+}_I:[✓]{/dox-}_E:{/ene{_limbo/}}
```
