# スキルYAML品質を安定させる方法

はじめにスキル定義のばらつきを減らし、レビューやCIで止まらない状態を作れます。結論は、最小テンプレートと機械検証を組み合わせることです。プロンプト設計者や開発者が、定義漏れや翻訳ズレを防ぎ、再現性を高められます。
本記事は、スキルYAMLの品質を安定させるための「濃縮チェックリスト」と最小テンプレート、すぐ使えるCI/AJV/Zodルールをまとめました。

SKILL frontmatter 濃縮チェックリスト必須キーskill.name（小文字ハイフンのケバブケース）
