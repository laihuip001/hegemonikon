# Morphism Guide — 射の提案の理解と評価

## PURPOSE

WF 完了時に出力される「射の提案 (BC-8)」の構造を理解し、
適切にレビュー・評価するためのガイド。

## 射の提案とは

WF 実行完了後、その結果を他の Series に接続する「次の一手」の候補。
HGK の 6 Series は独立ではなく、共有座標で接続されている。

## 提案の構造

```
🔀 射の提案 (trigonon: {Series}/{定理}/{認知タイプ})
⚖️ 均衡モード
├─ Bridge >> {遠い Series}: /wf1 /wf2 ...
├─ Anchor >> {近い Series}: via 中継
└─ (完了)
```

## Bridge vs Anchor

| 項目 | Bridge | Anchor |
|:-----|:-------|:-------|
| **距離** | 遠い Series に接続 | 近い Series に定着 |
| **目的** | 結果を実用化・外延 | 結果を深化・内縁 |
| **選ぶべき時** | 確信が低い / 別視点が必要 | 確信が高い / 結果を固めたい |

## 認知タイプと接続パターン

| Series | 認知タイプ | 主な Bridge 先 | 主な Anchor 先 |
|:-------|:----------|:--------------|:--------------|
| **O** (Ousia) | Pure (純粋認知) | S, H | なし (始点) |
| **S** (Schema) | Mixed (混合) | H, P | O |
| **H** (Hormē) | Mixed (混合) | S, K | O, A |
| **P** (Perigraphē) | Pragmatic (実用) | K | S |
| **K** (Kairos) | Pragmatic (実用) | A | P, H |
| **A** (Akribeia) | Pragmatic (実用) | なし (終点) | H, K |

## 評価基準

射の提案を評価する際のチェックリスト:

1. **文脈適合性**: 現在のタスクに関連する Series か？
2. **深度一致**: L2+ のタスクで出力されているか？
3. **理由の明確さ**: 「なぜこの射か」の説明があるか？
4. **実行可能性**: 提案された WF は実行可能な状態か？
