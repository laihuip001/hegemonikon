# PROOF.md — 専門家レビュー記録

PURPOSE: Specialist Review の結果を記録・蓄積する
REASON: 監査結果を永続化し、改善プロセスを追跡可能にするため

> **∃ reviews/** — この場所は存在しなければならない

---

## 公理

```
A0: 予測誤差最小化 (Free Energy Principle)
```

## 演繹

```
A0: 予測誤差最小化
  ↓ [FEP の定義]
P1: システムは予測誤差を最小化する
  ↓ [監査の必要性]
P2: 誤差を検出するためには監査 (Audit) が必要
  ↓ [記録の必要性]
P3: 監査結果は記録され、参照されなければならない
  ↓ [場所の定義]
P4: その記録場所を reviews/ と呼ぶ
```

**語源**: re- (再び) + videre (見る) = 再考・批評
