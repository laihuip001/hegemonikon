# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/anamnesis/index.py`

## 判定
発言（要改善）

## 発見事項
- **同一モジュール内での比喩混在 (Medium)**:
  - `GnosisIndex` (神秘・知識 / 図書館のメタファー) と `Embedder` (技術・工場のメタファー) が混在している。`Gnosis` (知識) や `Anamnesis` (想起) という認知・哲学的ドメインに対し、`Embedder` という即物的な機能名が唐突に現れる。
  - **改善案**: `Embedder` を `GnosisEncoder` や `MemoryEncoder` にリネームし、ドメインの言葉に合わせるか、`mekhane.anamnesis.infrastructure` 等に分離する。

- **命名規則の不統一 (Low)**:
  - `GnosisIndex` は `[Domain][Type]` (名詞句) であるのに対し、`Embedder` は `[Action]er` (動作者) である。
  - `Embedder` はシングルトンとして振る舞う重要なコンポーネントだが、名前が汎用的すぎる。
  - **改善案**: `Embedder` を `TextEmbedder` (機能明示) または `GnosisEncoder` (ドメイン統合) に変更する。

- **抽象度の不統一 (Low)**:
  - ドメインの核となる `GnosisIndex` と、その下請けである `Embedder` が同列にトップレベルクラスとして定義されている。
  - **改善案**: `Embedder` をプライベートクラス `_Embedder` にするか、別モジュールに移動する。

## 重大度
Medium
