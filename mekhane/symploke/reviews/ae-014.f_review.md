# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/mcp/sophia_mcp_server.py`

## 判定
発言（要改善）

## 発見事項
- **比喩体系の混在 (Factory/Garden)**: `Mekhane` (機械/工場) の空間内に、`Sophia` (知恵/庭園) や `Oikos` (家) の概念が浸食しています。特に `EmbeddingAdapter` (工業部品) が `Sophia` (女神/知恵) を扱う構成は、無機質と有機質の不調和を感じさせます。`KnowledgeConnector` や `SophiaGateway` のような、異なる領域をつなぐ儀礼的な命名、あるいは `SophiaIndex` のような整然とした命名が望まれます。
- **抽象度の不統一**: `StdoutSuppressor` という配管工のようなヘルパークラスが、`Server` という支配人や `list_tools` という主要業務と同列のトップレベルに鎮座しています。これは工場のレイアウトとして美しくありません。別モジュール (`utils.py`) へ移動し、メインフロアをすっきりさせるべきです。
- **命名規則の不統一**: `SophiaBacklinker` (ドメイン名主体) と `EmbeddingAdapter` (実装パターン主体) が混在しています。`SophiaSearcher` 対 `SophiaLinker` のようにドメインで統一するか、役割で統一するべきです。
- **動詞の活用不統一**: ツールの説明文において、`search` は "Returns relevant knowledge" (返却する) ですが、`stats` や `backlinks` は "Get statistics" (取得する) となっています。`Retrieve` (探索して持ち帰る) や `Fetch` (取ってくる)、あるいは `Show` (提示する) など、サービスの振る舞いに合わせて統一された動詞を使用すべきです。
