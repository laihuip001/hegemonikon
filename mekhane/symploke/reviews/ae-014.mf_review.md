# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/ochema/cli.py`

## 判定
発言（要改善）

## 発見事項
- [Medium] 隠喩の混合: モジュール全体で比喩体系が混在している。
    - 工場/システム: `status`, `quota`, `archive`, `models`
    - 庭園/生物: `health` (Context Rot), `brain_id`
    - 物理/SF: `Antigravity`, `ask`
    - 提案: `Ochēma` (乗り物) / `Antigravity` (物理) の世界観に統一するため、`health` -> `diagnostics`, `brain` -> `store` / `memory_bank`, `Context Rot` -> `Entropy` 等への変更を推奨。
- [Low] 命名規則の不統一: コマンド名が動詞と名詞で混在している。
    - 動詞: `ask`, `chat`, `archive`
    - 名詞: `status`, `models`, `quota`, `sessions`, `episodes`, `health`
    - 形容詞: `smart`
    - 提案: 全て動詞（例: `check-status`, `list-models`, `diagnose`）または名詞（例: `inquirer`, `archiver`）に統一するか、CLI標準的な動詞-目的語形式（例: `get-status`, `list-models`）への変更を推奨。特に `smart` は `auto-ask` や `consult` などの動詞に変更すべきである。
- [Low] 抽象度の不統一: `cmd_smart_ask` が `client._select_model`（保護メソッド）に直接アクセスしており、他のコマンドと抽象度が異なる。
    - 提案: `AntigravityClient` に公開メソッド `select_model` を追加し、CLIからはそれを呼び出すように修正することを推奨。
