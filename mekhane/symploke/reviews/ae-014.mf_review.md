# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/mcp/ochema_mcp_server.py`

## 判定
発言（要改善）

## 発見事項
- [Medium] 比喩混在: `Ochēma` (乗り物/物理) ドメイン内に `Context Rot` (庭園/生物) が混入している。「腐敗」ではなく `Context Drift` (漂流) や `Signal Decay` (減衰) が物理メタファーとして適切。
- [Medium] 抽象度の不統一: `session_info` ツールが `OchemaService` を介さず `ls_client._rpc` を直接呼び出している。Server 層 (Manager) が RPC 層 (Mechanic) の詳細を知りすぎている。
- [Low] 命名規則の不統一: `ask` (LS暗黙) vs `ask_cortex` (明示)、`models` (LSのみ) vs `cortex_quota` (Cortexのみ) の非対称性。`ask_ls` / `ask_cortex` のように対称化すべき。
- [Low] 動詞の活用不統一: `get_service` (動詞) と `svc.quota` / `svc.status` (名詞) が混在している。
