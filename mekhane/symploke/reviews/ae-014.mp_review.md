# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/mcp/ochema_mcp_server.py`

## 判定
発言（要改善）

## 発見事項
- (Medium) 比喩体系の混在: 機械論的/物理的メタファー ("Cascade", "Trajectories", "Bridge", "Vehicle") が支配的な中で、突如として生物学的/有機的メタファー ("Context Rot", "Rot Risk", "HEALTHY") が出現している。
- (Medium) 抽象度の不統一: `call_tool` 関数が、ツールの振り分け（Manager の責務）と各ツールの具体的な実装詳細（Worker の責務）を同列に扱っており、巨大な関数となっている。
- (Low) 命名規則の不統一: MCP ツールの命名が、動詞 ("ask", "ask_cortex") と名詞 ("status", "models", "session_info", "cortex_quota") で混在している。
- (Low) 動詞の不統一: コード内のコメントで "Fetch" ("Fetch all trajectories") が使われているが、関数名や API 呼び出しでは "Get" ("get_service", "GetAllCascadeTrajectories") が使われており、用語が揺れている。

## 重大度
Medium
