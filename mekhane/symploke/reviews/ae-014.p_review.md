# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/ochema/antigravity_client.py`

## 判定
発言（要改善）

## 発見事項
- **[比喩混在] (Medium)**: クラス全体は `Antigravity` (反重力), `Trajectory` (軌道), `Cascade` (カスケード) といった「物理/SF/乗り物」の比喩体系に基づいています。しかし、`context_health` メソッド内では "Context Rot" (コンテキストの腐敗) という「庭園/生物」の比喩が使用されています。乗り物であれば "Degradation" (劣化) や "Overload" (過負荷) などが適切です。また、`session_episodes` での "Brain" (脳) という表現も生物的であり、物理的メタファーと混在しています。
- **[抽象度の不統一] (Medium)**: `AntigravityClient` は本来、Language Server との通信を行う「Helper/Driver」層の責務を持つべきですが、`smart_ask` (モデルルーティング)、`context_health` (健全性判定ポリシー)、`archive_sessions` (ファイルシステム管理) といった「Manager/Orchestrator」層の責務が混入しています。
- **[命名規則/動詞の不統一] (Low)**: 公開メソッドの命名において、動詞の位置や有無が一貫していません。
    - `verb_noun`: `get_status`, `list_models`, `archive_sessions`
    - `noun_noun` (動詞なし): `quota_status`, `session_info`
    - `noun_verb`: `session_read`
    - 推奨: 全て `verb_noun` (例: `get_quota_status`, `get_session_info`, `read_session`) に統一することを提案します。
