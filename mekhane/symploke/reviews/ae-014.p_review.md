# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/ergasterion/digestor/scheduler.py`

## 判定
発言（要改善）

## 発見事項
- **[Medium] 比喩の混在 (工場 vs 生物 vs 物流)**
  - `Digestor` (生物: 消化) と `Pipeline` (工場: パイプライン) が `DigestorPipeline` や `run_digestor` 内で混在しています。
  - `eat_*.md` (生物: 食べる) というファイル名に対し、`incoming`/`expired` (物流/在庫: 入荷/期限切れ) というフォルダ構成が用いられています。
  - `candidates` (人事/選抜) も混ざっており、ドメインの世界観が統一されていません。

- **[Low] 抽象度の不統一 (Manager と helper の同居)**
  - 全体進行を管理する `main`, `run_digestor` と、低レイヤーの作業を行う `save_pid`, `notify_desktop`, `log` が並列に定義されています。
  - `cleanup` (シグナルハンドラ/システム停止) と `cleanup_expired` (ファイル整理/業務ロジック) が同列にあり、かつ名前が類似しているため、役割の階層が不明瞭です。
