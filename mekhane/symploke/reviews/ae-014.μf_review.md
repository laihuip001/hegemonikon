# 比喩一貫性の詩人 レビュー

## 対象ファイル
`mekhane/anamnesis/session_indexer.py`

## 判定
発言（要改善）

## 発見事項
- **[Medium] 比喩混在 (生物学的比喩の混入)**:
  - `_BRAIN_DIR` という命名は「脳 (Brain)」という生物学的比喩を用いており、ファイル全体の「図書館/アーカイブ (Indexer, Record, Paper)」という比喩体系と衝突しています。`_STEPS_DIR` や `_GENERATED_DATA_DIR` のような即物的な命名、あるいはアーカイブ的な命名への変更を推奨します。

- **[Low] 命名規則不統一 (Session vs Conversation)**:
  - `sessions_to_records` と `conversations_to_records` が混在しています。同一のドメイン概念（対話履歴）を扱っているにもかかわらず、用語が揺れています。`Session` または `Conversation` のどちらかに統一すべきです（`GnosisIndex` の文脈では `Session` が優勢に見えます）。

- **[Low] 関数動作と命名の不一致**:
  - `parse_step_outputs` はディレクトリ探索を含んでいますが、他の `parse_*` 系関数は単一入力の変換を指しています。`collect_step_outputs` や `scan_step_outputs` のような、収集動作を含意する動詞が適切です。

- **[Low] 動詞の不統一 (Fetch vs Index/Parse)**:
  - `fetch_all_conversations` で `fetch` が使われていますが、他のデータ取得箇所と統一されていません。アーキビスト（司書）の隠喩に合わせるなら `retrieve_all_conversations` や `collect_all_conversations` が好ましいです。
