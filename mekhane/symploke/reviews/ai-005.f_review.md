# コード/コメント矛盾検出者 レビュー

## 対象ファイル
`mekhane/ochema/cortex_client.py`

## 判定
発言（要改善）

## 発見事項
- **_call_api メソッドの token_override の挙動矛盾 (High)**
  - Docstring: `token_override: Use specific token (skip auto-refresh)` とあり、指定トークンの使用と自動更新の回避を謳っている。
  - Code: 401/403 エラー時のリトライ処理で `token = self._get_token()` が実行され、`token_override` が無視されてデフォルトのトークン（`_CREDS_FILE` 由来）が使用される。これは "skip auto-refresh" および "Use specific token" の説明と矛盾する（指定トークンが失効している場合、別の認証情報でリトライしてしまう）。

- **内部メソッドの Args/Returns ドキュメント欠落 (Medium)**
  - `_build_request`: 多数の引数 (`contents`, `model` 等) をとるが、Docstring に `Args` セクションがない。
  - `_parse_response`: `response` 引数と戻り値 `LLMResponse` の説明がない。
  - `_get_project`: `token` 引数の説明がない。
  - `_load_oauth_config`: 戻り値 `tuple[str, str]` と送出する `FileNotFoundError` の `Raises` 記述がない。

- **CortexAPIError の Args 欠落 (Low)**
  - `__init__` またはクラス Docstring に `message`, `status_code`, `response_body` の `Args` 説明がない。
