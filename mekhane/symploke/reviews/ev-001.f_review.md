# Series適所配置検査官 レビュー

## 対象ファイル
`mekhane/anamnesis/session_indexer.py`

## 判定
発言（要改善）

## 発見事項
- **Series不一致 (Medium)**:
  - 当ファイルは「データの収集・変換・保存実行」を行う **S-Series (Method/Execution)** の性質を持つ ETL パイプラインである。
  - しかし、配置場所は **K-Series (Context/Knowledge)** である `mekhane/anamnesis/` となっている。
  - `anamnesis` は「記憶の定義」「検索インターフェース」「知識モデル」（静的・受動的）を責務とすべきであり、CLI 実行や外部プロセス起動を含む「作業（Ergon）」は `mekhane/ergasterion` (Workshop) または `scripts/` の責務である。

- **責務の越境 (Medium)**:
  - `subprocess.run` によるシェルスクリプト実行や、`AntigravityClient` を用いたネットワーク I/O などのインフラロジックが、知識層 (Anamnesis) に混入している。
  - `GnosisIndex` クラスの内部 DB (`index.db`) を直接操作して書き込みを行っており、カプセル化を破壊している（本来は `GnosisIndex` に `add_sessions` などのメソッドを追加するか、`ergasterion` 側のパイプラインで処理すべき）。

- **ロジック配置違反 (Low)**:
  - `main()` 関数や `argparse`、`print` 出力などの **Presentation/UI Logic** が知識層に含まれている。これは `mekhane/fep` の UI ロジック混入と同様のアンチパターンである。

## 重大度
Medium
