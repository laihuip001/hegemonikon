# Series適所配置検査官 レビュー

## 対象ファイル
`mekhane/mcp/hgk_gateway.py`

## 判定
発言（要改善）

## 発見事項
- **責務の越境 (Medium)**: `hgk_search` が `GnosisIndex` の呼び出しに加え、`DOXA_DIR` や `SESSIONS_DIR` のファイルシステム検索を直接実装している。これは知識検索オーケストレーションであり、本来 `mekhane.pks.gateway_bridge` や `mekhane.anamnesis` が担うべき責務である。
- **責務の越境 (Medium)**: `hgk_status`, `hgk_health`, `hgk_notifications` が `_MNEME_DIR` 内のファイル直接読み込みやプロセス監視を実装している。これはシステム健全性監視ロジックであり、本来 `mekhane.peira` (Trial/Health) に集約されるべきである。
- **Series不一致 (Low)**: `hgk_sop_generate` 内に「調査依頼書」の Markdown テンプレートがハードコードされている。知識テンプレートの定義は `mekhane.ergasterion` (Workshop) または `kernel` (Theory) に属すべきであり、L2/Infrastructure 層である `mcp` には適さない。
- **ロジック漏洩 (Low)**: `hgk_pks_search` 等が `subprocess` を使用して兄弟モジュールである `mekhane.pks.pks_cli` を呼び出している。同一プロセス内で Python API (`PKSEngine` 等) を直接インポートして利用すべきである。

Medium
