# Series適所配置検査官 レビュー

## 対象ファイル
`mekhane/anamnesis/cli.py`

## 判定
発言（要改善）

## 発見事項
- **S-series (Planning/Policy) の家出**: `cmd_check_freshness` は「いつ収集すべきか」というスケジューリング・判断ロジック（S3 Chronos / P3 Eukairia）を含んでいる。`anamnesis` (K3 Recollection) は「収集・検索する能力」を提供すべきであり、「収集のタイミング判断」という戦略的責務を持つべきではない。これは典型的な "家出" である。
- **責務の越境 (Sibling Coupling)**: `cmd_proactive` や `cmd_links` において、`mekhane.pks` (Personal Knowledge System) を直接 import し、その機能をラップして提供している。`anamnesis` と `pks` は並列のモジュールであり、一方が他方の CLI を兼ねる構造は責務境界を曖昧にする。統合 CLI は `scripts/` や上位の `hermeneus` に配置されるべきである。
- **P-series (State) の埋め込み**: `update_state` が直接 `state.json` を操作しており、永続化の状態管理（P4 Stasis）が CLI インタフェース層に混入している。
