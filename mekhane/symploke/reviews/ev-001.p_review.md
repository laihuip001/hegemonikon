# Series適所配置検査官 レビュー

## 対象ファイル

## 判定
発言（要改善）

## 発見事項
- **責務の越境 (Medium)**: `session_info` ツール実装において、`OchemaService` が公開している `session_info()` や `context_health()` メソッドを使用せず、プライベートメソッド `_get_ls_client()` を経由して `AntigravityClient` の内部 RPC (`GetAllCascadeTrajectories`) を直接呼び出している。これは「Vehicle (Ochēma)」による抽象化を破壊し、本来隠蔽されるべき通信詳細（プロトコル、エンドポイント）に依存している。
- **ロジックの重複 (Low)**: Context Rot (コンテキスト腐敗) のリスク判定ロジック (`step_count > 50` 等) が `AntigravityClient.context_health()` と重複している。健全性判断ロジックはサービス層または `peira/` (Health) に集約すべきであり、アダプター層である MCP サーバーにハードコードすべきではない。
