# ストア派制御審判 レビュー

## 対象ファイル
`mekhane/mcp/ochema_mcp_server.py`

## 判定
発言（要改善）

## 発見事項
- `ask_cortex` ツール定義および実装において `timeout` パラメータが欠如しており、外部API呼び出しの制御が不完全である (High)
- `async def call_tool` 内で同期メソッド `svc.ask` を呼び出しており、イベントループをブロックしている（非同期制御の放棄） (High)
- Language Server (LS) と Cortex API 間の自動フォールバック戦略（LS不可時にCortexへ、またはその逆）が実装されていない (Medium)
- `OchemaService` の取得や利用において、接続失敗時やタイムアウト時の再試行ロジック（リトライ）やサーキットブレーカーが存在しない (Medium)

## 重大度
High
