# 定理整合性監査官 レビュー

## 対象ファイル
`mekhane/ochema/cli.py`

## 判定
発言（要改善）

## 発見事項
- (Medium) 全関数 (`cmd_status`, `cmd_models`, `cmd_ask`, `cmd_chat`, `cmd_quota`, `cmd_health`, `cmd_smart_ask`, `cmd_archive`, `cmd_sessions`, `cmd_episodes`, `main`) が `[L2-auto]` (インフラストラクチャ座標) を使用しており、O/H/A/S/P/K の定理体系から逸脱しています。
- (Low) 各関数の docstring に、定理に基づいたフロントマターまたは参照が欠如しています。
- (Low) X-series (定理間の依存関係) の明示的な記述がありません。

## 修正提案
各関数の `[L2-auto]` を適切な定理座標 (例: `[H1]`, `[K1]`, `[O1]`, `[S1]` など) に置き換え、X-series を追加することを推奨します。

例:
```python
# PURPOSE: [H1] LS 接続ステータスを表示。 <- [K1] Client.connect
def cmd_status(args: argparse.Namespace) -> None:
    ...
```
