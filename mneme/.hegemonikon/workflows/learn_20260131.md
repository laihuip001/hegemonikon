## プロセス学び

### 4. B → D → C 順序の効率性

| 順序 | 理由 |
|:-----|:-----|
| B (ドキュメント) | 依存なし、即時着手可能 |
| D (コード品質) | 並行可能、環境依存なし |
| C (自動化) | 環境依存あり、最後に |

**教訓**: 依存関係が少ないものから着手。
