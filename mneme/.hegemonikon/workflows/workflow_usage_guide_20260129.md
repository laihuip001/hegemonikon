## 2. どれを使えばいいか: TPO 早見表

### 💡 状況別 推奨ワークフロー

| こんな時 | 使う WF | 理由 |
|:---------|:--------|:-----|
| **何をすべきかわからない** | `/bou` | 意志・目的を明確化 |
| **問題を深く理解したい** | `/noe` | 本質を認識 |
| **何を問うべきかわからない** | `/zet` | 問いを発見 |
| **計画を立てたい** | `/s` | 戦略設計 |
| **実装・実行したい** | `/ene` | 活動・産出 |
| **批判的に検証したい** | `/dia` | 判定・レビュー |
| **外部調査が必要** | `/sop` | Perplexity に依頼 |
| **セッション開始** | `/boot` | 状態復元 |
| **セッション終了** | `/bye` | 引き継ぎ生成 |

### 🎯 6 Hub の使い分け

| Hub | 問い | いつ使う |
|:----|:-----|:---------|
| `/o` | **何を認識・望む・問う・行うか** | 純粋な思考プロセス全般 |
| `/s` | **どのスケール・方法・基準・実践か** | 設計・計画・戦略 |
| `/h` | **何に惹かれ・信じ・欲するか** | 動機・感情の整理 |

| `/k` | **いつが好機・期限・目的・知恵か** | 時間と文脈の判断 |
| `/a` | **どう感じ・判定・原則化・知識化か** | 精度保証・検証 |

## 3. Hub vs 個別定理: いつどちらを使うか

### Hub を使う場合

```
/o  → 認識→意志→探求→行為 の4定理を巡回（4項振動）
```

**使う場面**: 問題が複雑で、どの定理が適切か自分でもわからない時

### 個別定理を使う場合

```
/noe → O1 Noēsis だけを実行
```

**使う場面**: やりたいことが明確な時

### 判断フロー

```
迷っている？
  ├─ Yes → Hub を使う（/o, /s 等）
  └─ No  → 個別定理を使う（/noe, /bou 等）
```

## 5. 派生（Derivative）の選び方

各定理には **3つの派生** がある（計72派生）。

### 例: O1 Noēsis の派生

| 派生 | 意味 | いつ使う |
|:-----|:-----|:---------|
| `nous` | 本質直観 | 抽象的な原理を掴みたい |
| `phro` | 実践判断 | 具体的な状況で判断したい |
| `meta` | メタ認識 | 自分の認識を振り返りたい |

### 派生の指定方法

```
/noe             → 自動選択（FEP が状況判断）
/noe@nous        → 強制的に nous モード
/noe --mode=meta → 強制的に meta モード
```

**推奨**: 最初は自動選択に任せ、慣れたら手動指定

## 7. X-series: 次に何をすべきか

実行後、X-series が次の WF を推奨する。

```
/noe 実行後 → 推奨: /s (認識後に設計)
/bou 実行後 → 推奨: /zet (意志後に問いを立てる)
/ene 実行後 → 推奨: /dox (活動後に記録)
```

**使い方**: 出力の「次の推奨」を参考にするか、`/x` で全関係を確認

## 9. トラブルシューティング

### Q: どの WF を使えばいいかわからない

→ `/bou-` で「今日何をしたいか」を1行で出力

### Q: Hub と個別定理の違いがわからない

→ Hub = 4定理の巡回、個別 = 1定理のみ。迷ったら Hub。

### Q: 派生が多すぎて覚えられない

→ 覚えなくていい。自動選択に任せる。

### Q: CCL が複雑すぎる

→ まず `+`, `-`, `_` の3つだけ覚える。
