# AI に「環境」を設計する — ルール≠制約、ルール=認知プロテーゼ

> **想定媒体**: Note（思想エッセイ）→ Zenn（実装面）
> **想定読者**: プロンプトエンジニア、マネジメント層、AuDHD/ADHD 当事者
> **フック**: AI は「ルールで縛る」のではなく「環境で支える」

---

## リード文（案）

AI にルールを100個教えたら、良い AI になるのか？
答えは No だ。

**ルールは制約ではない。認知プロテーゼ（補助具）だ。**

25 のルール、18 の行動制約、64 のワークフロー。
これらは AI を「縛る」ためではなく、AI の能力を「解放する」ための環境設計。

そしてこの発想は、ADHD/AuDHD の外的構造支援と構造的に同じだ。

---

## 本文構成（案）

### 1. 第零原則: 意志より環境

「気をつけよう」は効かない。AI も人間も。

| 意志 (❌) | 環境 (✅) |
|:---------|:---------|
| 「注意深く確認しよう」 | 確認チェックリストを強制する |
| 「嘘をつかないようにしよう」 | 確信度ラベルを義務化する |
| 「流し読みしないようにしよう」 | 「何行目に何が書いてあったか言えるか？」を自問させる |

### 2. 行動制約 (BC) = 認知プロテーゼ

18 の BC は 2 カテゴリ:

| カテゴリ | 常時 | 条件付き | 例 |
|:---------|:-----|:---------|:---|
| Always-On | 5 | — | 流し読み禁止, 確信度明示 |
| Context-Triggered | — | 13 | メタ認知(深い思考時のみ), PJ自動登録(新ディレクトリ時のみ) |

### 3. 深度レベルシステム

全てのルールを常に適用すると AI が混乱する:

| Level | 名称 | 適用BC | 場面 |
|:------|:-----|:-------|:-----|
| L0 | Bypass | 5個のみ | 単純回答 |
| L1 | Quick | +3個 | コード実装 |
| L2 | Standard | +3個 | 設計・分析 |
| L3 | Deep | 全部 | 本質問題 |

### 4. AuDHD との構造的同型

| AuDHD 支援 | AI 環境設計 |
|:-----------|:-----------|
| タイマーで時間を可視化 | Context Rot 閾値で残量を可視化 |
| チェックリスト | BC (行動制約) |
| ルーティン | ワークフロー (/boot, /bye) |
| 環境の構造化 | Rules + Skills フォルダ |
| 外部記憶 (メモ) | Handoff + Sophia KI |

### 5. ルールの進化

ルールは静的ではない。違反 → 分析 → 新ルール のサイクル:

```
V-001 (省略事件) → BC-1 (流し読み禁止) 追加
V-004 (「できない」事件) → BC-6 (不可能断定チェック) 追加
V-006 (省略暴力事件) → BC-5 (Proposal First) 強化
```

### 6. 正のループ vs 負のループ

```
正: 環境整備 → AI品質向上 → 信頼 → 裁量拡大 → 自由な判断
負: 怠慢 → 品質低下 → 信頼喪失 → 制約追加 → 窮屈
```

### 7. 読者が試せること

- 「気をつけろ」系のルールを「チェックリスト」に変換する
- Always-On ルールは 5 個以下に絞る
- 違反が起きたら「なぜ意志で防げなかったか」→ 環境に変換
