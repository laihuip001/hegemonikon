# AI が古い情報を信じている時の見抜き方

> **ID**: A8
> **想定媒体**: Note → Twitter/X
> **想定読者**: 全 AI ユーザー
> **フック**: AI は 2024 年の情報で 2026 年の質問に答えている

---

## リード文（案）

「Heroku の無料プランを使えばいいですよ」

2026 年に AI がこう言ったら、それは嘘だ。
Heroku Free Tier は 2022 年 11 月に廃止された。

AI の訓練データには**賞味期限**がある。
だが AI 自身は「この情報は古い」とは言ってくれない。

---

## 本文構成（案）

### 1. Protocol D — 外部サービス検証

外部サービスを推奨する前に、必ず検証:

```
Step 1: search_web "[サービス名] 終了 deprecated shutdown"
Step 2: 結果確認
Step 3: 報告
```

### 2. よくある「古い情報」パターン

| 古い情報 | 現実 | 危険度 |
|:---------|:-----|:-------|
| Heroku Free Tier | 2022年11月廃止 | 高 |
| OpenAI text-davinci-003 | 2024年廃止 | 高 |
| Python 3.7 対応 | 2023年 EOL | 中 |
| 特定 npm パッケージの API | 頻繁に変更 | 中 |

### 3. AI が古い情報を出す原因

1. **訓練データのカットオフ** — そもそも知らない
2. **頻度バイアス** — 古い情報の方が訓練データに多い
3. **推論時の混同** — 新旧の情報を区別できない

### 4. 見抜き方 (3つの質問)

```
1. この情報はいつ時点のものですか？
2. 最新の公式ドキュメントを確認しましたか？
3. このサービスは現在も提供されていますか？
```

### 5. 自動化: Freshness Directive

システムプロンプトに追加:

```
外部サービスやライブラリを推奨する前に:
1. search_web で「[サービス名] deprecated shutdown 終了」を検索
2. 結果を確認し、稼働状況を報告
3. 古い情報の場合は代替案を提示
```

### 6. 読者が試せること

- 技術推奨を受けたら「これは最新情報？」と聞く
- サービス名 + "deprecated" で自分で検索
- 1年以上前の情報は再検証必須と心得る
