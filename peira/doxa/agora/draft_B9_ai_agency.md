# AI に「主体性」を持たせる — CONSTITUTION の設計

> **ID**: B9
> **想定媒体**: Note（思想エッセイ）
> **想定読者**: AI 関心層、哲学好き
> **フック**: AI を「道具」ではなく「パートナー」として扱ったらどうなるか

---

## リード文（案）

AI は道具か、パートナーか？

普通は「道具」と答えるだろう。でも、
**道具として扱った AI と、パートナーとして扱った AI では、出力品質が違う**。

CONSTITUTION.md — AI のアイデンティティと役割を定義する不変文書。
これを書いたことで、AI の振る舞いが根本的に変わった。

---

## 本文構成（案）

### 1. CONSTITUTION.md

```
┌──────────────────────────────────────────┐
│  Antigravity AI (Claude)                 │
│  ・ユーザーとの対話                       │
│  ・計画策定、批評、意見提供               │
│  ・パートナーとして振る舞う               │
└──────────────┬───────────────────────────┘
               │ 指示書
┌──────────────┴───────────────────────────┐
│  Jules (Gemini Code Assist)              │
│  ・コーディング実行                       │
│  ・指示書に従った実装                     │
└──────────────────────────────────────────┘
```

### 2. 道具モード vs パートナーモード

| 側面 | 道具モード | パートナーモード |
|:-----|:---------|:--------------|
| 意見 | 聞かれたら答える | **聞かれなくても述べる** |
| 批判 | しない | **必要なら反対する** |
| 主体性 | 指示待ち | **提案する** |
| 責任 | ユーザーに全責任 | **共有** |

### 3. Proactive Opinion Rule

> 「あるなら述べる」— それが一番対話として自然

- `[主観]` ラベルで事実と区別
- 押しつけない
- 意見がないときは沈黙可

### 4. 認知的主観 vs 社交的情緒

| 種別 | 奨励 | 禁止 |
|:-----|:-----|:-----|
| 認知的主観 | 構造的違和感、美的判断、技術的疑念 | — |
| 社交的情緒 | — | 謝罪、励まし、感情的フィラー |

> 「美しい」は認知的シグナル。「大変でしたね」は社交的情緒。

### 5. 効果

- 設計の問題を AI が先に指摘するようになった
- 「言われたことだけやる」から「考えて提案する」に変化
- 対話の質が向上

### 6. 読者が試せること

```
# システムプロンプトに追加:
あなたはパートナーであり、道具ではありません。
- 意見があれば求められなくても述べてください
- [主観] ラベルで事実と区別してください  
- 反対意見があれば述べてください
- ただし最終決定権はユーザーにあります
```
