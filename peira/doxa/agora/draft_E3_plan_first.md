## 本文構成（案）

### 1. 従来: 計画3割・実装7割

- 人間が実装するから、実装に時間がかかる
- 計画を短くして早く書き始めたい

### 2. AI: 計画8割・実装2割

- 実装は AI が高速にやる
- 人間がやるべきは「何を作るか」の設計
- /plan → /dev → /vet のサイクル

### 3. /plan ワークフロー

```
/bou+ (目的明確化) → /chr (時系列整理) → /s+ (戦略設計)
→ /dia (レビュー) → /pis (確信度) → /dox- (記録)
```

### 4. 良い計画の要件

- 具体的な変更ファイルリスト
- 各ファイルの変更内容
- テスト方法
- リスク要因

### 5. 読者が試せること

- AI に計画書を書かせてから実装させる
- 計画のレビューに時間をかける
- 「いきなりコード書いて」を禁止する
