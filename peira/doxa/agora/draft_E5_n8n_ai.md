# n8n と AI を連携させる

> **ID**: E5
> **想定媒体**: Zenn（技術記事）
> **想定読者**: ノーコード開発者、自動化エンジニア
> **フック**: ノーコード × AI で定型業務を自動化

---

## 本文構成（案）

### 1. n8n の役割

- ノーコード自動化プラットフォーム
- スケジューリング + API 連携 + データ変換
- AI の「定型処理」部分を担当

### 2. AI と n8n の棲み分け

| 処理 | ツール | 理由 |
|:-----|:-------|:-----|
| 定時実行 | n8n | CRON スケジューラ |
| API 連携 | n8n | HTTP ノード |
| 判断・分析 | AI | LLM が必要 |
| テキスト生成 | AI | LLM が必要 |

### 3. 実装例: 論文自動収集

```
n8n CRON (毎日6:00)
→ Semantic Scholar API 呼び出し
→ 新着論文フィルタリング
→ incoming/ に保存
→ Claude に /eat 指示
```

### 4. Cortex API Bridge

- n8n から直接 Gemini API を呼び出し
- Google Cloud の認証にn8nのOAuth2ノード
- レスポンスを後続ノードで処理
