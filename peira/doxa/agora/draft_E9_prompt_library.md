# プロンプトライブラリ 112 本の設計

> **ID**: E9
> **想定媒体**: Zenn（技術記事）
> **想定読者**: プロンプトエンジニア
> **フック**: 112本のプロンプトを体系的に管理・検索する

---

## 本文構成（案）

### 1. なぜライブラリが必要か

- プロンプトが増えると管理が破綻
- 「あのプロンプトどこだっけ」問題
- 重複・矛盾の発生

### 2. 112本の内訳

| カテゴリ | 本数 | 例 |
|:---------|:-----|:---|
| 分析 | 20 | 根本原因分析、SWOT |
| 設計 | 15 | アーキテクチャ、API |
| コード | 25 | レビュー、リファクタ |
| ライティング | 20 | 記事、レポート |
| 管理 | 15 | タスク分解、優先順位 |
| 専門 | 17 | 圏論、FEP、医療 |

### 3. 動的ルーティング

```
ユーザー入力 → Taxis (分類) → 該当プロンプト特定
→ コンテキスト注入 → 実行
```

### 4. Prompt Library Skill

- キーワード検索 (`/lib "レビュー"`)
- 関連モジュールの自動提示
- コンテキスト依存のプロンプト選択

### 5. Tekhne — プロンプト品質スコアラー

- プロンプトの具体性・実行可能性を自動採点
- スコアの低いプロンプトを改善候補として提示
