# C2: Cross-paper 振動消化

> **ステータス**: 設計段階 (2026-02-15)

---

## 概要

2本以上の論文を個別に消化するのではなく、
`Paper_A ~* Paper_B` として振動消化し、
論文間の構造的共鳴を検出する。

---

## 理論的基盤

### Terminal Coalgebra としての ~*

```
状態: (F(Paper_A), F(Paper_B))
遷移: 交互比較 → 共通構造抽出 → 差異分析 → 統合
不動点: Paper_A と Paper_B の "共通言語" = HGK 概念
```

### /eat との違い

| 側面 | 個別 /eat | Cross-paper ~* |
|:-----|:---------|:---------------|
| 入力 | 1本の論文 | 2本以上の論文 |
| F(X) | 論文 → HGK embedding | 2本の embedding 間の振動 |
| 出力 | 個別の随伴 η/ε | 共鳴構造 + 差異マップ |
| 失敗モード | ε ≠ 1 のみ | 収束しない (概念が直交) |

---

## プロトコル案

```
Phase 0: 圏の特定 (論文 A と B の分野)
Phase 1: 個別 F(A), F(B) を構築
Phase 2: ~* 振動
  Iteration 1: F(A) の概念を F(B) で解釈
  Iteration 2: F(B) の概念を F(A) で解釈
  ...
  収束条件: 新たな共通概念が出なくなったら停止
Phase 3: 共鳴マップ作成
  共通概念: {C1, C2, ...}
  A 固有: {A1, A2, ...}
  B 固有: {B1, B2, ...}
Phase 4: HGK 統合判定
```
