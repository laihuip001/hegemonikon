## 1. 完全対応

| Rust | Hegemonikón | 成立度 | 解説 |
|:-----|:------------|:------:|:-----|
| **Ownership** | Flow — I/A 対立 | ★★★★★ | 所有権 = 認知リソースの単一所有。I(推論)とA(行為)間のリソース移動 |
| **Borrow Checker** | S1 Metron (測定) | ★★★★★ | コンパイル時の静的検証 = 事前測定。実行前に安全性を保証 |
| **`Result<T, E>`** | H2 Pistis (確信度) | ★★★★★ | `Ok(T)` = [確信], `Err(E)` = [仮説]。エラー = 予測誤差 |
| **Cargo (ビルドシステム)** | mekhane | ★★★★★ | 依存解決 + ビルド + テスト = 認知基盤の自動管理 |
| **Pattern Matching** | A2 Krisis (判定) | ★★★★★ | match 式 = 網羅的判定。パターンの欠落 = コンパイルエラー = 判定漏れ防止 |

## 2. 高成立

| Rust | Hegemonikón | 成立度 | 解説 |
|:-----|:------------|:------:|:-----|
| **Lifetime `'a`** | K2 Chronos (時間) | ★★★★☆ | 参照の生存期間 = 認知リソースの有効期間。スコープ = 時間的境界 |
| **Trait** | Hub WF (6 Series) | ★★★★☆ | 振る舞いの抽象化 = Series の共通インターフェース。`impl Trait` = 定理の具体化 |
| **`unsafe`** | BC-5 LBYL / I-1 | ★★★★☆ | 安全保証を明示的に外す = BC-5 の破壊的操作承認。「私が責任を取る」宣言 |
| **Enum (ADT)** | 定理の 2×2 マトリクス | ★★★★☆ | 代数的データ型 = 直積(struct) × 直和(enum)。定理群の T1-T4 構造と同型 |
| **Zero-Cost Abstractions** | 第零原則 (意志＜環境) | ★★★★☆ | 高レベル抽象に実行時コストなし = 環境設計に認知コストなし |
