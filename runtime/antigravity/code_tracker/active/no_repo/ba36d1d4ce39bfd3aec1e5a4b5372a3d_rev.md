# /rev ワークフロー

> **Hegemonikón Module**: M7 Dokimē (検証) + M8 Anamnēsis

過去7日間のチャット履歴と現在のコンテキストをスキャンし、「今日のレビュー」を生成する。

## 入力ソース

1. `/vault/chat-history/` 内の直近7日間のファイル
2. `current.md`（現在のコンテキスト）
3. カレンダー情報（Phase 2で追加予定）

## タスク抽出ロジック

### 高確度シグナル（抽出対象）
- "ToDo", "やります", "やる", "する予定"
- "期限", "締め切り", "までに"
- "確認", "チェック", "検証"
- 明示的なコミットメント表現

### 除外シグナル
- "完了", "done", "終わった"
- "情報収集のみ", "参考まで"
- 過去形での報告

## 出力形式

```markdown
# 📋 Today's Review (YYYY-MM-DD)

## 🔴 要アクション（Urgent）
<!-- 期限あり or 明示的コミット -->
- [ ] [タスク1] — 期限: [日付]
- [ ] [タスク2] — ソース: [会話リンク]

## 🟡 フォローアップ候補（Follow-up）
<!-- 暗黙的 or 保留中 -->
- [ ] [タスク3] — 検討中
- [ ] [タスク4] — 追加情報待ち

## 🟢 完了確認待ち（Verify）
<!-- 報告済みだが未確認 -->
- [ ] [タスク5] — 完了報告あり、要確認

## 💡 提案（Suggestions）
<!-- プロアクティブな提案 -->
- [提案1]: [根拠]
- [提案2]: [根拠]

---
┌─[Hegemonikón]──────────────────────┐
│ M7 Dokimē: レビュー検証完了        │
│ スキャン期間: 7日間                │
│ タスク抽出: [N]件                  │
│ 信頼度: [高/中/低]                 │
└────────────────────────────────────┘
*Generated by Hegemonikón Phase 1*
```

## 制約

- 各セクション最大5件
- 重複タスクはマージ
- 7日以上前のタスクは「長期保留」として別枠

## 実行後

生成されたレビューを表示し、ユーザーに確認を求める。
